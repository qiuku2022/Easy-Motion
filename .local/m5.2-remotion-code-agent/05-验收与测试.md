## 手动验收（需 API Key + 打开用户项目）

| ID | 操作 | 预期 |
|----|------|------|
| H1 | 打开任意 `.em` 项目，AI 输入「在 components/custom 写一个全屏红色背景的 Remotion 组件 RedBg，并加到时间线」 | 预览全屏红底 |
| H2 | 「把 RedBg 改成蓝色」 | patch 后预览变蓝 |
| H3 | 输入含 `eval` 的生成请求 | 拒绝写入，对话提示 E2408 |
| H4 | 仅「标题 Hello」（无组件关键词） | 走 timeline，不新建 tsx |
| H5 | 未打开项目发消息 | E2105 |
| H6 | compile 故意写错类型 | Agent 收到错误并尝试修正（或提示失败） |

Live 脚本：`node scripts/test-remotion-code-agent-live.js`（待建，类似 test-agent-live.js）

## 与 M5 eval 的关系

| 集合 | 路径 | 测什么 |
|------|------|--------|
| Timeline eval | `docs/agent-eval/tasks.json` | timeline-ops 黄金路径 |
| Remotion code eval | `docs/agent-eval/remotion-code-tasks.json` | 沙箱读写 + registry |

两者独立；`pnpm test:m5.2` 只跑后者 + 沙箱单测。
