# 00 — 概述与目标

## 背景

M5 已完成 **Timeline Agent**：通过 10 个工具修改 `subproject.json` 中的 timeline JSON，驱动预览（timeline 驱动 `MainSequence` 或 Generator 生成 TSX）。

用户实际使用时：

- Remotion 项目位于**用户磁盘**任意目录，例如 `D:\Videos\MyIntro.em\subprojects\default\remotion\`
- 与 EasyMotion **开发仓库**（`d:\Work\Easy-Motion\`）无关
- 内置 `resources/templates/default-project` 仅在**新建项目**时复制 scaffold，之后以用户目录为准

**缺口**：用户说「写一个粒子片头」「仿这张海报的 Remotion 动画」时，timeline 工具表达能力不足；Agent **无法读写** `remotion/src` 下的 TSX。

## 目标

1. AI 助手可**读/写/创建**当前打开用户项目内的 Remotion 源码（沙箱内）
2. 写入后自动 **security-scan + compile 检查**，失败则回滚、错误反馈给 Agent 重试
3. 新组件可**挂到 timeline**（animation 轨道）并在预览中可见
4. 与现有 Timeline Agent **并存**：简单编辑走 timeline，自由创作走 Remotion Code

## 非目标

- ❌ 修改 EasyMotion 软件源码（`apps/electron` 等）— 那是 Cursor 开发者的事
- ❌ 修改用户项目外的路径（其他 `.em`、系统目录）
- ❌ 无校验地覆盖 `MainSequence.tsx` / `presets/registry.ts`（自动生成文件）
- ❌ 在 M5.2 第一版实现完整 Planner/Evaluator 多 Agent（留 M5.3）

## 用户可见能力（M5.2 完成后）

| 用户说法 | Agent 行为 |
|----------|------------|
| 「创建一个标题写着 Hello」 | Timeline Agent（现有） |
| 「写一个粒子背景组件，全屏红色粒子」 | 写 `components/custom/ParticleBg.tsx` → 注册 → apply 到 timeline |
| 「把 MyIntro 组件的粒子数量改成 200」 | `readRemotionFile` → `patchRemotionFile` → compile check |
| 「读取当前 Remotion 项目有哪些组件」 | `listRemotionFiles` |

## 创作模式（UI，可选 M5.2b）

| 模式 | 路由 |
|------|------|
| 快速编辑（默认） | 仅 Timeline Tools |
| 自由创作 | Timeline + Remotion Code Tools |
| 自动 | Router 根据关键词（「组件」「TSX」「interpolate」等）选路 |

第一版可**默认开启 Remotion Tools**，由 system prompt 决定何时使用。

## 成功标准

- [ ] 对用户项目 `remotion/src/components/custom/` 写入 TSX 后，预览 HMR 或重载可见
- [ ] 含 `eval(` 的代码被拒绝写入（E2408）
- [ ] TypeScript/Remotion 编译失败时不落盘或回滚，Agent 收到错误信息
- [ ] 不打开项目时行为与 M5 一致（E2105）
- [ ] `pnpm test:m5.2` 全绿（无 API Key）
