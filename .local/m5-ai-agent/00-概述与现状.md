# 00 — 概述与现状

## 1. M5 目标

实现对话式 AI 交互：用户通过自然语言**生成和调整** Remotion 动画，而非仅获得文字建议。

对齐路线图 §M5 四大交付块：

1. **对话面板 UI** — 聊天、消息类型、图片上传、收起/展开
2. **LLM 服务集成** — 代理、流式、Key 安全、设置面板
3. **Agent 工作流** — 意图解析、工具调用链、视觉解析、代码校验
4. **对话调整** — 参数微调、元素定位、冲突检测

## 4. 总体架构

```
渲染进程                          主进程
─────────                        ─────
AIAssistantPanel                 agent-service.js
    ↓                                ↓
conversationStore ──IPC──→  conversation-service → conversation.json
    ↓                                ↓
timelineStore ←──刷新──    timeline-tools.js → timeline-service
    ↓                                ↓
预览自动更新               generator → remotion HMR
```

**原则**：Agent 逻辑放**主进程**（timeline 写权限、generator、文件服务）；渲染进程负责 UI + store + 预览刷新。

## 5. 技术决策（M5 锁定）

| 决策 | 选择 | 理由 |
|------|------|------|
| Agent 框架 | 自研 `agent-service` + 原生 tool schema | 避免 LangChain 重量；后期可替换 |
| LLM 协议 | Anthropic tool_use + OpenAI functions 双支持 | MiniMax / Claude / OpenAI 兼容 |
| 对话入口 IPC | `main:conversation:send` 替代 UI 直调 `llm:stream` | Agent 需要编排多步工具 |
| UI 位置 | 右侧 Tab（已有） | M4 已接入；BottomBar 延后 |
| 降级策略 | 模板硬编码 `createTrack + createClip("Hello")` | 对齐技术规格「简化模式」 |
