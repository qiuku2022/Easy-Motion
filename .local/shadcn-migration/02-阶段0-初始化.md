## 2. Tailwind v3 与官方文档差异（重要）

shadcn 官方 Vite 文档部分示例针对 **Tailwind CSS v4**（`@import "tailwindcss"`、`@tailwindcss/vite`）。

**本项目保持 Tailwind v3**，初始化时注意：

- `components.json` → `tailwind.config` 应指向 `tailwind.config.js`
- `tailwind.css` 应指向 `src/renderer/src/index.css`
- **不要**在未评估的情况下把 `index.css` 整文件替换为 v4 模板
- init 后人工 **合并** CSS，保留现有字体 import、`.scrollbar-hidden`、时间线 keyframes、`prefers-reduced-motion`

---

## 3. 执行 `shadcn init`

```powershell
cd apps/electron
npx shadcn@latest init
```

### 3.1 推荐交互选项

| 提示项 | 推荐值 | 理由 |
|--------|--------|------|
| Framework | Vite | 已使用 |
| TypeScript | Yes | `tsx: true` |
| Style | **new-york** | 官方推荐（`default` 已废弃） |
| Base color | neutral / zinc | 后续用自定义 CSS 变量覆盖为 Cinema Dark |
| CSS variables | **Yes** | 与 UI 文档、MASTER.md 一致 |
| RSC | **No** | Electron 客户端渲染，非 Next.js |
| `components.json` 位置 | `src/renderer/src/components.json` 或 CLI 默认 | 需与 vite root 一致 |
| `utils` alias | `@/lib/utils` | 已有 |
| `ui` alias | `@/components/ui` | 标准 |

### 3.2 init 预期产出

- `src/renderer/src/components.json`
- 更新 `tailwind.config.js`（可能追加 `theme.extend.colors` 语义色、`tailwindcss-animate`）
- 更新 `index.css`（`@layer base` 等 — **需合并**）
- 可能更新/确认 `lib/utils.ts`

### 3.3 `components.json` 目标形态（参考）

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "rsc": false,
  "tsx": true,
  "tailwind": {
    "config": "tailwind.config.js",
    "css": "src/renderer/src/index.css",
    "baseColor": "neutral",
    "cssVariables": true,
    "prefix": ""
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  },
  "iconLibrary": "lucide"
}
```

> **路径说明**：因 Vite `root` 为 `src/renderer`，CLI 询问路径时以 **相对于 `apps/electron`** 为准；若生成错误，参照 [components.json 文档](https://ui.shadcn.com/docs/components-json) 手动修正 `tailwind.css` 与 `config`。

## 4. init 后必做合并项

### 4.1 保留 `index.css` 中的业务样式

合并后必须仍存在：

```css
@import url("https://fonts.googleapis.com/css2?family=JetBrains+Mono...");

.scrollbar-hidden { ... }
.clip-floating-toolbar-above { ... }
.clip-floating-toolbar-below { ... }
@media (prefers-reduced-motion: reduce) { ... }
```

### 4.2 保留 `tailwind.config.js` 中的 `em` 色板（过渡期）

init 可能改写 `content` 路径，确认包含：

```js
content: ["./src/renderer/index.html", "./src/renderer/src/**/*.{ts,tsx}"]
```

### 4.3 安装动画插件

若 init 未自动添加：

```powershell
pnpm add -D tailwindcss-animate
```

```js
// tailwind.config.js
plugins: [require("tailwindcss-animate")],
```

## 6. 根级 Provider 骨架（阶段 4 前可先占位）

依据 [Dark Mode - Vite](https://ui.shadcn.com/docs/dark-mode/vite)：

**v1.0 策略**：`index.html` 已写死 `class="dark"`，**可不引入 ThemeProvider**；v1.1 做 light 主题时再添加。

阶段 4 需在 `App.tsx` 或 `main.tsx` 添加：

```tsx
import { TooltipProvider } from "@/components/ui/tooltip";
// 阶段 4 后
import { Toaster } from "@/components/ui/sonner";
```

## 8. 常见问题

| 问题 | 处理 |
|------|------|
| `Cannot resolve @/components/ui/button` | 检查 `vite.config.ts` alias 与 `tsconfig.renderer.json` paths |
| init 覆盖了整个 `index.css` | 从 git 恢复后手动合并 shadcn `@layer base` 块 |
| CLI 检测到 Tailwind v4 | 本项目为 v3，手动指定 `tailwind.config.js` |
| React 19 peer dependency 警告 | 项目为 React 18，一般可忽略；若失败见 [React 19 说明](https://ui.shadcn.com/react-19) |
| monorepo 路径混乱 | 始终在 `apps/electron` 执行，或使用 `-c apps/electron` |
