# 10 — 风险、回滚与排期

> 参考：[技术风险分析](../../docs/requirements/技术风险分析.md) · [开发里程碑与路线图](../../docs/requirements/开发里程碑与路线图.md)

---

## 1. 风险矩阵

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| `shadcn init` 覆盖 `index.css` / tailwind 配置 | 中 | 高 | init 前建 git 分支；合并而非覆盖；见阶段 0 |
| Tailwind v3 vs 官方 v4 文档不一致 | 中 | 中 | 坚持 v3；手动指定 `tailwind.config.js` |
| z-index 浮层被时间线遮挡 | 中 | 中 | `z-[90]`/`z-[100]` 显式覆盖 |
| 表单双写 store 与 RHF 状态不一致 | 中 | 中 | store 为 truth；`reset` on selection change |
| `ContextMenu` API 从坐标式改包裹式工作量大 | 高 | 中 | 分步迁移；保留旧 API 至调用方改完 |
| `Resizable` 替换破坏面板状态 | 中 | 高 | 先样式迁移（方案 A），后结构迁移 |
| bundle 体积增加（Radix） | 低 | 低 | 按组件安装；tree-shaking |
| 与 Electron CSP 冲突 | 低 | 低 | shadcn 无外部 CDN；已允许 Google Fonts |

## 2. 回滚策略

### 2.1 分支策略

```text
main
 └── feat/shadcn-migration
      ├── phase-0-init
      ├── phase-1-theme
      ├── phase-2-primitives
      └── ...
```

每阶段合并前必须通过 [09-验收与回归.md](./09-验收与回归.md) 核心项。

### 2.2 阶段回滚

| 阶段 | 回滚方式 |
|------|----------|
| 0 init | 删除 `components.json`、`components/ui/`、恢复 tailwind/css |
| 1 主题 | 恢复 `tailwind.config.js` `index.css` |
| 2–5 组件迁移 | 按文件 `git checkout` 业务组件 |
| 依赖 | `pnpm remove` 对应 `@radix-ui/*` 包 |

### 2.3 保留 `em-*` 别名

过渡期内保留 `em` 色板映射，可在 shadcn 迁移失败时快速回退视觉而不改 25 个文件。

## 5. 与里程碑关系

[开发里程碑与路线图](../../docs/requirements/开发里程碑与路线图.md) 中前端技术栈已声明 shadcn/ui。本迁移不单独占里程碑编号，但应：

- 在 **M3/M4 预览与时间线 UI 稳定前** 完成 primitive 层（阶段 0–3）
- 在 **属性面板 / 对话面板扩展前** 完成表单层（阶段 5）
- 避免与新功能并行改同一文件（如同时做 `PropertiesPanel` 新字段 + shadcn 迁移）
