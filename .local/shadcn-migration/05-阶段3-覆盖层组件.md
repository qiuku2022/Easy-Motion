## 3. z-index 协调

[MASTER.md](../../docs/design-system/easymotion/MASTER.md) 定义：

| 层 | z-index |
|----|---------|
| Modal / Dialog | 100 |
| Dropdown / ContextMenu | 90 |
| Tooltip | 80 |
| Floating toolbar | 60 |

shadcn/Radix 默认 z-index 通常为 `z-50`。若与时间线浮层（`z-[60]`）冲突：

- 通过 `className="z-[90]"` 覆盖 `DropdownMenuContent` / `ContextMenuContent`
- Dialog 使用 `z-[100]`
- **不要**全局改 `components/ui` 源码；用 wrapper 或 `className` 传入

## 4. 样板 A：`PanelTabs` → `Tabs`

**文件**：`components/common/PanelTabs.tsx`  
**引用**：`LeftPanel.tsx`, `RightPanel.tsx`

### 要点

- 使用 [Tabs](https://ui.shadcn.com/docs/components/tabs) 受控模式：`value` + `onValueChange`
- 面板风格：无圆角底边、`TabsList` 透明背景，匹配 Premiere 风格

```tsx
<Tabs value={active} onValueChange={(v) => onChange(v as T)}>
  <TabsList className="h-9 w-full justify-start rounded-none border-b border-border bg-transparent p-0">
    {tabs.map((tab) => (
      <TabsTrigger
        key={tab.id}
        value={tab.id}
        className="rounded-t-sm data-[state=active]:bg-muted data-[state=active]:text-foreground"
      >
        {tab.label}
      </TabsTrigger>
    ))}
  </TabsList>
</Tabs>
```

### 注意

- 若父组件需要 `TabsContent`，当前 `PanelTabs` 仅渲染 tab 条；保持 API 不变，内容仍在父级按 `active` 切换
- 设置 `defaultValue` 或受控 `value`（[官方建议](https://ui.shadcn.com/docs/components/tabs)）

## 5. 样板 B：`AddTrackMenu` → `DropdownMenu`

**文件**：`components/timeline/AddTrackMenu.tsx`

### 删除的逻辑

- `useState(open)` + `useEffect` pointerdown 关闭
- 手写 `role="menu"` div

### 替换结构

```tsx
<DropdownMenu>
  <DropdownMenuTrigger asChild>
    <Button variant="outline" size="sm" className="h-7 gap-1 rounded-sm text-xs">
      <Plus className="h-3.5 w-3.5" />
      轨道
      <ChevronDown className="h-3 w-3 opacity-60" />
    </Button>
  </DropdownMenuTrigger>
  <DropdownMenuContent align="start" className="z-[90] min-w-[9rem]">
    {ADD_TRACK_TYPES.map((type) => (
      <DropdownMenuItem key={type} onClick={() => { ... }}>
        {TRACK_TYPE_LABELS[type]}
      </DropdownMenuItem>
    ))}
    <DropdownMenuSeparator />
    <DropdownMenuItem onClick={() => { ... }}>…</DropdownMenuItem>
  </DropdownMenuContent>

```

### 官方准则

- 动作用 `DropdownMenu`，不要用 `Popover` 冒充菜单（[准则](https://ui.shadcn.com/docs/components/dropdown-menu)）
- 使用 `DropdownMenuSeparator` 分组

## 6. 样板 C：`ContextMenu` → shadcn `ContextMenu`

**文件**：`components/common/ContextMenu.tsx`

### 策略

当前 API：`{ x, y, items, onClose }` 由父组件传坐标。两种方案：

| 方案 | 说明 | 推荐 |
|------|------|------|
| A. 保留坐标 API | 内部用 Radix `ContextMenu` + 虚拟 trigger | 改动大 |
| B. 改为包裹式 API | `<ContextMenuWrapper items={...}>{children}</ContextMenuWrapper>` | **推荐** |

### 方案 B 结构

```tsx
<ContextMenu>
  <ContextMenuTrigger asChild>{children}</ContextMenuTrigger>
  <ContextMenuContent className="z-[90]">
    {items.map((item) => (
      <ContextMenuItem
        key={item.id}
        disabled={item.disabled}
        variant={item.danger ? "destructive" : "default"}
        onClick={item.onClick}
      >
        {item.label}
      </ContextMenuItem>
    ))}
  </ContextMenuContent>
</ContextMenu>
```

### 迁移步骤

1. 新增 `ContextMenuWrapper.tsx`（shadcn 包裹式）
2. 逐步替换调用方（`TrackHeader`, `ClipBlock` 等）从坐标式改为包裹式
3. 坐标式 `ContextMenu.tsx` 标记 deprecated，全部替换后删除
