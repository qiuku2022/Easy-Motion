# 06 — 阶段 4：布局与导航

> 官方： [Tabs](https://ui.shadcn.com/docs/components/tabs) · [Scroll Area](https://ui.shadcn.com/docs/components/scroll-area) · [Resizable](https://ui.shadcn.com/docs/components/resizable) · [Sonner](https://ui.shadcn.com/docs/components/sonner)

## 1. 目标

统一面板滚动、尺寸调整与全局 Toast；对齐 [UI 文档](../../docs/requirements/UI布局与交互设计-优化版.md) 默认面板尺寸。

---

## 2. 安装

```powershell
npx shadcn@latest add scroll-area resizable sonner
```

---

## 3. Scroll Area

### 适用文件

- `assets/AssetsPanel.tsx`
- `project/ProjectPanel.tsx`
- `properties/PropertiesPanel.tsx`
- `layout/LeftPanel.tsx` / `RightPanel.tsx` 内容区

### 与时间线 `.scrollbar-hidden` 的关系

| 区域 | 策略 |
|------|------|
| 素材库、项目树、属性列表 | `ScrollArea` |
| 时间线横向滚动 | **保留** `.scrollbar-hidden`（Premiere 风格，[UI 文档](../../docs/requirements/UI布局与交互设计-优化版.md)） |

```tsx
<ScrollArea className="h-full">
  <div className="p-2">{children}</div>
</ScrollArea>
```

## 4. Resizable 面板

### 适用文件

`layout/PanelResizer.tsx` + `layout/AppLayout.tsx`

[MASTER.md](../../docs/design-system/easymotion/MASTER.md) 默认尺寸：

| 面板 | 默认 | 最小 | 最大 |
|------|------|------|------|
| Left | 220px | 160 | 350 |
| Right | 280px | 200 | 400 |
| Timeline | 220px | 150 | 350 |

### 迁移策略

**方案 A（推荐）**：保留现有 `PanelResizer` 拖拽逻辑（已调优），仅将视觉样式改为 shadcn token（`bg-border hover:bg-ring/50`）

**方案 B**：整体改用 `ResizablePanelGroup` + `ResizablePanel` + `ResizableHandle`

```tsx
<ResizablePanelGroup direction="horizontal">
  <ResizablePanel defaultSize={15} minSize={12} maxSize={25}>
    <LeftPanel />
  </ResizablePanel>
  <ResizableHandle />
  <ResizablePanel defaultSize={55}>...</ResizablePanel>
  ...
</ResizablePanelGroup>
```

> 方案 B 需验证与现有 `AppLayout` 状态（折叠、最小宽度）的兼容性；建议先 A 后 B。

## 5. Sonner Toast

依据 [Sonner 文档](https://ui.shadcn.com/docs/components/sonner)：

### 安装后

```tsx
// main.tsx
import { Toaster } from "@/components/ui/sonner";

createRoot(root).render(
  <StrictMode>
    <TooltipProvider>
      <App />
      <Toaster theme="dark" position="bottom-right" />
    </TooltipProvider>
  </StrictMode>
);
```

### 使用场景（对齐组件库清单）

| 场景 | API |
|------|-----|
| 保存成功 | `toast.success("项目已保存")` |
| 导入失败 | `toast.error("导入失败", { description: err })` |
| Remotion 同步 | `toast.loading` → `toast.success` |

**不要**在每个页面单独放 `<Toaster />`（官方要求全局唯一）。

## 6. 执行步骤

1. 安装 scroll-area, resizable, sonner
2. `main.tsx` 挂载 `Toaster`
3. `AssetsPanel` / `ProjectPanel` 接入 `ScrollArea`
4. 评估 `PanelResizer` 方案 A 或 B
5. 在主进程 IPC 错误回调处（如有 UI 层）试用 `toast.error`
