# 08 — 阶段 6：时间线边界（保持自研）

> 参考：[时间线编辑.md](../../docs/requirements/时间线编辑.md) · [UI布局与交互设计-优化版](../../docs/requirements/UI布局与交互设计-优化版.md)

---

## 1. 原则

时间线是 EasyMotion 的核心交互面，**性能与帧精度优先于组件库统一**。本阶段明确「不迁移」与「仅换 primitive」的边界，防止过度 shadcn 化。

---

## 2. 完全不迁移（保持自研 DOM/Canvas）

| 组件 | 原因 |
|------|------|
| `TimelineBody.tsx` | 拖拽、框选、滚轮缩放、吸附计算 |
| `ClipBlock.tsx` | 片段几何、选中态、裁剪手柄 |
| `Playhead.tsx` | 帧级定位，禁止 CSS transition |
| `TimelineRuler.tsx` | 刻度尺绘制 |
| `SnapGuides.tsx` | 吸附参考线 SVG |
| `TimelineMarkers.tsx` | 标记点渲染 |

这些文件可继续使用 `em-*` 或逐步改为语义色，但 **不使用** shadcn 的 `Card`/`Slider` 等包裹整体结构。

---

## 3. 仅替换内部控件

| 组件 | 可换 shadcn 部分 | 保留自研部分 |
|------|------------------|--------------|
| `TrackHeader.tsx` | 图标按钮 → `Button`；显隐 → `Switch` | 轨道拖拽排序、重命名 inline edit |
| `ClipFloatingToolbar.tsx` | `Button`, `Separator`, `Tooltip` | 定位算法、动画 class |
| `TimelineTransport.tsx` | `Button` | 时间码显示、与 store 同步 |
| `TimelineZoomControls.tsx` | `Button` | 缩放逻辑 |
| `AddTrackMenu.tsx` | 已完成 `DropdownMenu` | — |
| `TimelineDevMenu.tsx` | `DropdownMenu` | dev-only 动作 |

---

## 4. 性能约束

依据 [非功能需求](../../docs/requirements/非功能需求.md) 与编辑器实践：

- 时间线区域避免引入重量级 Context（如整树 `FormProvider`）
- 浮层组件（Toolbar、Menu）懒挂载或条件渲染
- 不对 `ClipBlock` 列表项包裹 `Tooltip`（N 个 clip 时性能差）；仅工具栏图标使用 Tooltip
- 拖拽过程禁用不必要的 React re-render（现有 zustand 选择器模式保持）

---

## 5. 动画约束

[UI 文档 §0.5](../../docs/requirements/UI布局与交互设计-优化版.md) / [MASTER.md](../../docs/design-system/easymotion/MASTER.md)：

| 元素 | 动画 |
|------|------|
| 播放头 | **无** transition |
| 片段浮动工具栏 | 保留现有 200ms opacity/transform keyframes |
| 按钮 hover | 150ms color only |
| shadcn 菜单进入 | Radix 默认，受 `prefers-reduced-motion` 影响 |

---

## 6. 快捷键与焦点

[可访问性设计 §2](../../docs/requirements/可访问性设计.md)：

- 时间线焦点在 `TimelineBody` 时，播放快捷键由 `useTimelineShortcuts` 处理
- 打开 `DropdownMenu` / `ContextMenu` 时，Radix 管理焦点陷阱；关闭后应恢复时间线焦点（必要时 `onCloseAutoFocus` 阻止默认并 focus 容器）

```tsx
<DropdownMenuContent onCloseAutoFocus={(e) => e.preventDefault()} />
```

仅在需要时添加，避免破坏 Radix 默认 a11y。
