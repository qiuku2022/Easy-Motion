# 00 — 概述与原则

## 1. 迁移目标

将 EasyMotion Electron 渲染进程（`apps/electron/src/renderer`）的 UI 层与开发文档声明的技术栈对齐：

| 文档声明 | 当前实现 | 迁移后 |
|----------|----------|--------|
| React + shadcn/ui + Tailwind | React + Tailwind + 自研 primitive | React + shadcn/ui + Tailwind |
| CSS 变量暗色主题（`.dark`） | 部分变量 + `em-*` 硬编码色板 | 语义 token 为主，`em-*` 仅作过渡别名 |
| Radix 无障碍行为 | 手写 focus/escape/portal | shadcn（Radix）标准行为 |

**不在本次迁移范围：**

- Remotion 子项目模板内的组件（`resources/templates/.../remotion/`）
- Electron 主进程（`src/main/`）
- Python FastAPI 服务
- 尚未实现的 UI（如 `BottomConversationBar`、`TopMenuBar`）— 新功能直接用 shadcn 开发

## 3. 设计原则（迁移时必须遵守）

### 3.1 视觉不变优先

v1.0 仅 dark 主题（`index.html` 已设 `class="dark"`）。迁移后用户可见差异应极小；允许的差异仅限：

- 焦点环更清晰（符合 a11y）
- 菜单/弹层动画与 Radix 一致
- 字体与间距保持 Plus Jakarta Sans / JetBrains Mono

### 3.2 分层替换

```
┌─────────────────────────────────────────┐
│  业务组件（TimelineBody, ClipBlock…）    │  ← 保留自研，仅换内部 primitive
├─────────────────────────────────────────┤
│  复合 UI（PanelTabs, ContextMenu…）      │  ← 优先迁移到 shadcn
├─────────────────────────────────────────┤
│  shadcn/ui primitive（Button, Input…）   │  ← CLI 安装
├─────────────────────────────────────────┤
│  设计 token（CSS 变量 + Tailwind 映射）   │  ← 阶段 1 必须先完成
└─────────────────────────────────────────┘
```

### 3.3 颜色使用规范

依据 [shadcn Theming](https://ui.shadcn.com/docs/theming)：

- **Do**：`bg-background`、`text-muted-foreground`、`border-border`、`bg-primary`
- **Don't**：组件内 `#E11D48`、`bg-blue-500` 等硬编码
- **过渡期**：`em-*` 映射到 CSS 变量，见 [03-阶段1-主题与Token.md](./03-阶段1-主题与Token.md)

### 3.4 无障碍

依据 [可访问性设计.md](../../docs/requirements/可访问性设计.md) 与 shadcn a11y 指南：

- 表单必须有 `Label` / `FormLabel`，禁止仅靠 placeholder
- `Dialog` / `AlertDialog` 必须含 `Title`（可用 `sr-only`）
- 根级包裹 `TooltipProvider`（[官方要求](https://ui.shadcn.com/docs/components/tooltip)）
- 尊重 `prefers-reduced-motion`（`index.css` 已有，迁移后保留）

### 3.5 ui-ux-pro-max 技能

执行各阶段前可运行：

```powershell
python .cursor/skills/ui-ux-pro-max/scripts/search.py "<关键词>" --stack shadcn -n 10
python .cursor/skills/ui-ux-pro-max/scripts/search.py "accessibility focus keyboard" --domain ux -n 8
```

**不要**用 `--design-system --persist` 覆盖 `docs/design-system/easymotion/MASTER.md`。

## 4. 成功标准

1. `apps/electron` 存在 `components.json` 与 `src/renderer/src/components/ui/`
2. 至少完成「阶段 3」三个样板迁移：`PanelTabs`、`AddTrackMenu`、`ContextMenu`
3. `PropertiesPanel` 使用 shadcn `Input` / `Textarea` / `Label`（阶段 5）
4. 根布局有 `TooltipProvider` + `Toaster`
5. `pnpm dev` 无报错，时间线核心交互无回归
6. 新代码不再新增 `em-*` 硬编码（旧代码可过渡）

---

## 5. 工作目录与 monorepo 说明

- **CLI 工作目录**：`apps/electron`（不是仓库根目录）
- **Vite root**：`src/renderer`（`vite.config.ts` 已配置）
- **`@/` 别名**：`src/renderer/src/*`（`tsconfig.renderer.json`）
- **Tailwind 版本**：当前为 **v3.4** + PostCSS（非官方文档示例中的 Tailwind v4 `@tailwindcss/vite`）
- **`packages/ui`**：文档规划存在，当前仅有 `packages/shared`；**v1 迁移落在 `apps/electron` 内**，后续再抽包

### shadcn monorepo 参考

官方支持从根目录指定子包：

```bash
npx shadcn@latest add button -c apps/electron
```

本项目建议在 `apps/electron` 内直接执行，避免路径歧义。
