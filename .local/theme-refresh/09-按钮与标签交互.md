## 1. 现状（代码里已有啥）

### Button（`components/ui/button.tsx`）

| 能力 | 实现 | 体感 |
|------|------|------|
| 键盘聚焦 | `focus-visible:ring-3 ring-ring/50` | Tab 键可见，**鼠标点不出现环**（符合 a11y 惯例） |
| 点击 | `active:translate-y-px` | 轻微下压 1px，neutral 主题下几乎看不出 |
| 过渡 | `transition-all` | hover 背景会变，但缺少「按下去」的层次 |

### PanelTabs（`line` 变体）

| 能力 | 实现 | 体感 |
|------|------|------|
| 选中 | `data-active` + 底边 `after:h-0.5` | 有下划线，**无滑动** |
| 聚焦 | `focus-visible:ring` | 键盘可用；鼠标点标签无明显反馈 |
| 切换 | 无 `TabsContent` 动画 | 内容瞬间切换 |
| 覆盖 | `data-active:bg-muted` | 与 line 变体略打架，选中像「灰块+线」 |

## 2. 设计原则（专业工具向）

参考 Linear / Figma / VS Code dark，**好看**通常意味着：

1. **克制** — 一次交互只动 1～2 个属性（背景、透明度、下划线位置）  
2. **分层** — hover 轻、active 稍重、focus-visible 给键盘用户  
3. **不抖布局** — 不用 scale 放大 hover；active 最多 `scale(0.98)` 或 `translateY(1px)`  
4. **统一时长** — 壳层 150ms `ease-out`；标签指示条 200ms  
5. **reduced-motion** — 指示条滑动可改为瞬时跳变

**不建议**：Material 水波纹、大面积发光、鼠标点击也常驻 focus 环（会显得「脏」）。

## 3. 按钮：三种方案

### 方案 B1 — 轻量按压（推荐默认）

适合：顶栏 ghost、outline、secondary 占多数。

```
hover:   bg-muted / 边框略亮     150ms
active:  bg-muted 更深 + opacity-90  80ms
         translateY(0.5px) 仅 default/outline（可选）
focus-visible: 保持 ring-3（仅键盘）
```

- **优点**：和 Neutral 主题搭、不抢戏、实现只改 `button.tsx`  
- **缺点**：icon 按钮反馈仍偏弱 → 可对 `size="icon"` 单独加 `active:bg-muted`

### 方案 B2 — 按压 + 微缩放

```
active: scale(0.98) + 背景变深
```

- **优点**：点击感更明显  
- **缺点**：MASTER 写 hover 不位移；scale 有极轻微布局风险（一般可接受）  
- **建议**：仅 `default` / `outline` 文字按钮，**不要**用于时间线密集图标

### 方案 B3 — 鼠标点击也显示聚焦环

```
focus:ring-2  （去掉 focus-visible 限制）
```

- **优点**：「点击聚焦」字面意思最直接  
- **缺点**：点完环常残留到 blur，桌面应用里显乱  
- **结论**：**不推荐**；用 active 态代替「点击反馈」

**推荐组合**：**B1**，顶栏唯一 CTA（若将来有）可叠加 B2 的 `scale(0.98)`。

## 4. 标签页：三种方案

### 方案 T1 — 滑动下划线（推荐）

```
TabsList border-b
TabsTrigger：无背景块，仅文字
指示条：单独 div，absolute bottom-0，width/left 随 active tab 过渡 200ms
```

- **参考**：Chrome 标签、Linear 设置页  
- **与 PanelTabs**：左/右面板 `variant="line"` 专用样式  
- **聚焦**：`TabsTrigger` 保留 `focus-visible:ring`；选中态 `text-foreground font-medium`

### 方案 T2 — 滑动药丸（Pill）

```
TabsList：bg-muted/50 rounded-lg p-1
指示条：bg-background shadow-sm rounded-md，滑动到当前项背后
```

- **参考**：iOS Segmented Control、shadcn `variant="default"`  
- **优点**：选中非常明显  
- **缺点**：占高度、偏「移动端」；和当前扁平面板头略违和

### 方案 T3 — 仅强化现有 line + 内容淡入

```
保持 after 下划线，加 transition-opacity
面板内容：切换时 opacity 0→1  150ms
```

- **优点**：改动最小  
- **缺点**：下划线仍不滑动，高级感有限

**推荐**：面板用 **T1**；若以后要顶部大 Tab 再考虑 T2。

### 标签「聚焦」

| 场景 | 行为 |
|------|------|
| 鼠标点击 | 文字变 `foreground` + 指示条到位（即选中反馈） |
| Tab 键 | 额外 `focus-visible:ring-2 ring-ring/40` 在 trigger 上 |
| 选中 ≠ 聚焦 | Radix 默认会 focus trigger；环与下划线可并存 |

### 内容区切换

可选一层（不冲突）：

```tsx
// 面板内容 wrapper
className="animate-in fade-in-50 duration-150"
key={activeTab}  // 触发重挂载淡入
```

时间线/属性内容较重时，**仅 fade 150ms**，不要 slide（易晕）。

## 5. 推荐落地组合（拍板用）

| 区域 | 方案 |
|------|------|
| 全局 Button | **B1** 改 `button.tsx` cva |
| Icon 按钮 | B1 + `active:bg-muted` |
| 左/右 PanelTabs | **T1** 滑动下划线 + 去掉 `data-active:bg-muted` 灰块 |
| 面板内容 | 可选 150ms fade（`key={tab}`） |
| 键盘 a11y | 保持 `focus-visible`，不加鼠标 focus 环 |
| reduced-motion | 指示条 `transition-none`，fade 关掉 |
