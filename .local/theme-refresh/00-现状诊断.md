# 00 — 现状诊断

## 1. 当前 token 来源

实现文件：`apps/electron/src/renderer/src/index.css`  
配置：`apps/electron/components.json` → `style: radix-nova`, `baseColor: neutral`

**矛盾点**：`components.json` 声明 neutral，但 `index.css` 手写了一套 **Cinema Dark + Action Red**（紫蓝 hue≈278° 背景 + 红 primary + 青 ring）。

```css
/* 现状摘要 */
--background: oklch(0.168 0.032 278.2);  /* 偏紫 #0F0F23 */
--primary:    oklch(0.577 0.245 27.3);   /* 饱和红 #E11D48 */
--ring:       oklch(0.574 0.104 181.2); /* 青色 #0D9488 */
--radius:     0.5rem;
```

## 2. 为什么观感「土」

### 2.1 配色年代感

| 元素 | 问题 |
|------|------|
| 背景 | 带紫调的深蓝灰（2018–2020 常见「dashboard 模板色」） |
| Primary | 高饱和玫瑰红用于**所有**主按钮，像促销页而非专业工具 |
| Ring | 青色聚焦环与红色 primary 无关联，像两套主题硬拼 |
| 边框 | `#2D2D4A` 与背景色相差小，显得「糊」而非「利落」 |

现代工具（Linear、Raycast、Figma dark、DaVinci 新 UI）倾向：**近中性灰黑底 + 单一低饱和 accent + 高对比边框（或 alpha 边框）**。

### 2.2 按钮与 shadcn nova 不匹配

`button.tsx`（nova）设计假设：

- 默认圆角偏大：`rounded-lg`、尺寸体系 `h-8`
- `default` variant = `bg-primary`（在 neutral 主题下应是**浅色实心**，不是红色）
- `ghost` / `outline` 承担工具栏 80% 操作
- `destructive` 是**浅红底 + 红字**，不是满屏大红块

业务层现状：

- `TopToolbar`、`ProjectPanel` 等大量 `variant="default"` → 全红
- 30+ 处手动 `className="rounded-sm"` 把 nova 压成「小方块」，与 Input/Popover 的 `rounded-lg` 不一致
- 旧文档要求 `rounded-sm`(4px) 按钮，与 nova 默认 **冲突**

### 2.3 文档与实现循环

`UI布局与交互设计-优化版.md` §0.1 仍规定：

- 主按钮 `bg-[#E11D48]`
- 输入聚焦 `ring-[#0D9488]`

ui-ux-pro-max 按产品名搜索时也会**复现同一套色**（#0F0F23 + #E11D48），说明「丑」来自规范本身，不是实现 bug。

## 3. 什么其实没问题

- **字体**：Plus Jakarta Sans + JetBrains Mono 仍然合适（可保留）
- **Lucide 图标**：尺寸与工具栏一致
- **暗色 only**：`html.dark` 策略正确
- **语义 token 架构**：`bg-background` / `em-*` 别名方向对，只换值即可

## 4. 诊断结论

| 层级 | 建议 |
|------|------|
| CSS 变量 | 整包替换为 neutral dark 或 editor-accent |
| 按钮用法 | 减 `default`，增 `ghost`/`outline`；删业务层 `rounded-sm` 覆写 |
| 文档 | 废弃 Cinema Dark 作为**壳层 UI** 主色；时间线 accent 可单独 token |
| 组件 | **不要**改 `components/ui/button.tsx` 核心，用 token + 用法修正 |
