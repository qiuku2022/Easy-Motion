# 01 — 方案对比

## 总览

| | 方案 A | 方案 B | 方案 C（不推荐） |
|---|--------|--------|------------------|
| 名称 | shadcn Neutral Dark | Editor Mono + Accent | 精修 Cinema Dark |
| 气质 | Linear / Vercel / GitHub dark | Premiere 现代灰 + 一点蓝 | 保留紫红青三色 |
| Primary 按钮 | 白/浅灰实心 | 白/浅灰实心 | 仍为红色 |
| 品牌色 | 几乎无（极简） | 仅时间线/播放头 | 满屏红 + 青环 |
| 改动量 | 小（ mainly CSS） | 中（CSS + 时间线 token） | 小但收益低 |
| 与你诉求 | ✅「黑白默认也不错」 | ✅ 专业工具有辨识度 | ❌ 仍可能显土 |

---

## 方案 A — shadcn Neutral Dark（推荐）

**参考**： [shadcn theming — neutral `.dark`](https://ui.shadcn.com/docs/theming)

特点：

- 背景 `oklch(0.145 0 0)` 真·中性黑灰，无紫调
- `primary` = 浅灰白按钮 + 深字（现代 SaaS 标准）
- `border` = `oklch(1 0 0 / 10%)` 细透明边，干净
- `destructive` 仅删除/错误场景
- `--radius: 0.625rem` 与 nova 一致

适合：你想快速变好看、少争论、跟 shadcn 生态一致。

---

## 方案 B — Editor Mono + Timeline Accent

在方案 A 基础上增加**仅用于时间线画布**的变量：

```css
--timeline-accent: oklch(0.62 0.19 264);   /* 偏靛蓝，类似专业 NLE */
--timeline-playhead: var(--timeline-accent);
--timeline-selection: oklch(0.62 0.19 264 / 0.35);
```

壳层 UI 仍用 neutral；播放头、片段选中、吸附线用 `--timeline-accent`，**不再**用 `primary`（红）或 `ring`（青）表达。

适合：希望「剪映/PR 专业感」但壳层要现代。

---

## 方案 C — 精修 Cinema Dark（不推荐为主方案）

可做但收益有限：

- 背景去紫：`oklch(0.14 0.01 260)` → `oklch(0.14 0.005 260)`
- Primary 降饱和：`oklch(0.55 0.18 25)` 
- Ring 与 primary 统一色相

问题：仍保留「红 CTA」心智，与 nova 白按钮体系不顺；容易回到满屏红。
