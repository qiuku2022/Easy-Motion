# 交互流程

> **实施状态（2026-06-26）**：M0–M8 已落地，M9（E2E/打包/性能）进行中。权威入口：仓库根 [README.md](../../README.md)、[开发里程碑与路线图.md](./开发里程碑与路线图.md)。

## 核心设计理念
- **简化流程**：解决剪辑师不会用 AE、不会写代码控制 Remotion 的痛点
- **多模态输入**：支持图片参考，一张排版参考图比大量文字描述更准确
- **对话驱动**：通过自然语言 + 参考素材快速生成动画

## 新建项目

### 正常流程
1. 用户点击"新建项目"
2. 弹出新建项目对话框
3. 用户选择：
   - **空白项目**：自定义分辨率、帧率、时长
   - **从模板开始**：选择内置模板（横屏 16:9、竖屏 9:16、方形 1:1 等）
4. 输入项目名称，选择保存路径
5. 系统自动创建项目文件夹结构和默认子项目

### 参数配置
| 参数 | 默认值 | 可选值 | 说明 |
|------|--------|--------|------|
| 分辨率 | 1920×1080 | 3840×2160、1080×1920 等 | 影响 Remotion Composition 的 width/height |
| 帧率 | 30fps | 24fps、25fps、60fps | 影响 Remotion Composition 的 fps |
| 时长 | 300 帧（10秒） | 用户自定义 | 影响 Remotion Composition 的 durationInFrames |
| 名称 | "未命名项目" | 用户输入 | 项目文件夹名，支持中英文、数字、下划线 |

### 异常与边界处理

| 场景 | 处理策略 | 用户反馈 |
|------|----------|----------|
| 目标路径已存在同名文件夹 | 弹窗提示，提供"打开现有项目"或"覆盖（删除后重建）"选项 | 确认弹窗 |
| 磁盘空间不足（< 500MB） | 阻止创建，提示用户清理磁盘 | Toast 错误提示 |
| 路径包含非法字符或过长（> 200 字符） | 输入框实时校验，阻止提交 | 输入框红色边框 + 错误文字 |
| 用户取消新建 | 关闭对话框，返回项目列表，不做任何操作 | 无 |
| 模板文件损坏或缺失 | 回退到空白项目模式，提示"模板加载失败，已使用默认配置" | Toast 警告 |

## 关闭项目

### 正常流程（2026-06-26 已落地）

1. 菜单 **≡ → 文件 → 关闭项目**（或等效入口）
2. 若有未保存时间线更改 → 确认丢弃
3. 主进程 `main:project:close` 清空当前项目句柄
4. 渲染层停止 Remotion 预览，重置时间线 / 素材 / AI 对话 store
5. UI 回到「未打开项目」：左侧项目列表、标题栏无项目名

### 与删除项目的区别

| 操作 | 磁盘数据 | 典型用途 |
|------|----------|----------|
| **关闭项目** | 保留 | 切换项目、回到启动态 |
| **删除项目** | 删除项目目录（需确认） | 彻底移除 |

## 核心工作流：参考文稿 + 参考排版 → 动画生成

这是产品的核心功能，以下是带异常处理的完整流程。

### 步骤 1：输入参考文稿

**正常流程：**
- 用户在对话面板输入文案、脚本、数据
- 支持粘贴纯文本、Markdown、CSV 表格
- 系统实时显示输入字数/字符数

**异常处理：**

| 场景 | 处理策略 |
|------|----------|
| 输入为空或仅空白字符 | 发送按钮禁用，提示"请输入文案内容" |
| 输入超过 5000 字符 | 截断提示，建议用户精简或分批输入 |
| 粘贴内容包含不可见控制字符 | 自动清理（保留换行和空格），提示"已自动清理格式" |
| 用户上传数据文件（CSV/JSON） | 解析为结构化数据，展示前 5 行预览供用户确认 |

### 步骤 2：上传参考排版图片

**正常流程：**
- 用户点击上传按钮或拖拽图片到对话面板
- 支持格式：PNG、JPG、WEBP、GIF
- 单张图片最大 10MB
- 最多同时上传 3 张参考图
- 上传后显示缩略图，支持删除和重新排序

**异常处理：**

| 场景 | 处理策略 |
|------|----------|
| 文件格式不支持 | 拒绝上传，提示"仅支持 PNG/JPG/WEBP/GIF 格式" |
| 单张图片超过 10MB | 拒绝上传，提示"单张图片最大 10MB，请压缩后重试" |
| 同时上传超过 3 张 | 拒绝第 4 张及以后，提示"最多支持 3 张参考图" |
| 图片分辨率过低（< 100×100） | 警告提示"图片分辨率过低，可能影响解析精度"，允许继续 |
| 上传失败（网络/IO 错误） | 自动重试 2 次；仍失败提示手动重试 |
| 用户未上传图片 | 允许继续（纯文字驱动模式），但提示"上传参考图片可获得更精准效果" |

### 步骤 3：Agent 生成动画

**正常流程：**
1. 用户点击"生成动画"按钮
2. 系统进入生成状态，显示进度指示器
3. Agent 执行：
   - **意图解析**：分析用户文字需求（动画类型、风格、时长要求）
   - **视觉解析**（如有图片）：提取参考图中的元素位置、颜色、字体、布局结构
   - **参数生成**：生成时间线配置 JSON（轨道、片段、关键帧）
   - **代码生成**：生成 Remotion 组件代码（`Root.tsx` + `MainSequence.tsx` + Layer 组件）
   - **代码校验**：通过 esbuild 语法检查 + 安全白名单扫描
4. 生成完成后，自动创建轨道和片段，写入 `subproject.json`
5. 预览窗口自动更新，展示生成结果

**异常处理：**

| 场景 | 处理策略 | 用户反馈 |
|------|----------|----------|
| LLM 调用超时（> 60 秒） | 首次超时自动重试 1 次；仍超时进入降级模式 | 提示"生成耗时较长，已切换至简化模式" |
| LLM API 返回错误（额度不足/Key 无效） | 停止生成，不写入任何文件 | 弹窗提示"LLM 服务异常：{具体错误}，请检查设置" |
| LLM 生成代码语法错误 | 捕获错误，尝试自动修复（如补全缺失 import）；修复失败则回退 | 提示"生成代码存在语法问题，已尝试自动修复"或"生成失败，请调整描述后重试" |
| 生成代码包含安全违规（禁止 API） | 拒绝写入，标记为"生成异常" | 提示"检测到不安全的生成内容，已阻止写入。请调整描述后重试。" |
| 生成结果为空或无法解析 | 回退到空白项目状态 | 提示"未能理解您的需求，请尝试更具体的描述" |
| 用户描述超出 Remotion 能力范围 | Agent 识别后给出替代方案 | 提示"您描述的效果（如 3D 粒子流体）超出当前支持范围，建议尝试：{替代方案}" |
| 生成过程中用户点击取消 | 立即中断 LLM 调用，不保存中间结果 | 提示"已取消生成"，回到生成前状态 |

**降级模式（LLM 超时/失败时）：**
- 简化解析：仅提取文字内容，生成基础文字动画（淡入淡出 + 简单位移）
- 跳过视觉解析：忽略参考图片，使用默认排版
- 用户可在生成后通过对话继续调整

### 步骤 4：视频预览

**正常流程：**
- Remotion Player 自动加载新生成的代码
- 播放/暂停控制实时响应
- 用户可在对话面板继续输入调整指令

**异常处理：**

| 场景 | 处理策略 |
|------|----------|
| Remotion Player 加载失败（iframe 崩溃） | 自动重载 iframe（最多 2 次），仍失败提示"预览加载失败，请检查生成内容" |
| 预览帧率过低（< 10fps 持续 3 秒） | 提示"当前预览帧率较低，建议降低分辨率或关闭部分特效" |
| 内存不足导致预览卡顿 | 提示"系统内存不足，建议关闭其他应用或重启软件" |

### 步骤 5：导出

见"预览与导出"章节。

## 对话调整

### 正常流程
1. 用户在对话面板输入调整指令（如"字体大一点"、"动画再快一些"）
2. AI 解析指令，定位目标元素
3. AI 计算参数变化（如 fontSize 64 → 96，durationInFrames 30 → 20）
4. 更新 `subproject.json` 的对应字段
5. 重新生成 Remotion 代码
6. 预览窗口自动刷新

### 调整指令类型与示例

| 类型 | 示例指令 | AI 行为 |
|------|----------|---------|
| 样式调整 | "字体大一点" | 识别选中的文字片段，按比例增大 fontSize |
| 位置调整 | "把标题移到左边" | 修改对应片段的 transform.position.x |
| 动画调整 | "入场动画慢一点" | 修改 inAnimation.durationInFrames |
| 颜色调整 | "改成红色" | 修改 style.color |
| 内容调整 | "标题改成'欢迎'" | 修改 source.content |
| 删除 | "删除第二个片段" | 从时间线移除对应 clip |
| 添加 | "加一个数据图表" | 创建新的 chart 轨道和片段 |

### 异常处理

| 场景 | 处理策略 |
|------|----------|
| AI 无法定位目标元素（如"把那个文字改一下"存在歧义） | 追问用户"您是指'标题'还是'副标题'？"，列出候选元素 |
| 调整后的参数超出有效范围（如 fontSize > 1000） | 自动截断到最大值，提示"已自动调整到最大支持值" |
| 调整导致片段重叠 | 自动后移冲突片段，或提示用户选择"覆盖"或"自动调整位置" |
| 用户指令模糊（如"好看一点"） | 提供 2-3 个预设选项（如"更动感""更柔和""更简约"）供用户选择 |
| 多轮对话上下文过长（> 10 轮） | 自动压缩早期对话历史，保留最近 5 轮 + 项目摘要 |

### 撤销/重做机制

- **粒度**：每次 AI 调整操作（一次对话回复可能包含多个操作，作为一个原子事务）
- **实现**：基于命令模式（Command Pattern）
  - 每个操作生成一个 `Command` 对象，包含 `execute()` 和 `undo()`
  - `execute()` 修改 timelineStore 并触发代码重新生成
  - `undo()` 恢复 timelineStore 到操作前状态，重新生成代码
- **存储**：
  - 撤销栈：内存数组，最大 50 步
  - 每 10 步自动快照到 `项目根目录/timeline-snapshots/{timestamp}.json`
  - 崩溃恢复：启动时检查快照，提示用户恢复到最近状态
- **快捷键**：Ctrl+Z / Ctrl+Y（macOS 用 Cmd）
- **边界**：
  - 切换子项目时撤销栈清空
  - 关闭项目时提示"是否保存当前状态"（撤销历史不保留到下次打开）

## 时间线编辑（可选）

### 正常流程
- 用户通过拖拽、属性面板手动调整时间线
- 支持操作：
  - 拖拽片段调整位置/时长
  - 拖拽轨道调整层级
  - 双击属性值直接编辑
  - 在时间线添加/删除关键帧
- 所有手动操作同样进入撤销栈
- 手动修改后，对话上下文同步更新（AI 知道当前最新状态）

### 对话与手动编辑的同步
- **单向优先**：手动编辑优先于 AI 建议
- **冲突检测**：如果用户手动修改了某属性，AI 后续调整该属性时，提示"您手动调整了此属性，是否覆盖？"
- **状态标记**：每个片段属性标记 `lastModifiedBy: 'user' | 'ai'`，用于冲突判断

## 素材导入

### 正常流程
1. 用户拖拽文件到项目面板或时间线
2. 系统识别文件类型（扩展名 + 文件头魔数）
3. 复制文件到项目 `assets/` 目录
4. 生成缩略图（图片缩略图/视频首帧）
5. 写入 SQLite 素材库
6. 若拖到时间线，自动创建对应类型的轨道和片段

### 文件类型识别与轨道分配

| 文件类型 | 识别方式 | 自动分配轨道 | 缩略图生成 |
|----------|----------|--------------|------------|
| PNG/JPG/WEBP | 扩展名 + 文件头 | `image` 轨道 | 等比缩放至 200px 宽 |
| SVG | 扩展名 + XML 头 | `image` 轨道 | 转 PNG 后生成缩略图 |
| GIF | 扩展名 + 文件头 | `image` 轨道（通过 `@remotion/gif` 组件直接渲染） | 首帧缩略图 |
| MP4/MOV/WEBM | 扩展名 + 文件头 | `video` 轨道 | FFmpeg 提取首帧 |
| MP3/WAV/AAC | 扩展名 + 文件头 | `audio` 轨道 | 音频波形缩略图 |
| CSV/JSON/XLSX | 扩展名 | 不自动创建轨道，存入 `data/` 目录 | 文本内容预览（前 5 行） |
| TTF/OTF | 扩展名 | 不自动创建轨道，复制到项目 `fonts/` 目录，通过 CSS `@font-face` 引用 | 字体名称 + 示例文字 |

### 异常处理

| 场景 | 处理策略 |
|------|----------|
| 文件格式不支持 | 拒绝导入，提示"不支持的文件格式：.xxx" |
| 文件损坏（无法读取/解析） | 拒绝导入，提示"文件损坏或格式异常，无法读取" |
| 磁盘空间不足（无法复制到 assets/） | 阻止导入，提示"磁盘空间不足，请清理后重试" |
| 同名文件已存在 | 弹窗询问"覆盖""重命名（添加序号）""跳过" |
| 文件过大（视频 > 2GB / 图片 > 50MB） | 警告提示"文件较大，可能影响预览性能"，允许继续 |
| 拖拽到不支持的区域（如属性面板） | 显示"禁止"光标，松开不执行任何操作 |
| 批量导入（> 20 个文件） | 显示批量进度条，支持取消；失败文件汇总提示 |

### 素材去重策略
- 计算文件内容的 SHA-256 哈希
- 若哈希已存在于素材库，提示"该素材已存在，是否引用现有文件？"
- 用户选择"引用"则创建新的素材记录但指向同一文件；"重新导入"则复制新副本

### 素材文件存储策略
- **命名规则**：`{uuid}_{原始文件名}`（如 `a1b2c3d4_logo.png`）
  - 保留原始文件名便于用户识别
  - 添加 UUID 前缀避免重名冲突
- **存放路径**：`项目根目录/assets/{素材类型}/`
  - 图片：`assets/images/`
  - 视频：`assets/videos/`
  - 音频：`assets/audios/`
  - 字体：`assets/fonts/`
- **缩略图**：`项目根目录/.thumbnails/`（按素材 UUID 命名）

## 预览与导出

### 实时预览
- Remotion Player 嵌入在 iframe 中，通过 `postMessage` 与主 UI 通信
- 预览分辨率：默认按项目设置，支持缩放（50%、75%、100%、125%、150%、200%）
- 预览质量：默认完整质量，性能不足时自动降级（降低分辨率或跳帧）

### 导出视频

**正常流程：**
1. 用户点击"导出"按钮
2. 弹出导出设置对话框
   - 格式：MP4 (H.264) / WEBM
   - 分辨率：原始 / 1080p / 720p（缩放导出）
   - 质量：高（CRF 18）/ 中（CRF 23）/ 低（CRF 28）
   - 输出路径：默认 `项目文件夹/output/`
3. 用户确认后开始渲染
4. 显示渲染进度（帧数 / 百分比 / 预估剩余时间）
5. 渲染完成，提示"导出成功"，提供"打开文件夹"和"播放"按钮

**异常处理：**

| 场景 | 处理策略 |
|------|----------|
| 输出路径无写入权限 | 提示"无法写入目标路径，请选择其他位置" |
| 磁盘空间不足（渲染前预估） | 根据分辨率×帧数×时长预估所需空间，不足时阻止渲染并提示 |
| FFmpeg 渲染失败 | 捕获 stderr 日志，定位错误帧/组件；提示"渲染在第 X 帧失败：{错误原因}"，提供"跳过该帧继续"或"修复后重试" |
| 渲染过程中用户点击取消 | 优雅中断，删除已生成的临时文件，提示"已取消渲染" |
| 渲染完成后文件损坏（无法播放） | 检测文件头有效性，损坏时提示"导出文件异常，请重新渲染" |
| 长时间渲染（> 30 分钟） | 显示"渲染时间较长，建议关闭其他应用以提升速度" |

### 导出 Remotion 工程
- 将当前子项目的 `remotion/` 文件夹打包为 ZIP
- 包含：源码、素材（复制）、package.json、README.md
  - **素材复制**：打包时将所有素材文件复制到 ZIP 内的 `assets/` 目录（不保留原目录结构，统一扁平存放），确保解压后可直接运行，不受原始路径影响
  - 软链接仅用于开发时的性能优化（macOS/Linux 下启用），打包导出时统一使用复制
- 用户解压后执行 `npm install && npm run start` 即可在浏览器中独立运行
