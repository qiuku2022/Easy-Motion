# 数据字典汇总

> **实施状态（2026-06-30）**：**M0–M9 ✅**（Windows MVP 预发行 v0.1.2）；后续以功能增强为主。权威入口：仓库根 [README.md](../../README.md)、[开发里程碑与路线图.md](./开发里程碑与路线图.md)。

## 概述

本文档汇总 EasyMotion 项目中所有数据结构的字段定义，作为快速参考手册。修改数据模型时，请先查阅本文档确保关联字段同步更新。

> **字段来源标注**：每个字段末尾标注 `[来源文档]`，指向详细定义所在的文档。

> **命名约定**：
> - **JSON / TypeScript 层**：统一使用 `camelCase`（如 `createdAt`, `isFavorite`, `contentHash`）
> - **数据库层**：统一使用 `snake_case`（如 `created_at`, `is_favorite`, `content_hash`）
> - 两者通过 ORM/数据访问层自动转换，开发者只需关注 JSON 层的 camelCase

## JSON ↔ 数据库字段映射表

以下字段在 JSON/TypeScript 层使用 `camelCase`，在 SQLite 数据库层使用 `snake_case`：

| JSON 字段（camelCase） | DB 字段（snake_case） | 类型（JSON） | 类型（DB） | 说明 |
|------------------------|----------------------|-------------|-----------|------|
| `createdAt` | `created_at` | number | INTEGER | Unix 毫秒时间戳 |
| `modifiedAt` | `modified_at` | number | INTEGER | Unix 毫秒时间戳 |
| `lastOpenedAt` | `last_opened_at` | number | INTEGER | Unix 毫秒时间戳 |
| `contentHash` | `content_hash` | string | TEXT | SHA-256 哈希值 |
| `isDeleted` | `is_deleted` | boolean | INTEGER (0/1) | 软删除标记 |
| `isFavorite` | `is_favorite` | boolean | INTEGER (0/1) | 收藏标记 |
| `isBuiltin` | `is_builtin` | boolean | INTEGER (0/1) | 内置预设标记 |
| `mimeType` | `mime_type` | string | TEXT | MIME 类型 |
| `originalName` | `original_name` | string | TEXT | 原始文件名 |
| `projectId` | `project_id` | string | TEXT | 所属项目 UUID |
| `sizeBytes` | `size_bytes` | number | INTEGER | 文件大小（字节） |
| `thumbnailPath` | `thumbnail_path` | string | TEXT | 缩略图路径 |
| `durationMs` | `duration_ms` | number | INTEGER | 时长（毫秒） |
| `durationInFrames` | `duration_in_frames` | number | INTEGER | 时长（帧） |
| `startInFrames` | `start_in_frames` | number | INTEGER | 开始帧 |
| `lastUsedAt` | `last_used_at` | number | INTEGER | 最后使用时间 |
| `usageCount` | `usage_count` | number | INTEGER | 使用次数 |
| `remotionVersion` | `remotion_version` | string | TEXT | Remotion 版本 |

**转换规则**：
- JSON 层 `true/false` ↔ DB 层 `1/0`
- JSON 层 `number`（Unix 毫秒）↔ DB 层 `INTEGER`
- 所有字符串字段保持原值，不转换大小写

## 完整数据结构

### 1. project.json（总项目配置）

```json
{
  "version": "1.0",
  "name": "我的项目",
  "createdAt": 1717056000000,
  "modifiedAt": 1717142400000,
  "settings": {
    "defaultResolution": { "width": 1920, "height": 1080 },
    "defaultFps": 30,
    "defaultDurationInFrames": 300
  },
  "subprojects": [
    {
      "id": "sub-001",
      "name": "开场动画",
      "path": "subprojects/开场动画",
      "createdAt": 1717056000000,
      "modifiedAt": 1717142400000
    }
  ],
  "remotionVersion": "4.0.269"
}
```

| 字段 | 类型 | 必填 | 说明 | 来源 |
|------|------|------|------|------|
| `version` | string | ✅ | 项目格式版本 | [项目管理.md] |
| `name` | string | ✅ | 项目名称 | [项目管理.md] |
| `createdAt` | number | ✅ | 创建时间（Unix 时间戳毫秒） | [项目管理.md] |
| `modifiedAt` | number | ✅ | 最后修改时间 | [项目管理.md] |
| `settings` | object | ✅ | 项目级默认设置 | [项目管理.md] |
| `settings.defaultResolution` | object | ✅ | 默认分辨率 | [项目管理.md] |
| `settings.defaultFps` | number | ✅ | 默认帧率 | [项目管理.md] |
| `settings.defaultDurationInFrames` | number | ✅ | 默认时长 | [项目管理.md] |
| `subprojects` | array | ✅ | 子项目列表 | [项目管理.md] |
| `subprojects[].id` | string | ✅ | 子项目 UUID | [项目管理.md] |
| `subprojects[].name` | string | ✅ | 子项目名称 | [项目管理.md] |
| `subprojects[].path` | string | ✅ | 子项目相对路径 | [项目管理.md] |
