## Remotion 许可证详细说明 ⭐ 关键

### 许可证类型

Remotion 使用**自定义商业许可证**（不是纯开源许可证），分为几个层级：

| 使用场景 | 费用 | 限制 |
|----------|------|------|
| **个人/学习/小团队** | 免费 | 年收入 < $200k，团队 < 3 人 |
| **商业团队** | 付费订阅 | 按座位收费，$15-$50/月/座 |
| **云服务/重分发** | 需特殊授权 | 不能作为 SaaS 提供 Remotion 渲染服务 |

### 对 EasyMotion 的影响

EasyMotion 的商业计划需要明确：

| 场景 | Remotion 许可要求 | 建议 |
|------|------------------|------|
| **免费桌面应用** | 如果团队 ≤ 3 人且年收入 < $200k，免费使用 | 确认符合免费条件 |
| **付费桌面应用** | 需要购买商业许可证 | 预算 $15-50/月/开发座位 |
| **作为服务提供** | 明确禁止 | 不能将 EasyMotion 作为在线服务提供 |

### 合规措施

1. **许可证声明**：在应用关于页面和 `LICENSE` 文件中声明 Remotion 的使用
2. **不修改源码**：不修改 Remotion 核心包，通过 Generator 生成标准 Remotion 代码
3. **独立安装**：子项目的 Remotion 依赖由用户 `npm install` 安装（不是打包进应用）
4. **用户责任**：导出 Remotion 工程后，用户自行负责其 Remotion 许可证合规性

## 依赖安全审计

### 审计频率

| 触发条件 | 审计内容 |
|----------|----------|
| 每周 | `pnpm audit` 检查已知漏洞 |
| 每月 | 检查依赖更新（`pnpm outdated`） |
| 每次发布前 | 完整依赖树审计，生成 SBOM（软件物料清单） |

### 审计命令

```bash
# 检查已知漏洞
pnpm audit

# 检查过时依赖
pnpm outdated

# 生成 SBOM
pnpm sbom --format json > sbom.json

# Python 依赖审计
pip-audit --requirement apps/python/requirements.txt
```
