# 开发环境搭建指南

> **实施状态（2026-06-30）**：**M0–M9 ✅**（Windows MVP 预发行 v0.1.2）；后续以功能增强为主。权威入口：仓库根 [README.md](../../README.md)、[构建与部署.md](./构建与部署.md)。

## 概述

本文档指导开发者从零开始搭建 EasyMotion 项目的本地开发环境，包括系统依赖安装、项目克隆、依赖安装、首次运行和常见问题排查。

## 依赖安装

### 1. Node.js

**要求：** Node.js v20 LTS（使用 nvm 或 fnm 管理版本）

```bash
# macOS / Linux（使用 nvm）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc  # 或 ~/.zshrc
nvm install 20
nvm use 20

# 验证
node -v  # v20.x.x
npm -v   # 10.x.x
```

```powershell
# Windows（使用 nvm-windows）
# 下载安装：https://github.com/coreybutler/nvm-windows/releases
nvm install 20.0.0
nvm use 20.0.0

# 验证
node -v
```

### 2. pnpm

**要求：** pnpm v10+（仓库锁定 `pnpm@10.13.1`）

```bash
# 全局安装 pnpm
npm install -g pnpm@latest

# 验证
pnpm -v  # 10.x.x
```

**启用 pnpm 全局存储（推荐）：**
```bash
# 配置全局存储路径（可选）
pnpm config set store-dir ~/.pnpm-store
```

### 3. Python

**要求：** Python 3.10+（推荐 3.11）

```bash
# macOS（使用 Homebrew）
brew install python@3.11

# Ubuntu/Debian
sudo apt update
sudo apt install python3.11 python3.11-venv python3.11-pip

# Windows
# 从 https://www.python.org/downloads/ 下载安装包
# 安装时勾选 "Add Python to PATH"

# 验证
python3 --version  # Python 3.11.x
# Windows: python --version
```

### 4. Playwright 浏览器二进制

```bash
# 安装 Playwright 依赖
pip install playwright

# 下载浏览器二进制
playwright install

# 验证
playwright install --help  # 查看已安装的浏览器
```

**注意：**
- Playwright 浏览器二进制约 100MB，首次下载需要一定时间
- 在 CI 环境中，可使用 `playwright install --with-deps` 自动安装系统依赖

### 5. FFmpeg

Remotion 依赖 FFmpeg 进行视频渲染，通常随 Remotion 自动安装。如需手动安装：

```bash
# macOS
brew install ffmpeg

# Ubuntu/Debian
sudo apt install ffmpeg

# Windows
# 从 https://www.gyan.dev/ffmpeg/builds/ 下载
# 添加到 PATH

# 验证
ffmpeg -version
```

### 6. Git

```bash
# 验证
git --version  # 2.40+

# 配置（首次使用）
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

## 项目克隆与安装

### 1. 克隆仓库

```bash
git clone https://github.com/EasyMotion/EasyMotion.git
cd EasyMotion
```

### 2. 安装依赖

```bash
# 安装所有 workspace 依赖（根 package.json 已预批准 electron/esbuild 的 postinstall）
pnpm install

# 安装 Python 依赖
cd apps/python
pip install -r requirements.txt
```

**依赖安装耗时：**
- pnpm install：约 3-5 分钟（取决于网络）
- Python 依赖：约 1-2 分钟
- 总计：约 5-8 分钟

### 3. 验证安装

```bash
pnpm ls
pnpm lint
pnpm test          # m1 + m2 + m3 + m5 + m5.2
```

## 首次运行

### 开发模式启动

```bash
# 日常 UI 开发（推荐）：Vite + Electron，不启动 Python
pnpm dev

# 含 Python FastAPI（爬虫/数据处理调试）
pnpm dev:all
```

**启动成功标志：**
- Electron 主窗口打开（预设库 + 时间线 + 预览）
- Vite 运行于 **`http://127.0.0.1:5173`**（勿用 `localhost`，Windows 可能仅 IPv6）
- `pnpm dev:all` 时 Python 监听 `http://127.0.0.1:8000`（占用时自动试 8001–8019）

### 首次启动初始化

应用首次运行时，会自动创建 `~/.easymotion/`（设置、`secrets.json`、日志等）。AI Key 可在应用内配置或复制 `apps/electron/.env.example` → `.env`。

**预设动图缩略图（首次克隆必做）**：81 个 RVE 预设的动画 WebP **不纳入 Git**（体积大）。需本地生成一次：

```bash
cd apps/electron
pnpm generate:preset-thumbnails    # 需 Chrome + ffmpeg（PATH）
```

- 输出：`resources/presets/thumbnails/` 与 `src/renderer/public/presets/thumbnails/`
- 可选：`--only rve-pie-chart`、`--skip-existing`
- 未生成时预设库仍可用，卡片显示渐变占位
- 缩略图渲染：`ThumbnailComposition` 将预设按 1920×1080 画布整体缩放至 320×180，避免裁切

**打安装包前**：`build:renderer` 会从 `resources/presets/thumbnails/` 同步 WebP 到 `public/`（`sync-preset-thumbnails.cjs`），本地须已生成过缩略图。

### VS Code / Cursor 调试（F5）

| 配置 | 用途 |
|------|------|
| **EasyMotion: Dev** | 主进程断点 |
| **EasyMotion: Dev + React** | 主进程 + 渲染进程（Chrome attach 9333） |

F5 前 `debug: prepare` 会：检查/下载 Electron 二进制 → 释放 CDP 9333 → 确保 Vite 5173 就绪。

### 验证核心功能

1. **项目**：新建 1080p 项目 → 自动启动 Remotion 预览
2. **预设**：双击 RVE 预设 → 时间线与预览更新
3. **Agent**（需 API Key）：AI 助手输入「添加标题 Hello」
4. **导出**：工具栏导出 MP4 → `pnpm --filter @easymotion/electron test:m8` 可脚本验证

## 常用开发命令

```bash
# 开发
pnpm dev                    # Vite + Electron
pnpm dev:all                # + Python
pnpm dev:renderer           # 仅 Vite
pnpm dev:electron           # 等 5173 后启动 Electron

# 测试（里程碑脚本，非 Vitest）
pnpm test                   # m1+m2+m3+m5+m5.2
pnpm test:m1                # 项目/文件服务
pnpm --filter @easymotion/electron test:m8   # 导出
pnpm --filter @easymotion/electron test:m5.2 # Remotion Code Agent

# 代码质量
pnpm lint / pnpm lint:fix
pnpm format / pnpm format:check

# 预设
cd apps/electron && pnpm vendor:rve-presets
cd apps/electron && pnpm generate:preset-thumbnails   # 首次克隆；需 Chrome + ffmpeg
cd apps/electron && node tests/test-preset-parameters.js

# 打包（Windows x64，本机构建）
pnpm build:win                 # NSIS 安装包
pnpm build:python              # 仅 Python venv → resources/python/
pnpm --filter @easymotion/electron build:dir   # win-unpacked，不生成 .exe 安装器
```

> `pnpm test:e2e`、`pnpm typecheck` 等 **尚未配置**（M9 规划）。详见 [测试策略.md](测试策略.md)、[构建与部署.md](构建与部署.md)。

## 环境变量

开发时可通过 `.env` 文件或环境变量配置以下选项：

```bash
# apps/electron/.env（开发环境）

# Electron 开发模式
NODE_ENV=development
ELECTRON_ENABLE_LOGGING=1

# Python 服务
PYTHON_PORT=8000
PYTHON_DEBUG=1

# LLM（可选，用于快速测试，不提交到版本控制）
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-...

# 代理（如需要）
HTTP_PROXY=http://proxy:8080
HTTPS_PROXY=http://proxy:8080
NO_PROXY=localhost,127.0.0.1
```

**注意：**
- `.env` 文件已加入 `.gitignore`，不会提交到版本控制
- 生产环境配置通过 CI/CD 环境变量注入

## 常见问题排查

### Q1: `pnpm install` 失败，提示网络错误

**原因：** pnpm 或 npm 镜像源不稳定。

**解决：**
```bash
# 使用国内镜像
pnpm config set registry https://registry.npmmirror.com

# 或使用官方源
pnpm config set registry https://registry.npmjs.org

# 重新安装
pnpm install
```

### Q2: Playwright 安装失败，提示浏览器二进制下载失败

**原因：** 网络问题或权限不足。

**解决：**
```bash
# 设置 Playwright 镜像（国内）
export PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright

# 重新安装
playwright install

# 如果仍失败，手动下载对应平台二进制并放到 ~/.cache/ms-playwright/
```

### Q3: Python FastAPI 启动失败，提示端口被占用

**原因：** 8000 端口被其他服务占用。

**解决：**
```bash
# 查找占用进程
lsof -i :8000  # macOS/Linux
netstat -ano | findstr :8000  # Windows

# 修改端口
# 在 apps/python/.env 中添加：
PORT=8001
```

### Q4: Electron 窗口白屏

**原因：** 渲染进程开发服务器未启动或 vite 配置错误。

**排查：**
```bash
# 检查渲染进程是否运行在 http://127.0.0.1:5173
curl http://127.0.0.1:5173

# 检查 Electron 日志
cat ~/.easymotion/logs/app.log

# 手动启动渲染进程
cd apps/electron
pnpm dev:renderer
```

### Q5: TypeScript 类型检查失败

**原因：** 类型定义不完整或依赖版本不匹配。

**解决：**
```bash
# 清理并重新安装依赖
pnpm clean:all
pnpm install

# 检查 TypeScript 版本
pnpm tsc --version

# 单独运行类型检查查看详细错误
pnpm typecheck
```

### Q6: Windows 本地打安装包

**命令：**

```powershell
pnpm build:win
```

**产物：** `apps/electron/release/EasyMotion-Setup-0.1.2.exe`（未签名测试包，SmartScreen 可能拦截）。

**前提：** 本机 Python 3.10+（打 venv bundle）、预设缩略图 WebP 已生成。Python bundle 与 Electron 均需在 **Windows 本机** 构建，不可交叉编译。

### Q7: Windows 上 Python 路径问题

**原因：** Windows 上 `python` 和 `python3` 命令可能指向不同路径。

**解决：**
```powershell
# 检查 Python 路径
where python
where python3

# 在 package.json 中使用 cross-env 确保一致性
# "build:python": 由 apps/python/scripts/build-python-bundle.cjs 处理
```

### Q8: F5 调试报错 `ENOENT path.txt`

**原因：** pnpm 10 默认拦截部分包的 postinstall，Electron 二进制未下载完整（缺少 `path.txt` / `electron.exe`）。

**解决：**

```powershell
pnpm install
# 仍失败：
pnpm approve-builds   # 勾选 electron
pnpm install
```

F5 的 `ensure-electron-binary.cjs` 也会尝试自动下载修复。根 `package.json` 已配置 `pnpm.onlyBuiltDependencies: ["electron", "esbuild"]`。

## 开发工作流

```
1. 从 develop 分支切出功能分支
   git checkout develop
   git pull origin develop
   git checkout -b feature/xxx

2. 开发
   - 编写代码
   - 运行测试：pnpm test
   - 类型检查：pnpm typecheck
   - 代码格式化：pnpm format

3. 提交
   git add .
   git commit -m "feat(timeline): 实现片段拖拽"
   
4. 推送并创建 PR
   git push origin feature/xxx
   # 在 GitHub 创建 PR，指向 develop 分支

5. Code Review + CI 通过
   - PR 需要至少 1 个 Reviewer 批准
   - CI 检查通过（build + lint + test）

6. 合并到 develop
   # Squash and Merge
```
