# 开发环境搭建指南

## 概述

本文档指导开发者从零开始搭建 EasyMotion 项目的本地开发环境，包括系统依赖安装、项目克隆、依赖安装、首次运行和常见问题排查。

---

## 系统要求

### 硬件要求

| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 4 核 | 8 核以上 |
| 内存 | 8GB | 16GB |
| 磁盘空间 | 20GB 可用空间 | 50GB（含项目缓存） |
| 网络 | 稳定网络（用于下载依赖） | 高速网络 |

### 操作系统要求

| 系统 | 版本 | 说明 |
|------|------|------|
| macOS | 12+ (Monterey) | 支持 Intel 和 Apple Silicon |
| Windows | Windows 10 1909+ | 64 位 |
| Linux | Ubuntu 20.04 LTS+ | 其他发行版需自行验证 |

## 依赖安装

### 1. Node.js

**要求：** Node.js v20 LTS（使用 nvm 或 fnm 管理版本）

```bash
# macOS / Linux（使用 nvm）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc  # 或 ~/.zshrc
nvm install 20
nvm use 20

# 验证
node -v  # v20.x.x
npm -v   # 10.x.x
```

```powershell
# Windows（使用 nvm-windows）
# 下载安装：https://github.com/coreybutler/nvm-windows/releases
nvm install 20.0.0
nvm use 20.0.0

# 验证
node -v
```

### 2. pnpm

**要求：** pnpm v9+

```bash
# 全局安装 pnpm
npm install -g pnpm@latest

# 验证
pnpm -v  # 9.x.x
```

**启用 pnpm 全局存储（推荐）：**
```bash
# 配置全局存储路径（可选）
pnpm config set store-dir ~/.pnpm-store
```

### 3. Python

**要求：** Python 3.10+（推荐 3.11）

```bash
# macOS（使用 Homebrew）
brew install python@3.11

# Ubuntu/Debian
sudo apt update
sudo apt install python3.11 python3.11-venv python3.11-pip

# Windows
# 从 https://www.python.org/downloads/ 下载安装包
# 安装时勾选 "Add Python to PATH"

# 验证
python3 --version  # Python 3.11.x
# Windows: python --version
```

### 4. Playwright 浏览器二进制

```bash
# 安装 Playwright 依赖
pip install playwright

# 下载浏览器二进制
playwright install

# 验证
playwright install --help  # 查看已安装的浏览器
```

**注意：**
- Playwright 浏览器二进制约 100MB，首次下载需要一定时间
- 在 CI 环境中，可使用 `playwright install --with-deps` 自动安装系统依赖

### 5. FFmpeg

Remotion 依赖 FFmpeg 进行视频渲染，通常随 Remotion 自动安装。如需手动安装：

```bash
# macOS
brew install ffmpeg

# Ubuntu/Debian
sudo apt install ffmpeg

# Windows
# 从 https://www.gyan.dev/ffmpeg/builds/ 下载
# 添加到 PATH

# 验证
ffmpeg -version
```

### 6. Git

```bash
# 验证
git --version  # 2.40+

# 配置（首次使用）
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

## 项目克隆与安装

### 1. 克隆仓库

```bash
git clone https://github.com/EasyMotion/EasyMotion.git
cd EasyMotion
```

### 2. 安装依赖

```bash
# 安装所有 workspace 依赖
pnpm install

# 安装 Python 依赖
cd apps/python
pip install -r requirements.txt
```

**依赖安装耗时：**
- pnpm install：约 3-5 分钟（取决于网络）
- Python 依赖：约 1-2 分钟
- 总计：约 5-8 分钟

### 3. 验证安装

```bash
# 检查 workspace 结构
pnpm ls

# 检查 TypeScript 编译
pnpm typecheck

# 检查 ESLint
pnpm lint
```

## 首次运行

### 开发模式启动

```bash
# 方式 1：同时启动 Electron + Python（推荐）
pnpm dev

# 方式 2：分别启动（调试时使用）
# 终端 1：启动 Electron
pnpm dev:electron

# 终端 2：启动 Python FastAPI
cd apps/python
python main.py

# 终端 3：启动渲染进程开发服务器（如需要单独调试）
cd apps/electron
pnpm dev:renderer
```

**启动成功标志：**
- Electron 窗口打开，显示 "Hello EasyMotion"
- Python 服务监听 `http://localhost:8000`
- 终端无错误输出

### 首次启动初始化

应用首次运行时，会自动创建以下目录和文件：

```
~/.easymotion/
├── settings.json          # 默认应用设置
├── secrets.json           # 敏感信息（空，待用户配置）
├── ui-state.json          # UI 状态
├── library.db             # SQLite 数据库
├── library.db.bak         # 数据库备份（首次为空）
├── logs/
│   ├── app.log            # 应用日志
│   └── error.log          # 错误日志
├── temp/                  # 临时文件（自动清理）
└── thumbnails/            # 全局缩略图缓存
```

### 验证核心功能

1. **项目系统**
   - 点击"新建项目" → 填写名称 → 创建成功
   - 检查文件系统：`~/EasyMotion/` 下出现新项目文件夹

2. **Python 服务**
   ```bash
   curl http://localhost:8000/health
   # 预期响应：{"success": true, "data": {"status": "healthy"}}
   ```

3. **Generator**
   - 打开项目 → 手动创建简单 timeline → 检查 `remotion/src/` 下是否生成 TSX 文件

## 常用开发命令

```bash
# 开发
pnpm dev                    # 启动完整开发环境
pnpm dev:electron           # 仅启动 Electron
pnpm dev:python             # 仅启动 Python FastAPI

# 构建
pnpm build                  # 构建所有应用
pnpm build:renderer         # 构建渲染进程
pnpm build:main             # 构建主进程
pnpm build:python           # 构建 Python 模块

# 测试
pnpm test                   # 运行所有测试
pnpm test:unit              # 运行单元测试
pnpm test:e2e               # 运行 E2E 测试（需要先构建）
pnpm test:python            # 运行 Python 测试

# 代码质量
pnpm lint                   # ESLint 检查
pnpm lint:fix               # 自动修复 ESLint 问题
pnpm typecheck              # TypeScript 类型检查
pnpm format                 # Prettier 格式化

# 打包
pnpm build:local            # 本地打包（当前平台，不签名）
pnpm build:mac              # macOS 打包
pnpm build:win              # Windows 打包
pnpm build:linux            # Linux 打包

# 清理
pnpm clean                  # 清理构建产物
pnpm clean:all              # 清理所有（包括 node_modules）
```

## 环境变量

开发时可通过 `.env` 文件或环境变量配置以下选项：

```bash
# apps/electron/.env（开发环境）

# Electron 开发模式
NODE_ENV=development
ELECTRON_ENABLE_LOGGING=1

# Python 服务
PYTHON_PORT=8000
PYTHON_DEBUG=1

# LLM（可选，用于快速测试，不提交到版本控制）
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-...

# 代理（如需要）
HTTP_PROXY=http://proxy:8080
HTTPS_PROXY=http://proxy:8080
NO_PROXY=localhost,127.0.0.1
```

**注意：**
- `.env` 文件已加入 `.gitignore`，不会提交到版本控制
- 生产环境配置通过 CI/CD 环境变量注入

## 常见问题排查

### Q1: `pnpm install` 失败，提示网络错误

**原因：** pnpm 或 npm 镜像源不稳定。

**解决：**
```bash
# 使用国内镜像
pnpm config set registry https://registry.npmmirror.com

# 或使用官方源
pnpm config set registry https://registry.npmjs.org

# 重新安装
pnpm install
```

### Q2: Playwright 安装失败，提示浏览器二进制下载失败

**原因：** 网络问题或权限不足。

**解决：**
```bash
# 设置 Playwright 镜像（国内）
export PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright

# 重新安装
playwright install

# 如果仍失败，手动下载对应平台二进制并放到 ~/.cache/ms-playwright/
```

### Q3: Python FastAPI 启动失败，提示端口被占用

**原因：** 8000 端口被其他服务占用。

**解决：**
```bash
# 查找占用进程
lsof -i :8000  # macOS/Linux
netstat -ano | findstr :8000  # Windows

# 修改端口
# 在 apps/python/.env 中添加：
PORT=8001
```

### Q4: Electron 窗口白屏

**原因：** 渲染进程开发服务器未启动或 vite 配置错误。

**排查：**
```bash
# 检查渲染进程是否运行在 http://localhost:5173
curl http://localhost:5173

# 检查 Electron 日志
cat ~/.easymotion/logs/app.log

# 手动启动渲染进程
cd apps/electron
pnpm dev:renderer
```

### Q5: TypeScript 类型检查失败

**原因：** 类型定义不完整或依赖版本不匹配。

**解决：**
```bash
# 清理并重新安装依赖
pnpm clean:all
pnpm install

# 检查 TypeScript 版本
pnpm tsc --version

# 单独运行类型检查查看详细错误
pnpm typecheck
```

### Q6: macOS 上运行 `pnpm build:local` 失败，提示签名问题

**原因：** macOS 需要开发者签名才能运行打包后的应用。

**解决：**
```bash
# 本地测试时跳过签名
pnpm build:local --publish=never

# 或在 electron-builder.yml 中设置：
# mac:
#   identity: null  # 本地测试时不签名
```

### Q7: Windows 上 Python 路径问题

**原因：** Windows 上 `python` 和 `python3` 命令可能指向不同路径。

**解决：**
```powershell
# 检查 Python 路径
where python
where python3

# 在 package.json 中使用 cross-env 确保一致性
# "build:python": "cross-env PYTHON=python pyinstaller ..."
```

## 开发工作流

```
1. 从 develop 分支切出功能分支
   git checkout develop
   git pull origin develop
   git checkout -b feature/xxx

2. 开发
   - 编写代码
   - 运行测试：pnpm test
   - 类型检查：pnpm typecheck
   - 代码格式化：pnpm format

3. 提交
   git add .
   git commit -m "feat(timeline): 实现片段拖拽"
   
4. 推送并创建 PR
   git push origin feature/xxx
   # 在 GitHub 创建 PR，指向 develop 分支

5. Code Review + CI 通过
   - PR 需要至少 1 个 Reviewer 批准
   - CI 检查通过（build + lint + test）

6. 合并到 develop
   # Squash and Merge
```
