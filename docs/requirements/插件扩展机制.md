# 插件与扩展机制

> **实施状态（2026-06-30）**：**M0–M9 ✅**（Windows MVP 预发行 v0.1.2）；后续以功能增强为主。权威入口：仓库根 [README.md](../../README.md)、[开发里程碑与路线图.md](./开发里程碑与路线图.md)。

## 概述

本文档定义 EasyMotion 的插件与扩展架构预留方案。当前版本（v1.0）以内置预设和官方组件为主，但架构上预留插件接口，为未来第三方扩展和生态建设做准备。

> **版本**：v0.1  
> **最后更新**：2026-06-26  
> **关联文档**：代码生成规范.md、LLM-Agent设计.md、应用架构详细设计.md、组件库清单.md

## 1. 设计原则

### 1.1 当前策略

- **v1.0**：不支持第三方插件，所有功能通过内置预设和官方组件实现。
- **架构预留**：在代码设计上预留扩展点，避免后期大规模重构。
- **未来方向**：v2.0 根据用户需求评估是否开放插件 API。

### 1.2 为什么预留

| 风险 | 说明 |
|------|------|
| 硬编码预设 | 预设系统直接耦合在核心代码中，后期难以外部化 |
| 组件无注册机制 | 新组件需要修改 Generator 源码 |
| Agent 工具固定 | LLM 可调用工具写死在 Agent 中 |
| 导出格式封闭 | 无法支持社区贡献的新导出格式 |

## 2. 扩展点设计

### 2.1 预设扩展（Preset Extension）

#### 数据结构

```typescript
// shared/types/preset.ts
interface Preset {
  id: string;
  name: string;
  description: string;
  category: 'data-chart' | 'title' | 'transition' | 'intro' | 'custom';
  author: string;              // 'builtin' 或插件作者
  version: string;
  icon: string;                // SVG 路径或 Lucide 图标名
  
  // 预设定义
  definition: {
    // 生成的时间线片段模板
    clipTemplate: ClipTemplate;
    // 可配置参数（用户可在属性面板调整）
    parameters: PresetParameter[];
    // 默认样式
    defaultStyle: Partial<StyleConfig>;
  };
  
  // 兼容性
  remotionVersion: string;     // 最低兼容版本
  minAppVersion: string;       // 最低应用版本
}

interface PresetParameter {
  key: string;
  label: string;
  type: 'text' | 'number' | 'color' | 'select' | 'range';
  defaultValue: unknown;
  options?: { label: string; value: unknown }[];  // select 类型
  min?: number;                // range/number 类型
  max?: number;
  step?: number;
}
```

#### 内置预设 vs 用户预设 vs 插件预设

| 类型 | 存储位置 | 来源 | 可编辑 |
|------|----------|------|--------|
| 内置预设 | `resources/presets/builtin/` | 随应用发布 | ❌ 不可编辑 |
| 用户预设 | `项目/presets/user/` | 用户基于现有动画保存 | ✅ 可编辑、删除 |
| 插件预设 | `~/.easymotion/plugins/{plugin}/presets/` | 第三方插件 | ❌ 不可编辑（可卸载插件） |

#### 预设注册机制（预留）

```typescript
// 未来插件系统激活时的注册接口
interface PresetRegistry {
  // 注册预设
  register(preset: Preset): void;
  
  // 注销预设（插件卸载时）
  unregister(presetId: string): void;
  
  // 获取所有预设（内置 + 用户 + 插件）
  getAll(): Preset[];
  
  // 按分类获取
  getByCategory(category: string): Preset[];
  
  // 搜索
  search(query: string): Preset[];
}
```

interface ComponentRegistry {

### 2.3 Agent 工具扩展（Tool Extension）

#### 工具接口（预留）

```typescript
// shared/types/agent-tool.ts
// 工具接口在《LLM-Agent设计.md》中统一定义，此处引用而非重复定义，
// 避免 Zod/JSONSchema 两种不兼容的参数描述格式。
// 
// 推荐的插件工具定义方式：用 zod-to-json-schema 包把 zod Schema 转 JSON Schema，
// 在运行时供 LLM function-calling 解析。

interface AgentTool {
  name: string;                 // camelCase，与 LLM Agent 工具名规范保持一致
  description: string;
  // 内部运行时校验用 zod（强类型，IDE 友好）
  parameters: import('zod').z.ZodTypeAny;
  // 暴露给 LLM 的 Schema 在初始化时由 zod-to-json-schema 转换
  execute: (params: unknown, context: ToolContext) => Promise<ToolResult>;
}

interface ToolContext {
  projectId: string;
  subprojectId: string;
  timeline: Timeline;
  assets: Asset[];
}

interface ToolResult {
  success: boolean;
  data?: unknown;
  error?: string;
}

// 转换示例
import { zodToJsonSchema } from 'zod-to-json-schema';
const llmVisibleSchema = zodToJsonSchema(myTool.parameters);
```

#### 内置工具（v1.0）

| 工具名 | 功能 | 示例调用 |
|--------|------|----------|
| `add_track` | 添加轨道 | `add_track({ type: 'text', name: '标题' })` |
| `add_clip` | 添加片段 | `add_clip({ trackId: '...', type: 'text', content: 'Hello' })` |
| `modify_clip` | 修改片段 | `modify_clip({ clipId: '...', style: { color: '#fff' } })` |
| `add_keyframe` | 添加关键帧 | `add_keyframe({ clipId: '...', property: 'opacity', value: 0 })` |
| `delete_element` | 删除元素 | `delete_element({ id: '...' })` |
| `apply_preset` | 应用预设 | `apply_preset({ presetId: '...' })` |
| `import_asset` | 导入素材 | `import_asset({ url: '...' })` |
| `query_assets` | 查询素材 | `query_assets({ type: 'image', search: 'logo' })` |

#### 未来扩展示例

```typescript
// 插件注册自定义爬虫工具（预留接口）
const crawlNewsTool: AgentTool = {
  name: 'crawl_news',
  description: '抓取新闻网站内容并生成文字动画',
  parameters: z.object({
    url: z.string().url(),
    summaryLength: z.number().min(50).max(500).default(200),
  }),
  execute: async (params, context) => {
    // 插件自定义逻辑
    const news = await crawlNews(params.url);
    const summary = summarize(news, params.summaryLength);
    
    // 调用内置工具创建文字轨道
    return context.tools.add_clip({
      trackId: await context.tools.add_track({ type: 'text' }),
      content: summary,
      style: { fontSize: 48 },
    });
  },
};

// 注册到 Agent
toolRegistry.register(crawlNewsTool);
```

## 4. 安全沙箱

### 4.1 插件权限模型

| 权限 | 说明 | 风险 |
|------|------|------|
| `preset:register` | 注册预设 | 低 |
| `component:register` | 注册组件 | 中（可能注入危险代码） |
| `tool:register` | 注册 Agent 工具 | 中（可能滥用 API） |
| `asset:read` | 读取素材 | 低 |
| `asset:write` | 写入素材 | 中（可能污染项目） |
| `export:register` | 注册导出格式 | 低 |
| `network:request` | 网络请求 | 高（数据外泄） |
| `fs:read` | 读取文件系统 | 高 |
| `fs:write` | 写入文件系统 | 高 |

### 4.2 代码审查要求

- 插件组件代码需通过 Generator 的 AST 安全扫描（同内置组件）。
- 插件工具调用受限：只能操作当前项目，不能访问其他项目。
- 网络请求需用户授权（弹窗确认）。

## 7. 开发规范

### 7.1 预留扩展点的注释规范

```typescript
// ✅ 正确：明确标注预留扩展点
// [PLUGIN-EXT] 未来插件系统激活时，通过 registerLayer() 注册新组件
const BUILTIN_LAYERS = { ... };

// ✅ 正确：接口设计考虑扩展
interface TimelineActions {
  addTrack(type: TrackType, name?: string): void;
  // [PLUGIN-EXT] v2.0 支持插件添加新轨道类型
}

// ❌ 避免：不留痕迹的硬编码
const LAYER_TYPES = ['text', 'image', 'video'];  // 没有扩展提示
```

### 7.2 命名约定

- 预留的接口/函数使用 `registerXxx`、`unregisterXxx` 命名。
- 预留的注释标记：`[PLUGIN-EXT]` 或 `[RESERVED]`。
