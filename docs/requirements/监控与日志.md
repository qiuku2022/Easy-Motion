## 1. 日志系统

### 1.1 日志目录结构

```
~/.easymotion/logs/
├── app.log                         # 应用主日志（滚动）
├── app.error.log                   # 错误日志（单独分离）
├── ipc.log                         # IPC 通信日志（开发模式）
├── security.log                    # 安全事件日志
├── csp-violations.log              # CSP 违规日志（生产模式）
├── performance.log                 # 性能指标日志
├── crashes/
│   ├── crash-2026-05-30-120305.dmp   # 崩溃转储文件
│   └── crash-2026-05-30-120305.log   # 崩溃上下文日志
└── archives/                       # 归档日志（按月）
    ├── app-2026-04.log.gz
    └── app-2026-05.log.gz
```

### 1.2 日志级别

| 级别 | 用途 | 输出位置 | 保留期限 |
|------|------|----------|----------|
| `TRACE` | 最详细的调试信息（函数入口/出口） | 仅开发模式控制台 | - |
| `DEBUG` | 开发调试信息（状态变更、IPC 调用） | 开发模式控制台 + 文件 | 7 天 |
| `INFO` | 正常运行信息（启动、保存、导出完成） | 文件 + 生产控制台 | 30 天 |
| `WARN` | 警告（降级操作、性能不足、文件损坏修复） | 文件 + 生产控制台 | 30 天 |
| `ERROR` | 错误（操作失败、渲染失败、API 错误） | 文件 + error.log | 90 天 |
| `FATAL` | 致命错误（未捕获异常、主进程崩溃） | 文件 + error.log + 崩溃转储 | 永久 |

### 1.3 日志格式

```
[2026-05-30 12:03:05.234] [INFO] [main] [project-service] 项目创建成功: Demo (proj-001)
[2026-05-30 12:03:05.567] [DEBUG] [renderer] [timelineStore] addTrack: text (track-001)
[2026-05-30 12:03:06.001] [WARN] [main] [preview] 预览帧率低于 15fps，已自动降级分辨率
[2026-05-30 12:03:10.123] [ERROR] [main] [generator] 代码生成失败: SyntaxError: Unexpected token (line 45)
```

```typescript
interface LogEntry {
  timestamp: string;         // ISO 8601 格式
  level: 'TRACE' | 'DEBUG' | 'INFO' | 'WARN' | 'ERROR' | 'FATAL';
  process: 'main' | 'renderer' | 'python';
  module: string;            // 模块名，如 'project-service'
  message: string;
  metadata?: Record<string, unknown>;  // 结构化数据（JSON）
  error?: {
    name: string;
    message: string;
    stack?: string;
    code?: string;
  };
}
```

### 1.4 日志实现

**主进程**：使用 `electron-log` + 自定义格式化。

```typescript
// main/utils/logger.ts
import log from 'electron-log';

log.transports.file.resolvePath = () => 
  path.join(app.getPath('userData'), 'logs', 'app.log');

log.transports.file.maxSize = 10 * 1024 * 1024;  // 10MB 滚动
log.transports.file.archiveLog = (oldLogFile) => {
  const gzip = zlib.createGzip();
  const readStream = fs.createReadStream(oldLogFile);
  const writeStream = fs.createWriteStream(`${oldLogFile}.gz`);
  readStream.pipe(gzip).pipe(writeStream);
};

export const logger = {
  trace: (module: string, message: string, meta?: unknown) => 
    log.verbose(`[${module}] ${message}`, meta),
  debug: (module: string, message: string, meta?: unknown) => 
    log.debug(`[${module}] ${message}`, meta),
  info: (module: string, message: string, meta?: unknown) => 
    log.info(`[${module}] ${message}`, meta),
  warn: (module: string, message: string, meta?: unknown) => 
    log.warn(`[${module}] ${message}`, meta),
  error: (module: string, message: string, error?: Error) => 
    log.error(`[${module}] ${message}`, { error: error?.stack }),
  fatal: (module: string, message: string, error?: Error) => {
    log.error(`[FATAL][${module}] ${message}`, { error: error?.stack });
    // 触发崩溃报告
    crashReporter.captureException(error || new Error(message));
  },
};
```

**渲染进程**：通过 IPC 将日志发送到主进程统一写入。

```typescript
// renderer/utils/logger.ts
export const logger = {
  info: (module: string, message: string, meta?: unknown) => {
    console.log(`[${module}] ${message}`, meta);
    window.electron.invoke('main:log', { level: 'INFO', module, message, meta });
  },
  // ... debug, warn, error 同理
};
```

### 1.5 敏感信息脱敏

日志中自动脱敏以下信息：

| 敏感字段 | 脱敏方式 |
|----------|----------|
| API Key / Token | 只保留前 4 位 + `****` |
| 文件绝对路径 | 替换为相对项目根目录的路径 |
| 用户目录名 | 替换为 `~` |
| LLM 提示内容 | 只记录长度，不记录内容 |
| 素材文件内容 | 不记录 |

## 2. 崩溃报告

### 2.1 崩溃捕获

**Electron 内置崩溃报告器：**

```typescript
// main/index.ts
import { crashReporter } from 'electron';

crashReporter.start({
  submitUrl: '',  // 不上传到远程，仅本地保存
  uploadToServer: false,
  ignoreSystemCrashHandler: true,
  extra: {
    appVersion: app.getVersion(),
    osVersion: os.release(),
    platform: process.platform,
  },
});

// 崩溃文件保存路径
const crashesDirectory = path.join(app.getPath('userData'), 'logs', 'crashes');
app.setPath('crashDumps', crashesDirectory);
```

**未捕获异常处理：**

```typescript
// 主进程
process.on('uncaughtException', (error) => {
  logger.fatal('main', '未捕获异常', error);
  // 保存崩溃上下文
  saveCrashContext(error);
  // 优雅退出
  app.quit();
});

process.on('unhandledRejection', (reason) => {
  logger.error('main', '未处理的 Promise 拒绝', reason as Error);
});

// 渲染进程
window.onerror = (message, source, lineno, colno, error) => {
  logger.error('renderer', '渲染进程错误', error || new Error(String(message)));
  return true;
};

window.onunhandledrejection = (event) => {
  logger.error('renderer', '未处理的 Promise 拒绝', event.reason);
};
```

### 2.2 崩溃上下文保存

崩溃时额外保存以下信息：

```typescript
async function saveCrashContext(error: Error) {
  const context = {
    timestamp: Date.now(),
    appVersion: app.getVersion(),
    platform: process.platform,
    arch: process.arch,
    memory: process.getSystemMemoryInfo(),
    cpuUsage: process.getCPUUsage(),
    lastActions: actionHistory.slice(-20),  // 最近 20 个用户操作
    currentProject: currentProjectId,
    currentSubproject: currentSubprojectId,
    openWindows: BrowserWindow.getAllWindows().map(w => ({
      id: w.id,
      title: w.getTitle(),
      url: w.webContents.getURL(),
    })),
    error: {
      name: error.name,
      message: error.message,
      stack: error.stack,
    },
  };
  
  const filename = `crash-${formatDate(new Date(), 'yyyy-MM-dd-HHmmss')}.json`;
  await fs.writeFile(
    path.join(crashesDirectory, filename),
    JSON.stringify(context, null, 2)
  );
}
```

## 3. 性能监控

### 3.1 监控指标

| 指标 | 采集方式 | 阈值 | 告警 |
|------|----------|------|------|
| 应用启动时间 | `performance.now()` | > 5s | WARN |
| 内存占用（主进程） | `process.memoryUsage()` | > 500MB | WARN |
| 内存占用（渲染进程） | `webContents.getProcessMemoryInfo()` | > 1GB | WARN |
| 预览帧率 | iframe 上报 | < 15fps 持续 3s | WARN |
| IPC 调用耗时 | 请求/响应时间戳差 | > 平均 3 倍 | WARN |
| 文件写入耗时 | `fs.writeFile` 耗时 | > 1s | WARN |
| SQLite 查询耗时 | 查询前后时间戳 | > 100ms | WARN |
| LLM 响应时间 | 请求/首 token 时间 | > 60s | ERROR |
| 导出速度 | 帧/秒 | < 0.25× 实时 | WARN |
| 卡顿检测（FPS） | `requestAnimationFrame` 计数 | < 30fps 持续 5s | WARN |

### 3.2 性能日志采样

```typescript
// 开发模式：记录所有性能指标
// 生产模式：采样记录（每 10 分钟记录一次快照）

const PERF_LOG_INTERVAL = process.env.NODE_ENV === 'development' ? 5000 : 600000;

setInterval(() => {
  const metrics = {
    timestamp: Date.now(),
    memory: process.memoryUsage(),
    cpu: process.getCPUUsage(),
    fps: previewStore.getState().fps,
    ipcLatency: calculateAverageIPCLatency(),
    sqliteLatency: calculateAverageSQLiteLatency(),
  };
  
  logger.debug('performance', '性能快照', metrics);
}, PERF_LOG_INTERVAL);
```

## 4. 用户行为分析（可选，匿名）

### 4.1 采集范围（仅统计，不含个人数据）

| 事件 | 说明 | 示例 |
|------|------|------|
| `feature_used` | 功能使用次数 | "时间线拖拽"、"关键帧编辑" |
| `preset_applied` | 预设使用统计 | "柱状图预设被使用 50 次" |
| `export_completed` | 导出完成统计 | 分辨率、格式、时长分布 |
| `error_occurred` | 错误发生统计 | 错误码频率分布 |
| `session_duration` | 单次使用时长 | 平均 30 分钟 |
| `project_complexity` | 项目复杂度分布 | 轨道数、片段数分布 |

### 4.2 隐私保护

- **默认关闭**：所有分析功能默认关闭，用户主动开启。
- **匿名化**：不上传用户 ID、项目内容、文件路径。
- **本地聚合**：数据先在本地聚合，不上传原始事件。
- **随时关闭**：设置面板可随时关闭并删除已收集数据。

## 5. 告警机制

### 5.1 开发模式告警

```
开发模式下，以下情况在控制台输出警告（黄色）：
├── IPC 调用耗时 > 1s
├── 状态更新导致多余重渲染（React DevTools Profiler）
├── 内存增长 > 100MB/小时
├── 预览帧率 < 20fps
└── SQLite 查询 > 50ms
```

### 5.2 生产模式告警

```
生产模式下，以下情况记录 WARN 日志：
├── 内存 > 1GB
├── 预览帧率 < 15fps 持续 5s
├── 导出失败
├── LLM 请求失败 > 3 次/会话
└── 文件写入失败
```

## 7. 日志清理策略

| 日志类型 | 自动清理条件 | 手动清理入口 |
|----------|-------------|-------------|
| 应用日志（app.log） | 滚动保留 30 天 | 设置 → 存储 → 清理日志 |
| 错误日志（app.error.log） | 保留 90 天 | 同上 |
| 崩溃转储 | 保留最近 10 个 | 同上 |
| 性能日志 | 保留 7 天 | 同上 |
| 归档日志（.gz） | 保留 6 个月 | 同上 |
| IPC 日志 | 开发模式不自动清理 | 同上 |
