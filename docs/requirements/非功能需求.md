# 非功能需求

## 概述

本文档定义 EasyMotion 在性能、可靠性、安全性、可维护性、兼容性和易用性方面的详细非功能需求指标和设计约束。

---

## 性能需求

### 1. 应用启动性能

| 指标 | 目标值 | 测试条件 | 说明 |
|------|--------|----------|------|
| 冷启动时间 | < 5 秒 | 普通 SSD，首次安装后启动 | 从双击图标到主窗口可交互 |
| 热启动时间 | < 2 秒 | 已创建过项目，非首次启动 | 复用缓存和预加载数据 |
| 项目窗口打开 | < 1 秒 | 项目列表 ≤ 50 个 | 从点击"打开项目"到窗口显示 |
| 子项目切换 | < 500ms | 子项目时间线 ≤ 10 轨道 | 从点击切换到预览就绪 |

### 2. 预览性能

| 指标 | 目标值 | 测试条件 | 说明 |
|------|--------|----------|------|
| 1080p 预览帧率 | ≥ 25fps | 单轨道、简单动画 | Remotion Player 实时渲染帧率 |
| 1080p 预览帧率（复杂项目） | ≥ 15fps | 8 轨道以上、含关键帧动画 | 复杂项目最低可接受帧率 |
| 4K 预览帧率 | ≥ 10fps | 单轨道 | 4K 作为预览选项，非默认 |
| 预览缩放响应 | < 100ms | 50% → 200% 缩放 | 缩放操作到画面更新 |
| 播放头拖拽响应 | < 50ms | 任意位置 seek | 拖拽播放头到画面更新 |

**帧率降级策略：**
- 当预览帧率持续 3 秒 < 15fps 时，自动降级：
  1. 降低预览分辨率（100% → 75% → 50%）
  2. 关闭特效（阴影、模糊、混合模式）
  3. 提示用户"预览性能不足，已自动优化"
- 用户可手动关闭自动降级

### 3. 渲染导出性能

| 指标 | 目标值 | 测试条件 | 说明 |
|------|--------|----------|------|
| 1080p 导出速度 | ≥ 实时（1:1） | 单轨道文字动画 | 1 秒视频 ≤ 1 秒渲染时间 |
| 1080p 导出速度（复杂） | ≥ 0.5× 实时 | 8 轨道、含视频素材 | 1 秒视频 ≤ 2 秒渲染时间 |
| 4K 导出速度 | ≥ 0.25× 实时 | 中等复杂度项目 | 1 秒视频 ≤ 4 秒渲染时间 |
| 导出进度更新 | 每 1 秒 | 任意导出任务 | 界面显示当前帧/总帧/百分比 |
| 取消导出响应 | < 2 秒 | 任意时刻点击取消 | 从点击到渲染进程终止 |

### 4. AI 生成性能

| 指标 | 目标值 | 测试条件 | 说明 |
|------|--------|----------|------|
| 简单生成响应 | < 15 秒 | 纯文字需求，1 个元素 | 从发送到预览更新 |
| 复杂生成响应 | < 45 秒 | 含图片参考，5 个元素 | 从发送到预览更新 |
| 对话调整响应 | < 10 秒 | 单属性修改 | 从发送到预览更新 |
| 流式反馈延迟 | < 2 秒 | 首次文本输出 | 用户发送后到看到 AI 开始回复 |

### 5. 内存与资源

| 指标 | 目标值 | 测试条件 | 说明 |
|------|--------|----------|------|
| 基础内存占用 | < 500MB | 空项目，主窗口打开 | Electron 主进程 + 渲染进程 |
| 正常使用内存 | < 1GB | 3 个子项目，含素材 | 含 Remotion Player、素材缓存 |
| 复杂项目内存 | < 2GB | 10+ 轨道，4K 预览 | 含高分辨率素材、大量关键帧 |
| 内存泄漏 | 无 | 连续操作 4 小时 | 内存增长 < 10% |
| 磁盘临时空间 | < 2GB | 正常使用 | `~/.easymotion/temp/` 和项目缓存 |
| SQLite 查询 | < 100ms | 素材库 ≤ 1000 条 | 常规查询响应时间 |

## 可靠性需求

### 1. 崩溃恢复

| 场景 | 恢复策略 | 数据保护 |
|------|----------|----------|
| 渲染进程崩溃 | 自动重载窗口，提示"界面已刷新" | 已保存的 project.json 不丢失 |
| 主进程崩溃 | 应用退出，下次启动提示恢复 | 快照文件恢复最近状态 |
| Python 子进程崩溃 | 自动重启，期间服务暂不可用 | 不影响已生成的 Remotion 代码 |
| 系统断电/强制退出 | 下次启动扫描快照，提示恢复 | 自动保存间隔 ≤ 30 秒 |

### 2. 自动保存

| 参数 | 设定值 | 说明 |
|------|--------|------|
| 常规自动保存间隔 | 30 秒 | timeline 无操作时触发 |
| 关键操作立即保存 | 是 | 删除轨道/片段、重命名、新建子项目 |
| AI 生成完成后保存 | 是 | 生成结果立即持久化 |
| 保存失败重试 | 3 次 | 每次间隔 1 秒，仍失败提示用户 |
| 保存目标 | project.json + subproject.json | 原子写入：先写临时文件，再重命名 |

### 3. 数据完整性

| 保障措施 | 说明 |
|----------|------|
| 原子写入 | 所有 JSON 文件写入采用"临时文件 → 重命名"策略，避免写一半中断 |
| 备份文件 | 每次覆盖 `project.json` 前，自动复制为 `project.json.bak`（保留最近 3 个） |
| JSON 校验 | 写入后读取验证 JSON 可解析性，失败则回退到备份 |
| 文件校验 | 素材文件复制完成后验证文件大小和 SHA-256，不一致则重新复制 |
| 项目损坏检测 | 打开项目时校验 `project.json` 结构，损坏时尝试从备份恢复 |

### 4. 可用性指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 平均无故障时间（MTBF） | > 48 小时 | 正常使用下的平均崩溃间隔 |
| 平均恢复时间（MTTR） | < 30 秒 | 从崩溃到恢复可用的时间 |
| 操作成功率 | > 99% | 新建项目、保存、导出等核心操作 |

## 安全性需求

### 1. LLM API Key 安全

| 措施 | 实现 |
|------|------|
| 存储加密 | AES-256-GCM，密钥由系统钥匙串派生 |
| 内存安全 | 仅在请求时临时解密，使用后覆写内存 |
| 传输安全 | 强制 HTTPS，禁止 HTTP 回退 |
| 访问控制 | 渲染进程无法直接读取 Key，通过 IPC 由主进程代理调用 |
| 泄露检测 | 日志中 Key 字段自动脱敏（显示前 4 位 + "****"） |

### 2. 用户数据隐私

| 措施 | 实现 |
|------|------|
| 本地运行 | 所有数据和素材存储在用户本地磁盘，不上传云端 |
| 项目隔离 | 每个项目独立文件夹，禁止跨项目访问 |
| 素材沙箱 | 生成的 Remotion 代码在 iframe 中运行，`sandbox="allow-scripts"` |
| 无遥测（默认） | 不收集用户行为数据、不上报崩溃信息（可手动开启调试模式） |
| 对话历史 | 仅存于本地 `conversation.json`，删除项目时一并删除 |

### 3. 代码执行安全

| 措施 | 实现 |
|------|------|
| 白名单机制 | LLM 生成的代码只允许使用预定义的 API 和模块 |
| AST 静态分析 | 写入前通过 AST 检查，禁止危险 API（eval、require、iframe 等） |
| 正则黑名单 | 补充检查禁止的字符串模式 |
| 安全违规处理 | 发现危险代码时拒绝写入，标记为异常，提示用户重新生成 |

### 4. 文件系统安全

| 措施 | 实现 |
|------|------|
| 路径限制 | 所有文件操作限定在项目根目录内 |
| 路径校验 | 解析路径后检查是否在项目目录内，防止 `../` 逃逸 |
| 敏感路径禁止 | 禁止访问 `/System`、`C:\Windows`、`~/.ssh` 等系统路径 |
| 文件类型限制 | 素材导入时校验文件头魔数，防止伪装扩展名攻击 |

## 可维护性需求

### 1. 日志系统

| 层级 | 输出 | 内容 | 保留期限 |
|------|------|------|----------|
| ERROR | `~/.easymotion/logs/error.log` | 崩溃、异常、失败操作 | 30 天 |
| WARN | 同上 | 降级处理、非预期行为 | 30 天 |
| INFO | `~/.easymotion/logs/app.log` | 关键操作（新建、保存、导出） | 7 天 |
| DEBUG | 内存（不写入文件） | 详细调用链路、状态变更 | 不保留 |

**日志格式：** `[ISO时间] [级别] [模块] 消息 {上下文JSON}`

**示例：**
```
[2026-05-30T14:23:45.123Z] [INFO] [project-service] 项目创建成功 {"path":"/Users/xxx/EasyMotion/我的项目","durationMs":234}
[2026-05-30T14:24:12.456Z] [WARN] [llm-service] LLM调用超时，进入重试 {"attempt":1,"maxRetries":2}
[2026-05-30T14:25:01.789Z] [ERROR] [generator] 生成代码安全校验失败 {"violations":["禁止导入模块: fs"],"clipId":"clip-001"}
```

### 2. 错误上报（可选）

| 参数 | 设定 |
|------|------|
| 默认状态 | 关闭（保护隐私） |
| 用户控制 | 设置中可手动开启"帮助改进产品" |
| 上报内容 | 错误类型、堆栈跟踪（脱敏）、应用版本、操作系统版本 |
| 不上报内容 | 用户文件路径、LLM Key、项目内容、对话历史 |
| 上报方式 | 崩溃后下次启动时询问用户是否上报 |

### 3. 调试模式

| 功能 | 触发方式 | 说明 |
|------|----------|------|
| 开发者工具 | F12 / Cmd+Option+I | 打开 Chromium DevTools |
| 日志级别切换 | 设置中开启"调试模式" | 所有 DEBUG 日志写入文件 |
| IPC 日志 | 调试模式下开启 | 记录所有 IPC 请求/响应 |
| Generator 输出保留 | 调试模式下开启 | 保留每次 Generator 的中间文件 |
| 性能监控 | 调试模式下开启 | 在界面显示 FPS、内存、Generator 耗时 |

### 4. 版本兼容性

| 场景 | 策略 |
|------|------|
| 应用升级 | 向后兼容：新版本可打开旧项目；升级时自动迁移项目结构 |
| 项目版本标记 | `project.json` 中 `version` 字段标记项目格式版本 |
| 迁移脚本 | 大版本升级时提供自动迁移脚本（如从 v1.0 → v2.0） |
| 降级限制 | 不支持降级打开（新项目格式不能在旧版本打开） |

## 兼容性需求

### 1. 操作系统支持

| 系统 | 最低版本 | 架构 | 说明 |
|------|----------|------|------|
| macOS | 12 (Monterey) | Intel / Apple Silicon | 原生支持 M1/M2/M3 |
| Windows | Windows 10 (1909) | x64 | 不支持 32 位和 ARM |
| Linux | Ubuntu 20.04 LTS | x64 | 其他发行版需验证 |

### 2. 硬件要求

| 组件 | 最低要求 | 推荐配置 | 说明 |
|------|----------|----------|------|
| CPU | 4 核 | 8 核以上 | 影响渲染速度和预览帧率 |
| 内存 | 8GB | 16GB | 复杂项目和 4K 预览需要更多 |
| 磁盘 | 2GB 可用空间 | 10GB+ | 含应用本身、项目文件、缓存 |
| 显卡 | 支持 WebGL | 独立显卡 | 影响 Remotion 预览性能 |
| 网络 | 可选 | 首次安装时需下载依赖 | 运行时不需要网络 |

### 3. 外部依赖兼容性

| 依赖 | 版本锁定 | 升级策略 |
|------|----------|----------|
| Remotion | v4.0.269 | 大版本升级需测试所有 Layer 组件 |
| Electron | v30.x | 跟随 Electron 稳定版，每 3 个月评估升级 |
| Node.js | v20 LTS | 跟随 Node.js LTS 版本 |
| Python | 3.10+ | 最低 3.10，推荐 3.11 |
| FFmpeg | 6.0+ | 随 Remotion 依赖 |

### 4. 显示器适配

| 参数 | 支持范围 |
|------|----------|
| 分辨率 | 最低 1280×720，推荐 1920×1080 以上 |
| 缩放 | 支持 100%、125%、150%、175%、200% DPI 缩放 |
| 高刷新率 | 支持 60Hz、120Hz、144Hz（预览帧率独立，不受显示器刷新率限制） |

## 易用性需求

### 1. 零代码门槛

| 指标 | 目标 |
|------|------|
| 首次使用完成动画 | 用户首次打开应用后，5 分钟内生成并预览第一个动画 |
| 功能发现性 | 核心功能（新建项目、对话生成、导出）在主界面直观可见 |
| 错误提示友好 | 所有错误提示使用自然语言，避免技术术语，提供解决建议 |
| 帮助入口 | 每个面板右上角提供"?"帮助按钮，链接到对应文档 |

### 2. 交互响应

| 操作 | 最大响应时间 | 反馈方式 |
|------|-------------|----------|
| 按钮点击 | < 100ms | 视觉反馈（按钮按下态） |
| 拖拽操作 | < 50ms | 实时跟随光标 |
| 面板展开/收起 | < 200ms | 平滑动画过渡 |
| 菜单弹出 | < 100ms | 即时显示 |
| 加载状态 | < 300ms | 显示骨架屏或进度条 |

### 3. 快捷键支持

| 功能 | Windows/Linux | macOS |
|------|---------------|-------|
| 新建项目 | Ctrl+N | Cmd+N |
| 保存 | Ctrl+S | Cmd+S |
| 撤销 | Ctrl+Z | Cmd+Z |
| 重做 | Ctrl+Y / Ctrl+Shift+Z | Cmd+Shift+Z |
| 播放/暂停 | Space | Space |
| 下一帧 | → | → |
| 上一帧 | ← | ← |
| 全屏预览 | F11 | Cmd+Ctrl+F |
| 开发者工具 | F12 | Cmd+Option+I |

### 4. 无障碍（Accessibility）

| 要求 | 实现 |
|------|------|
| 键盘导航 | 所有交互元素支持 Tab 键聚焦和 Enter/Space 激活 |
| 焦点指示 | 聚焦元素有明确的视觉高亮边框 |
| 颜色对比 | 文字与背景对比度 ≥ 4.5:1（WCAG AA 标准） |
| 屏幕阅读器 | 关键 UI 元素添加 `aria-label` 和 `role` 属性 |
| 减少动画 | 支持系统"减少动画"设置，关闭非必要动画 |
