# 项目管理

## 项目窗口

- 独立窗口，用于浏览、创建、打开、删除项目
- 显示项目列表（缩略图、名称、创建时间、修改时间）
- 支持搜索和筛选（按名称、最近打开时间）
- 支持右键菜单（打开、重命名、删除、在文件夹中显示）

---

## 项目层级结构

### 总项目（视频期）
- 代表一期完整的视频内容
- 包含多个子项目（动画片段）
- 总项目配置文件记录整体分辨率、帧率、时长等

### 子项目（动画片段）
- 每个子项目是独立的动画片段
- 子项目独立文件夹，避免混乱
- 子项目可单独预览、渲染、导出
- 总项目时间线可引用多个子项目进行组合

---

## 总项目结构

```
总项目文件夹/
├── project.json              # 总项目配置（分辨率、帧率、子项目列表等）
├── subprojects/              # 子项目目录
│   ├── 动画片段1/
│   │   ├── remotion/         # 完整的独立 Remotion 项目
│   │   │   ├── node_modules/ # npm 依赖（独立安装，版本隔离）
│   │   │   ├── src/          # 源代码（Root.tsx, 组件等）
│   │   │   ├── package.json  # 依赖配置和脚本
│   │   │   ├── remotion.config.ts # Remotion 配置
│   │   │   ├── tsconfig.json # TypeScript 配置
│   │   │   └── ...
│   │   ├── assets/           # 子项目素材
│   │   ├── output/           # 子项目渲染输出
│   │   ├── data/             # 数据文件
│   │   ├── .thumbnails/      # 素材缩略图缓存
│   │   ├── timeline-snapshots/  # 时间线自动快照（崩溃恢复用）
│   │   ├── conversation.json # 对话历史
│   │   └── subproject.json   # 子项目配置
│   └── 动画片段2/
│       └── ...
├── assets/                   # 总项目共享素材
│   ├── images/               # 图片素材
│   ├── videos/               # 视频素材
│   ├── audios/               # 音频素材
│   └── fonts/                # 字体文件
├── output/                   # 总项目最终输出（合并后的视频）
├── data/                     # 总项目共享数据文件
└── .thumbnails/              # 总项目共享素材缩略图
```

## 子项目隔离优势

- **完全隔离**：每个子项目的依赖互不干扰，避免版本冲突
- **版本锁定**：可针对不同子项目使用不同 Remotion 版本（默认统一）
- **导出方便**：直接导出 `remotion/` 文件夹，对方 `npm install` 即可独立运行

---

## 项目隔离

- 所有文件操作限定在项目文件夹内
- 素材自动复制到项目 `assets/` 目录，避免外部依赖
- 删除项目时可选择是否保留输出文件

---

## 项目模板

- 内置基础 Remotion 模板（含 `Root.tsx`、`MainSequence.tsx` 骨架）
- 新建总项目时自动创建模板结构
- 新建子项目时复制 Remotion 模板到子项目目录
- 支持用户自定义模板（保存当前子项目为模板，下次新建时可选）

### 模板结构

```
templates/
└── default-project/
    ├── project.json              # 默认总项目配置
    └── subprojects/
        └── default/
            ├── subproject.json   # 默认子项目配置
            ├── remotion/
            │   ├── package.json
            │   ├── remotion.config.ts
            │   ├── tsconfig.json
            │   └── src/
            │       ├── index.tsx
            │       ├── Root.tsx
            │       └── components/
            │           ├── MainSequence.tsx
            │           └── layers/
            │               ├── TextLayer.tsx
            │               ├── ImageLayer.tsx
            │               ├── VideoLayer.tsx
            │               ├── AudioLayer.tsx
            │               ├── ShapeLayer.tsx
            │               └── ChartLayer.tsx
            └── assets/
```

## 版本管理

### 自动保存
- 项目状态自动保存到 `project.json` 和 `subproject.json`
- 保存策略：修改后 30 秒内无新操作则触发保存（防抖）
- 关键操作（删除、重命名、新建子项目）立即保存
- 保存方式：原子写入（先写临时文件，验证 JSON 后重命名）

> **详细保存策略、并发控制和文件校验**见 `技术规格.md` → "项目文件并发控制" 和 "数据存储方案"章节。

### 手动快照
- 用户可随时创建手动快照（"文件 → 创建快照"）
- 快照包含完整的 `subproject.json` + `conversation.json`
- 快照命名：`快照_{日期时间}.json`
- 存储路径：`timeline-snapshots/`

### 回滚机制
- 支持回滚到最近的手动快照或自动快照
- 回滚时：读取快照文件 → 替换当前 `subproject.json` → 触发 Generator 重新生成 → 更新预览
- 回滚操作本身也进入撤销栈（用户可撤销回滚）

### 子项目版本独立
- 每个子项目的快照和回滚独立管理
- 切换子项目时，撤销栈自动清空（避免跨项目误操作）

## 项目文件详细定义

### project.json（总项目配置）

```json
{
  "version": "1.0",
  "name": "我的项目",
  "createdAt": "2026-05-30T10:00:00Z",
  "modifiedAt": "2026-05-30T14:30:00Z",
  "settings": {
    "defaultResolution": { "width": 1920, "height": 1080 },
    "defaultFps": 30,
    "defaultDurationInFrames": 300
  },
  "subprojects": [
    {
      "id": "sub-001",
      "name": "开场动画",
      "path": "subprojects/开场动画",
      "createdAt": "2026-05-30T10:00:00Z",
      "modifiedAt": "2026-05-30T12:00:00Z"
    }
  ],
  "remotionVersion": "4.0.269"
}
```

### subproject.json（子项目配置）

```json
{
  "version": "1.0",
  "id": "sub-001",
  "name": "开场动画",
  "timeline": {
    "fps": 30,
    "durationInFrames": 300,
    "width": 1920,
    "height": 1080,
    "tracks": [...]
  },
  "conversation": {
    "messages": [...]
  },
  "assets": [
    { "id": "asset-001", "name": "logo.png", "type": "image", "path": "assets/images/xxx_logo.png" }
  ]
}
```

> **timeline 和 conversation 的详细数据结构**见 `时间线编辑.md` 和 `交互流程.md`。
