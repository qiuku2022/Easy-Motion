# 素材处理流水线

> **实施状态（2026-06-26）**：M0–M8 已落地，M9（E2E/打包/性能）进行中。权威入口：仓库根 [README.md](../../README.md)、[开发里程碑与路线图.md](./开发里程碑与路线图.md)。

## 概述

本文档定义素材从**导入触发**到**渲染使用**的完整处理流程，包括格式验证、文件复制、元数据提取、缩略图生成、数据库存储和缓存管理。所有流程与 `交互流程.md`、`支持格式.md`、`数据库Schema设计.md`、`技术规格.md` 对齐。

> **版本**：v0.1  
> **最后更新**：2026-06-26  
> **关联文档**：交互流程.md、支持格式.md、数据库Schema设计.md、技术规格.md、IPC通信协议规范.md

## 1. 导入触发

### 触发方式

| 方式 | 入口 | 限制 | 适用场景 |
|------|------|------|----------|
| **拖拽导入** | 素材库面板、时间线轨道 | 单次最多 50 个文件 | 快速批量导入 |
| **文件选择** | 菜单栏「文件」→「导入素材」 | 单次最多 100 个文件 | 精确选择 |
| **项目目录扫描** | 右键「扫描文件夹」 | 递归子目录，最大 500 个文件 | 批量导入外部目录 |
| **AI 自动下载** | Agent 爬虫模块 | 受网络和存储限制 | 参考素材自动抓取 |

### 导入预处理

1. **去重检查**：基于文件名 + 文件大小快速预过滤（不计算完整哈希，避免阻塞 UI）。
2. **数量限制**：单次导入超过 50 个文件时，分批次处理（每批 10 个），每批完成后更新 UI。
3. **总大小预警**：导入文件总大小 > 1GB 时，弹窗提示"导入文件较大，可能需要较长时间"。

## 2. 格式验证

### 2.1 支持的格式清单

与 `支持格式.md` 一致：

| 类型 | 扩展名 | MIME Type | 最大大小 |
|------|--------|-----------|----------|
| 图片 | `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif` | `image/*` | 50MB |
| 视频 | `.mp4`, `.mov`, `.webm`, `.avi`, `.mkv` | `video/*` | 2GB |
| 音频 | `.mp3`, `.wav`, `.aac`, `.ogg`, `.flac` | `audio/*` | 500MB |
| 数据 | `.csv`, `.json`, `.xlsx` | `text/csv`, `application/json` | 50MB |
| 字体 | `.ttf`, `.otf`, `.woff`, `.woff2` | `font/*` | 20MB |

### 2.2 验证流程

```
文件进入验证队列
    │
    ├── 1. 扩展名检查
    │      └── 不在白名单 → 拒绝，错误码 E2031
    │
    ├── 2. MIME Type 检查（libmagic / file-type）
    │      └── 与扩展名不一致 → 警告但允许继续（如 .png 实际是 .jpg）
    │
    ├── 3. 文件头签名检查（Magic Number）
    │      └── 无法识别 → 拒绝，错误码 E2031
    │
    ├── 4. 文件大小检查
    │      └── 超过类型限制 → 拒绝，提示"文件过大"
    │
    ├── 5. 损坏检测（快速扫描）
    │      ├── 图片：检查 IHDR/EOI 标记
    │      ├── 视频：检查 moov/ftyp 标记
    │      └── 损坏 → 拒绝，提示"文件可能已损坏"
    │
    └── 6. 重复检查（完整 SHA-256）
           └── 已存在相同内容 → 提示"素材已存在，是否添加引用"
```

### 2.3 验证结果处理

| 结果 | UI 反馈 | 后续操作 |
|------|---------|----------|
| 通过 | 显示在导入队列中 | 进入复制阶段 |
| 格式不支持 | 红色标记，显示原因 | 跳过，提供转换建议 |
| 文件损坏 | 黄色警告，显示"可能损坏" | 用户确认后可选继续 |
| 重复文件 | 蓝色标记，显示"已存在" | 用户选择"跳过"或"添加引用" |
| 超出大小 | 红色标记，显示大小限制 | 跳过，建议压缩 |

## 3. 文件复制

### 3.1 目标路径结构

```
项目根目录/
├── assets/                     # 所有导入素材
│   ├── images/
│   │   └── {uuid}.png
│   ├── videos/
│   │   └── {uuid}.mp4
│   ├── audio/
│   │   └── {uuid}.mp3
│   ├── data/
│   │   └── {uuid}.csv
│   └── fonts/
│       └── {uuid}.ttf
├── .thumbnails/               # 缩略图缓存
│   └── {uuid}_{宽}x{高}.jpg
└── project.json
```

### 3.2 复制策略

| 策略 | 说明 | 默认行为 |
|------|------|----------|
| **复制模式**（默认） | 复制文件到项目 `assets/` 目录，保留原始文件 | ✅ 默认 |
| **引用模式** | 不复制，记录原始路径（外部素材） | 可选，需用户确认 |
| **移动模式** | 剪切原始文件到项目目录 | 不常用，高级选项 |

### 3.3 重名处理

1. 素材在项目内使用 **UUID + 原始文件名** 命名（`{uuid}_{originalName}.{ext}`），避免重名且保留可识别性。
2. `original_name` 字段记录用户上传时的原始文件名（用于显示和搜索）。
3. 同内容去重（SHA-256 相同）→ 复用已有文件，不重复存储。

### 3.4 复制进度反馈

- 大文件（> 100MB）复制时，每 10% 进度通过 IPC `renderer:asset:import-progress` 推送。
- 复制失败（磁盘满、权限不足）→ 回滚已复制的部分文件，返回错误码 `E2005`。

## 4. 元数据提取

### 4.1 提取工具

| 素材类型 | 工具 | 说明 |
|----------|------|------|
| 图片 | `sharp`（Node.js） | 快速、支持 WebP/AVIF |
| 视频 | `ffprobe`（FFmpeg） | 提取时长、分辨率、编码、帧率 |
| 音频 | `ffprobe` / `music-metadata` | 提取时长、比特率、采样率 |
| 数据 | 自定义解析器 | CSV/JSON/XLSX 行列数、编码 |
| 字体 | `opentype.js` | 提取字体名称、字重、字符集 |

### 4.2 提取字段

与 `数据库Schema设计.md` 中 `assets` 表字段对齐：

```typescript
interface AssetMetadata {
  // 基础信息
  id: string;                    // UUID
  name: string;                  // 原始文件名
  type: 'image' | 'video' | 'audio' | 'font' | 'data';
  mimeType: string;
  sizeBytes: number;
  contentHash: string;           // SHA-256

  // 媒体信息（图片/视频/音频）
  width?: number;                // 像素
  height?: number;               // 像素
  durationMs?: number;          // 视频/音频时长（毫秒）
  frameRate?: number;            // 视频帧率
  bitRate?: number;             // 音频/视频比特率（bps）
  sampleRate?: number;           // 音频采样率（Hz）
  channels?: number;             // 音频声道数

  // 数据文件信息
  rowCount?: number;             // CSV/Excel 行数
  columnCount?: number;          // CSV/Excel 列数
  encoding?: string;             // 文件编码（UTF-8/GBK 等）

  // 字体信息
  fontFamily?: string;
  fontWeight?: number;
  fontStyle?: string;

  // 缩略图
  thumbnailPath?: string;

  // 管理信息
  projectId: string;
  path: string;                  // 项目内相对路径
  absolutePath?: string;         // 绝对路径（缓存）
  createdAt: number;
}
```

### 4.3 提取失败处理

| 场景 | 处理策略 | 数据库记录 |
|------|----------|----------|
| ffprobe 无法解析视频 | 标记 `width=0, height=0, durationMs=0` | 仍然保存，显示"元数据不可用" |
| 图片无法解码 | 拒绝，提示"文件已损坏" | 不写入数据库 |
| CSV 编码错误 | 尝试 GBK/UTF-8 自动检测 | 记录检测到的编码 |
| XLSX 解析失败 | 降级为二进制文件处理 | 不解析行列数 |

## 5. 缩略图生成

### 5.1 生成策略

| 素材类型 | 缩略图来源 | 尺寸 | 格式 |
|----------|-----------|------|------|
| 图片 | 原图缩放 | 160×160（素材库）、320×180（预览） | JPEG，质量 85 |
| 视频 | 首帧截图（FFmpeg） | 320×180 | JPEG，质量 85 |
| 音频 | 波形图（FFmpeg + Canvas） | 320×60 | PNG |
| 数据 | 前 5 行文本截图（Canvas） | 200×120 | PNG |
| 字体 | 字符预览图（A-Z 0-9） | 200×80 | PNG |

### 5.2 生成流程

```
素材复制完成
    │
    ├── 图片 ──► sharp.resize(160, 160).jpeg({ quality: 85 })
    │             └── 保存到 .thumbnails/{uuid}_160x160.jpg
    │
    ├── 视频 ──► ffmpeg -i input -ss 00:00:00 -vframes 1 output.jpg
    │             └── 保存到 .thumbnails/{uuid}_320x180.jpg
    │
    ├── 音频 ──► ffmpeg -i input -filter_complex showwavespic output.png
    │             └── 保存到 .thumbnails/{uuid}_320x60.png
    │
    ├── 数据 ──► Canvas 绘制前 5 行文本
    │             └── 保存到 .thumbnails/{uuid}_200x120.png
    │
    └── 字体 ──► opentype.js 加载 + Canvas 绘制字符
                  └── 保存到 .thumbnails/{uuid}_200x80.png
```

### 5.3 异步与队列

- 缩略图生成是**异步操作**，不阻塞导入流程。
- 使用 **Promise 队列**限制并发：最多同时生成 5 个缩略图。
- 生成完成后通过 IPC 通知 Renderer 更新素材库 UI。

### 5.4 缩略图缓存

- 存储路径：`.thumbnails/{uuid}_{宽}x{高}.{ext}`
- 缓存策略：
  - 缩略图与素材绑定，素材删除时同步删除缩略图。
  - 启动时扫描 `.thumbnails/`，删除无对应素材的孤儿缩略图。
  - 缩略图总大小超过 500MB 时，按 LRU 清理最早生成的缩略图。

### 5.5 缩略图失败回退

| 场景 | 回退策略 |
|------|----------|
| 视频首帧提取失败 | 使用纯色占位图 + 文件名 |
| 音频波形生成失败 | 使用音频图标 + 时长文本 |
| 图片缩放失败 | 使用默认图片图标 |

## 6. 数据库存储

### 6.1 写入时机

1. 文件复制完成后 → 写入 `assets` 表（基础信息 + 路径）
2. 元数据提取完成后 → 更新 `assets` 表（宽高、时长等）
3. 缩略图生成完成后 → 更新 `thumbnail_path` 字段

### 6.2 批量写入优化

- 单次导入多个素材时，使用 SQLite 事务批量写入（BEGIN → INSERT × N → COMMIT）。
- 每批最多 50 条，避免事务过大。

### 6.3 与 project.json 的关系

- `library.db` 的 `assets` 表是**素材元数据索引**。
- 实际素材文件存储在项目 `assets/` 目录。
- 子项目时间线中通过 `assetId` 引用素材，不存储完整路径。
- 素材移动或重命名时，只更新数据库 `path` 字段，不修改时间线引用。

## 7. 渲染使用链路

### 7.1 素材到 Remotion 的映射

| 时间线 Source 类型 | 渲染时 Remotion 组件 | 素材读取路径 |
|-------------------|---------------------|-------------|
| `image` | `<Img src={assetPath} />` | `项目/assets/images/{uuid}.png` |
| `video` | `<Video src={assetPath} />` | `项目/assets/videos/{uuid}.mp4` |
| `audio` | `<Audio src={assetPath} />` | `项目/assets/audio/{uuid}.mp3` |
| `data` | 自定义 ChartLayer | `项目/assets/data/{uuid}.csv` |
| `font` | CSS `@font-face` | `项目/assets/fonts/{uuid}.ttf` |

### 7.2 路径解析

Generator 生成 TSX 时，将 `assetId` 解析为绝对路径：

```typescript
// generator/utils/asset-resolver.ts
function resolveAssetPath(assetId: string, projectRoot: string): string {
  const asset = db.assets.get(assetId);
  if (!asset) throw new Error(`素材不存在: ${assetId}`);
  return path.join(projectRoot, asset.path);
}

// 生成的 TSX 中
const imgSrc = resolveAssetPath('asset-uuid-001', '/Users/qiuku/Projects/Demo');
// => /Users/qiuku/Projects/Demo/assets/images/asset-uuid-001.png
```

### 7.3 素材缺失处理

渲染时发现素材文件缺失：
- **预览阶段**：显示占位图（灰色矩形 + 文件名），控制台输出警告。
- **导出阶段**：中断导出，提示"素材 {name} 缺失，请重新导入"。

## 8. 缓存与清理策略

### 8.1 缓存层级

| 层级 | 位置 | 内容 | 清理策略 |
|------|------|------|----------|
| L1 | 内存 | 最近使用的素材元数据 | 应用退出即释放 |
| L2 | `.thumbnails/` | 缩略图 | LRU，最大 500MB |
| L3 | `~/.easymotion/temp/` | 临时转码文件 | 启动时清理 24h 前的文件 |
| L4 | `项目/assets/` | 素材原文件 | 用户手动删除或项目删除时清理 |

### 8.2 自动清理任务

应用启动时执行：
1. 扫描 `~/.easymotion/temp/`，删除 24 小时前的临时文件。
2. 扫描 `.thumbnails/`，删除无对应素材的孤儿文件。
3. 如 `.thumbnails/` 总大小 > 500MB，按 LRU 清理至 400MB。

### 8.3 项目删除清理

删除项目时：
1. 删除 `library.db` 中该项目的所有素材记录。
2. 删除项目目录（含 `assets/`、`.thumbnails/`）。
3. 如删除失败（文件被占用），标记为"待清理"，下次启动时重试。

## 9. 错误处理与回滚

### 9.1 导入失败回滚

导入过程中任一环节失败时，回滚已完成的操作：

```
导入开始
    │
    ├── 复制文件失败（如磁盘满）
    │      └── 回滚：删除已复制的文件
    │
    ├── 元数据提取失败
    │      └── 回滚：删除已复制的文件 + 数据库记录（如有）
    │
    └── 缩略图生成失败
           └── 不阻塞，素材仍可用，缩略图显示占位图
```

### 9.2 错误码映射

| 错误场景 | 错误码 | 用户提示 |
|----------|--------|----------|
| 格式不支持 | `E2031` | "不支持的文件格式：{ext}" |
| 文件不存在 | `E2030` | "文件不存在或已被移动" |
| 磁盘空间不足 | `E2032` | "磁盘空间不足，请清理后重试" |
| 文件过大 | - | "文件超过大小限制（{limit}）" |
| 文件损坏 | - | "文件可能已损坏，无法导入" |
| 重复文件 | - | "素材已存在，是否添加引用？" |
| 缩略图生成失败 | `E2035` | "缩略图生成失败，素材仍可正常使用" |

## 10. 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单文件导入（< 10MB） | < 2 秒 | 从拖拽到素材库显示 |
| 单文件导入（< 100MB） | < 5 秒 | 含复制和元数据提取 |
| 单文件导入（> 500MB） | < 30 秒 | 大视频文件，显示进度条 |
| 批量导入（50 个文件） | < 30 秒 | 平均 < 0.6 秒/文件 |
| 缩略图生成（图片） | < 500ms | sharp 缩放 |
| 缩略图生成（视频首帧） | < 2 秒 | FFmpeg 截图 |
| 缩略图生成（音频波形） | < 3 秒 | FFmpeg 波形图 |
| 素材库搜索响应 | < 100ms | ≤ 1000 条素材 |
