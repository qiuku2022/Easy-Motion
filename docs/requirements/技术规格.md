# 技术规格

## Remotion 版本策略
- 基于 Remotion 官方发行版开发（如 v4.0.469）
- 不使用开发版/主分支，确保稳定性
- 版本锁定，避免自动更新导致兼容性问题
- 后续升级需经过充分测试

## 技术选型
- **GUI 框架**：Electron
- **前端 UI**：React + shadcn/ui + Tailwind CSS
- **动画引擎**：Remotion（官方发行版）
- **后端服务**：Node.js
## 数据分析模块
- **技术栈**：Python + FastAPI
- **主要职责**：
  - 网页爬取：抓取网页内容、数据、参考素材
  - 数据清洗与格式化：转换为结构化数据（JSON/CSV）
  - 本地存储：保存到项目 `data/` 目录
  - 为大模型提供结构化数据输入

## 爬虫模块
- **框架**：Playwright（模拟真实浏览器，支持动态页面渲染）
- **反爬策略**：
  - Playwright Stealth 模式（防检测）
  - 随机请求延迟（1-3 秒）
  - 常见浏览器 User-Agent 轮换
- **FastAPI 接口**：
  - `POST /crawl`：传入 URL，返回网页内容/结构化数据
  - `POST /crawl/save`：传入 URL 和目标路径，直接保存到 `data/` 目录
  - `POST /crawl/screenshot`：截图保存为图片参考
- **Agent 框架**：LangChain（用于构建多模态对话 Agent）
- **包管理器**：pnpm（全局存储节省空间，子项目依赖共享，安装速度快）

## LLM 与 Agent
- 基于 LangChain 构建对话 Agent
- 支持多模态模型（图片参考 + 文字描述）
- LLM 提供商可配置（OpenAI、Claude、本地模型等）
- Agent 负责：
  - 解析用户自然语言需求
  - 解析参考图片的视觉结构
  - 生成 Remotion 组件配置
  - 执行对话调整指令

## 通信方案
- **Python 与 Node.js 通信**：采用 Python FastAPI 本地 HTTP 服务
- Node.js 作为主进程管理 Python 子进程生命周期
- 通过 HTTP 请求调用 Python 数据分析接口

## 部署模式
- 纯本地运行，无需用户系统和云端渲染
- 利用本地算力，保护用户隐私
