## 架构原则

### 1. Store 职责单一 + 事件总线解耦
每个 Store 只管理一类状态，避免 God Object。**Store 之间禁止直接互相调用 `getState()`**，所有跨 Store 通信统一通过 `eventBus`（基于 mitt 实现的 EventEmitter）解耦：发起方发布事件，订阅方在自己的 Action 中处理（详见下文"跨 Store 通信机制"）。

### 2. 不可变更新
所有状态更新通过 Immer 处理，确保时间旅行调试和 React 重渲染优化。

### 3. 选择性持久化
- **自动持久化**：projectStore、timelineStore（写入 `project.json` / `subproject.json`）
- **会话级**：uiStore、playbackStore（重启后重置）
- **持久化+隔离**：conversationStore（每个子项目独立的 `conversation.json`）

### 4. 撤销栈独立
timelineStore 维护独立的撤销/重做栈，其他 Store 的变更不参与撤销（避免误操作）。

## Store 清单与接口定义

### 1. projectStore

**职责**：总项目与子项目的生命周期管理、当前激活上下文。

```typescript
// stores/projectStore.ts
interface ProjectState {
  // ── 项目列表 ──
  projects: ProjectSummary[];          // 最近打开的项目列表（来自 library.db）
  currentProjectId: string | null;   // 当前激活的总项目 UUID
  currentSubprojectId: string | null; // 当前激活的子项目 UUID

  // ── 加载状态 ──
  isLoading: boolean;                // 项目列表加载中
  isSaving: boolean;                 // 正在保存（防抖中显示指示器）
  lastSavedAt: number | null;         // 上次成功保存时间戳

  // ── 错误状态 ──
  error: ProjectError | null;
}

interface ProjectSummary {
  id: string;
  name: string;
  path: string;
  thumbnailPath?: string;
  resolution: string;     // 如 "1920x1080"
  fps: number;
  subprojectCount: number;
  modifiedAt: number;
  lastOpenedAt?: number;

  code: string;           // 如 'E2015'
  message: string;

}

interface ProjectActions {
  // 项目级操作
  loadProjects(): Promise<void>;                         // 从 library.db 加载项目列表
  createProject(config: ProjectConfig): Promise<string>; // 返回新项目 UUID
  openProject(projectId: string): Promise<void>;
  deleteProject(projectId: string): Promise<void>;
  renameProject(projectId: string, name: string): Promise<void>;

  // 子项目级操作
  createSubproject(name: string, template?: string): Promise<string>;
  switchSubproject(subprojectId: string): Promise<void>;
  renameSubproject(subprojectId: string, name: string): Promise<void>;
  deleteSubproject(subprojectId: string): Promise<void>;
  duplicateSubproject(subprojectId: string): Promise<string>;

  // 状态更新

  setCurrentSubproject(subprojectId: string | null): void;
  setError(error: ProjectError | null): void;
  markSaved(): void;

```

**状态流转图：**

```
          ┌─────────────┐
          │   initial   │
          │ (projects=[],
          │  current=null)
          └──────┬──────┘
                 │ loadProjects()
                 ▼
          ┌─────────────┐     createProject()
          │   loaded    │◄──────────────────┐
          │ (列表就绪)   │                     │
          └──────┬──────┘                     │
                 │ openProject()             │
                 ▼                           │
          ┌─────────────┐     deleteProject()
          │  project    │───────────────────┐
          │  opened     │                     │
          │ (current set)                     │
          └──────┬──────┘                     │
                 │ switchSubproject()          │
                 ▼                             │
          ┌─────────────┐                     │
          │ subproject  │─────────────────────┘
          │  active     │   (回到 loaded，
          └─────────────┘    保留项目列表)
```

**跨 Store 影响：**
- `openProject()` → 触发 `timelineStore.loadTimeline()`
- `switchSubproject()` → 触发 `timelineStore.loadTimeline()` + `conversationStore.loadConversation()`
- `createSubproject()` → 成功后自动 `switchSubproject()`

### 2. timelineStore

**职责**：时间线数据（轨道、片段、关键帧）、播放控制、历史栈。

```typescript
// stores/timelineStore.ts
interface TimelineState {

  // 播放控制

  pause(): void;
  seekTo(frame: number): void;
  setPlaybackSpeed(speed: number): void;
  setLoopRange(range: [number, number] | null): void;

  stepBackward(frames?: number): void;
```

**历史栈（撤销/重做）规则：**

| 操作类型 | 是否入栈 | 说明 |
|----------|----------|------|
| 添加/删除轨道 | ✅ | 完整 Timeline 快照 |
| 添加/删除/移动片段 | ✅ | 完整 Timeline 快照 |
| 修改关键帧 | ✅ | 但 500ms 防抖合并连续操作 |
| 播放控制（seek/play） | ❌ | 仅 UI 状态，不入栈 |
| 选中变更 | ❌ | 仅 UI 状态，不入栈 |
| AI 生成完成 | ✅ | 生成前后的完整快照 |

**状态流转图：**

```
         ┌──────────────┐
         │   initial    │
         │ (timeline=null)
         └──────┬───────┘
                │ loadTimeline()
                ▼
         ┌──────────────┐
         │    loaded    │◄──────────────┐
         │ (可操作)      │               │
         └──────┬───────┘               │
                │                       │
    ┌───────────┼───────────┐           │
    ▼           ▼           ▼           │
┌───────┐  ┌───────┐  ┌───────┐        │
│  add  │  │ move  │  │ delete│        │
│ track │  │ clip  │  │ keyframe      │
└───┬───┘  └───┬───┘  └───┬───┘        │
    │          │          │             │
    └──────────┴──────────┘             │
               ▼                        │
        ┌──────────────┐                │
        │ push history │                │
        │ set hasUnsavedChanges=true    │
        └──────┬───────┘                │
               │ undo()                 │
               ▼                        │
        ┌──────────────┐                │
        │   restore    │────────────────┘
        │  from past[] │   (redo() → restore from future[])
        └──────────────┘
```

**跨 Store 影响：**
- `addClip()` 等变更操作 → `hasUnsavedChanges = true`
- `saveTimeline()` → 调用 IPC `main:saveSubproject` → 成功后 `hasUnsavedChanges = false` → `projectStore.markSaved()`
- `play()` / `seekTo()` → 通知 `previewStore` 同步 iframe
- `selectClip()` → 通知 `uiStore` 更新属性面板显示

## 跨 Store 通信机制

### 核心原则

Store 之间**禁止直接互相调用** `getState()`，所有跨 Store 通信统一通过**事件总线**（`eventBus`）解耦。事件总线是一个轻量 EventEmitter（mitt / 自实现），不持有业务状态，只负责"通知"。

### 1. 事件总线（唯一推荐方式）

当一个 Store 的 Action 需要触发另一个 Store 的变更时，发布事件，订阅方在事件回调中调用自己的 Action。

**事件命名规范**：`{来源}.{动作}`（如 `timeline.addTrack`、`conversation.diffApplied`）。

**示例：Agent 生成完成后更新 Timeline**

```typescript
// eventBus.ts —— 全局事件总线
import mitt from 'mitt';
type Events = {
  // conversationStore 发布：检测到需要应用到 timeline 的 diff
  'conversation.diffReady': { subprojectId: string; diff: CodeDiff };
  // projectStore 发布：子项目切换
  'project.subprojectChanged': { subprojectId: string | null };
  // previewStore 发布：播放头变化
  'preview.frameChanged': { frame: number };
};
export const eventBus = mitt<Events>();
```

```typescript
// conversationStore.ts —— 只发布事件，不直接调用 timelineStore
applyCodeDiff: async (messageId) => {
  const diff = getMessageDiff(messageId);
  // ✅ 只发布事件，不关心谁订阅
  eventBus.emit('conversation.diffReady', {
    subprojectId: useProjectStore.getState().currentSubprojectId!,
    diff,
  });
}
```

```typescript
// timelineStore.ts —— 订阅事件，在自己的 Action 中处理
// 在 App 初始化时调用一次
useTimelineStore.subscribeToEventBus = () => {
  eventBus.on('conversation.diffReady', ({ subprojectId, diff }) => {
    // ✅ 调用自己的 Action，由 timelineStore 自身负责状态更新和持久化
    useTimelineStore.getState().addTrack(diff.newTrack);
    useTimelineStore.getState().saveTimeline();
  });
  // 订阅子项目切换，加载对应时间线
  eventBus.on('project.subprojectChanged', ({ subprojectId }) => {
    if (subprojectId) {
      useTimelineStore.getState().loadTimeline(subprojectId);
    }
  });
};
```

### 2. Zustand 原生 subscribe（补充用法）

仅用于**纯读派生**（如 UI 组件订阅某个 Store 切片后做派生计算），不允许在 subscribe 回调中调用其他 Store 的 Action。如需跨 Store 派生，请改用 `useStoreA(state => …)` + `useStoreB(state => …)` 的组合 selector。

**示例：UI 组件根据 currentSubprojectId 派生对话列表**

```typescript
// ConversationList.tsx
const messages = useConversationStore(
  (s) => s.messages[useProjectStore((s) => s.currentSubprojectId) ?? ''] ?? []
);
```

### 3. 禁止的做法

| ❌ 禁止 | ✅ 正确 |
|---------|---------|
| 在 Action 中 `useTimelineStore.getState().addTrack(...)` | 发布 `eventBus.emit('conversation.diffReady', ...)`，由 timelineStore 订阅 |
| `useTimelineStore.getState().timeline = newTimeline`（直接赋值） | `useTimelineStore.getState().addTrack(track)`（调用 Action） |
| Store A 直接修改 Store B 的 State | Store A 调自己的 Action，让 Action 通过事件总线广播 |
| 在 Action 内 `getState()` 读其他 Store 做条件分支 | 把需要的状态作为事件 payload 一并 emit，或重构为单 Store 内部状态 |

## 持久化策略

### 1. projectStore 持久化

**触发条件：**
- `createProject()` / `renameProject()` / `deleteProject()` → 立即同步 `library.db`
- `openProject()` → 更新 `last_opened_at`

**方式：** 通过 IPC `main:updateProjectIndex` → 主进程写 SQLite。

### 2. timelineStore 持久化

**触发条件：**
- 防抖 500ms：任何修改 `timeline` 的操作后 500ms 无新操作 → 触发保存
- 关键操作立即保存：删除子项目、重命名、切换子项目前
- 用户手动保存（`Ctrl+S`）→ 立即保存

**方式：**
```
timelineStore.saveTimeline()
  → IPC invoke('main:saveSubproject', { subprojectId, timelineJSON })
  → 主进程校验 JSON → 写入 subproject.json
  → 成功后 timelineStore.hasUnsavedChanges = false
```

### 3. conversationStore 持久化

**触发条件：** 每收到一条 Assistant 完整消息后自动保存（流式输出结束后）。

**方式：** 通过 IPC `main:saveConversation` → 写入 `conversation.json`。

### 4. uiStore 持久化

**触发条件：** 面板尺寸/折叠状态变化后防抖 1 秒。

**方式：** 通过 IPC `main:saveSettings` → 写入 `settings.json` 的 `ui` 字段。

## Store 与 Remotion iframe 的通信

```
┌─────────────────────────────────────────────────────────────┐
│                    Electron Renderer                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │ timelineStore│  │ previewStore │  │    UI Components    │  │
│  │ (播放控制)    │  │ (iframe 状态)│  │ (Player 容器)       │  │
│  └──────┬───────┘  └──────┬──────┘  └──────────┬──────────┘  │
│         │                 │                     │              │
│         └─────────────────┴─────────────────────┘              │
│                           │                                   │
│                    window.postMessage                          │
│                           │                                   │
└───────────────────────────┼───────────────────────────────────┘
                            │
                    ┌───────▼────────┐
                    │  iframe (Remotion│
                    │   Player)      │
                    │  - seekTo()    │
                    │  - play()      │
                    │  - pause()     │
                    └───────┬────────┘
                            │
                    window.parent.postMessage
                            │
                    ┌───────▼────────┐
                    │  帧变更事件     │
                    │  FRAME_CHANGE  │
                    └────────────────┘
```

**通信协议（iframe ↔ Renderer）：**

| 方向 | 消息类型 |  Payload | 说明 |
|------|----------|----------|------|
| Renderer → iframe | `PLAY` | - | 播放 |
| Renderer → iframe | `PAUSE` | - | 暂停 |
| Renderer → iframe | `SEEK` | `{ frame: number }` | 跳转到指定帧 |
| Renderer → iframe | `SET_SCALE` | `{ scale: number }` | 设置缩放 |
| iframe → Renderer | `FRAME_CHANGE` | `{ frame: number }` | 播放头位置变化 |
| iframe → Renderer | `READY` | - | Player 加载完成 |
| iframe → Renderer | `ERROR` | `{ message: string }` | 渲染错误 |
| iframe → Renderer | `FPS_REPORT` | `{ fps: number }` | 实时帧率上报 |

## 性能优化规则

1. **批量更新**：Generator 生成多个轨道时，使用单个 `setState` 批量写入，而非逐条 `addTrack`。
2. **选择器订阅**：组件只订阅需要的字段，避免无关状态变更触发重渲染。
   ```typescript
   // ✅ 只订阅 currentFrame，不订阅整个 timeline
   const currentFrame = useTimelineStore((state) => state.currentFrame);
   ```
3. **防抖持久化**：写盘操作统一防抖，避免频繁 IO。
4. **历史栈截断**：`past[]` 超过 50 步时，自动丢弃最早的记录。
