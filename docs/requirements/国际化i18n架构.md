# 国际化（i18n）架构

> **实施状态（2026-06-30）**：**M0–M9 ✅**（Windows MVP 预发行 v0.1.2）；后续以功能增强为主。权威入口：仓库根 [README.md](../../README.md)、[开发里程碑与路线图.md](./开发里程碑与路线图.md)。

## 概述

本文档定义 EasyMotion 的国际化架构预留方案。当前版本以中文为核心界面语言，但架构上提前预留多语言扩展能力，避免后期重构成本。

> **版本**：v0.1  
> **最后更新**：2026-06-26  
> **关联文档**：UI布局与交互设计-优化版.md、项目编码规范.md、组件库清单.md

## 1. 设计原则

### 1.1 当前策略

- **v1.0**：仅支持简体中文界面。
- **架构预留**：代码层面提前接入 i18n 框架，文案统一从翻译文件读取。
- **后续扩展**：v2.0 根据用户反馈决定是否添加英文等其他语言。

### 1.2 为什么要提前预留

| 风险 | 说明 |
|------|------|
| 硬编码文案 | 后期替换需要遍历所有组件，容易遗漏 |
| 文案分散 | UI 文案、错误消息、提示文案分散在不同文件 |
| 复数规则 | 中文无复数变化，但英语/俄语等有，需提前设计 |
| 布局适配 | 英文通常比中文长 30-50%，需预留弹性布局 |

## 2. 技术选型

| 方案 | 库 | 说明 |
|------|-----|------|
| **推荐** | `react-i18next` + `i18next` | 业界标准，支持 ICU 格式、复数、插值、嵌套 |
| 备选 | `lingui` | 编译时提取，Tree-shaking 友好 |
| 备选 | `formatjs`（React Intl） | Facebook 出品，支持 ICU MessageFormat |

**选择 `react-i18next` 的原因：**
- 与 React 生态无缝集成（Hook + HOC）。
- 支持 TypeScript 类型安全（通过 `i18next-resources-to-json` 生成类型定义）。
- 支持延迟加载语言包（按路由/组件懒加载）。
- Electron 主进程可通过 `i18next-node-fs-backend` 共享同一套翻译文件。

## 3. 翻译文件组织

### 3.1 目录结构

```
public/locales/                    # 翻译文件目录
├── zh-CN/                         # 简体中文（默认）
│   ├── common.json                # 通用文案（按钮、菜单、状态）
│   ├── timeline.json              # 时间线相关
│   ├── preview.json               # 预览相关
│   ├── conversation.json          # 对话面板
│   ├── project.json               # 项目管理
│   ├── asset.json                 # 素材库
│   ├── settings.json              # 设置面板
│   ├── errors.json                # 错误消息
│   └── shortcuts.json             # 快捷键标签
└── en/                            # 英语（预留）
    └── ...
```

### 3.2 命名空间（Namespace）

按功能模块拆分命名空间，避免加载不必要的翻译内容：

| 命名空间 | 内容 | 预估键数 |
|----------|------|----------|
| `common` | 按钮（确定/取消/保存）、通用状态（加载中/完成/失败） | 100+ |
| `timeline` | 轨道操作、片段操作、播放控制、关键帧 | 80+ |
| `preview` | 缩放、全屏、质量设置 | 30+ |
| `conversation` | 对话提示、Agent 状态、消息操作 | 60+ |
| `project` | 新建/打开/保存、子项目管理 | 50+ |
| `asset` | 导入、搜索、类型筛选、素材信息 | 40+ |
| `settings` | 各项设置标签和说明 | 60+ |
| `errors` | 所有错误码对应的用户友好消息 | 50+ |
| `shortcuts` | 快捷键名称和描述 | 40+ |

## 4. 代码集成规范

### 4.1 初始化配置

```typescript
// src/i18n/index.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
// ⚠️ Electron 渲染进程运行在 file:// 协议下，HTTP 请求被 CSP 阻止。
// 必须用 i18next-fs-backend 通过主进程 fs 读取本地翻译文件，
// 或者更简单：在打包时通过 Vite 的 import.meta.glob 把翻译文件内联进 bundle。
import Backend from 'i18next-fs-backend';

i18n
  .use(Backend)
  .use(initReactI18next)
  .init({
    lng: 'zh-CN',                    // 默认语言
    fallbackLng: 'zh-CN',
    ns: ['common', 'timeline', 'preview', 'conversation', 'project', 'asset', 'settings', 'errors', 'shortcuts'],
    defaultNS: 'common',
    backend: {
      // i18next-fs-backend 通过 preload 注入的 IPC 桥从主进程读取
      loadPath: '/locales/{{lng}}/{{ns}}.json',
    },
    interpolation: {
      escapeValue: false,             // React 已自动转义
    },
    react: {
      useSuspense: false,             // 避免初始渲染闪烁；Suspense 在 Electron 渲染进程可用但与 SSR-不一致
    },
  });

export default i18n;
```

### 4.2 组件中使用

```tsx
// ✅ 正确：使用 t() 函数
import { useTranslation } from 'react-i18next';

function TrackHeader({ name }: { name: string }) {
  const { t } = useTranslation('timeline');
  
  return (
    <div>
      <span>{t('track.name', { name })}</span>
      <button>{t('track.lock')}</button>
      <button>{t('track.delete')}</button>
    </div>
  );
}

// ✅ 正确：使用 Trans 组件处理嵌套
import { Trans } from 'react-i18next';

function WelcomeMessage() {
  return (
    <Trans i18nKey="welcome.message">
      欢迎使用 <strong>EasyMotion</strong>，开始创建动画吧！
    </Trans>
  );
}
```

### 4.3 禁止的做法

| ❌ 禁止 | ✅ 正确 |
|---------|---------|
| 硬编码中文：`return <span>播放</span>` | `return <span>{t('play')}</span>` |
| 拼接字符串：`时间线 + {name}` | `t('timeline.title', { name })` |
| 条件文本：`isLoading ? '加载中' : '完成'` | `t(isLoading ? 'status.loading' : 'status.done')` |
| 无命名空间：`useTranslation()` | `useTranslation('timeline')` |

## 5. 文案管理规范

### 5.1 键命名规则

```
[模块].[组件].[元素].[状态]
```

**示例：**

```json
{
  "track": {
    "name": "{{name}} 轨道",
    "lock": "锁定轨道",
    "unlock": "解锁轨道",
    "delete": "删除轨道",
    "deleteConfirm": "确定要删除轨道 "{{name}}" 吗？此操作不可撤销。"
  },
  "clip": {
    "move": "移动片段",
    "resize": "调整时长",
    "split": "在 {{frame}} 帧分割",
    "copy": "复制片段",
    "paste": "粘贴片段"
  },
  "playback": {
    "play": "播放",
    "pause": "暂停",
    "seekTo": "跳转到 {{frame}} 帧",
    "speed": "{{value}} 倍速"
  }
}
```

### 5.2 插值与复数

```json
{
  "asset": {
    "importProgress": "正在导入 {{count}} 个文件，已完成 {{percentage}}%",
    "importCompleted": {
      "one": "成功导入 1 个文件",
      "other": "成功导入 {{count}} 个文件"
    }
  }
}
```

```tsx
// 组件中使用复数
t('asset.importCompleted', { count: importedCount });
```

### 5.3 日期/时间/数字格式化

```tsx
import { useTranslation } from 'react-i18next';

function ExportInfo({ duration, fileSize, completedAt }: ExportInfoProps) {
  const { t, i18n } = useTranslation('common');
  
  // 使用 Intl API 格式化
  const formattedDuration = new Intl.NumberFormat(i18n.language).format(duration);
  const formattedSize = new Intl.NumberFormat(i18n.language, {
    style: 'unit',
    unit: 'megabyte',
  }).format(fileSize / 1024 / 1024);
  
  const formattedDate = new Intl.DateTimeFormat(i18n.language, {
    dateStyle: 'medium',
    timeStyle: 'short',
  }).format(new Date(completedAt));
  
  return (
    <div>
      <p>{t('export.duration', { value: formattedDuration })}</p>
      <p>{t('export.fileSize', { value: formattedSize })}</p>
      <p>{t('export.completedAt', { value: formattedDate })}</p>
    </div>
  );
}
```

## 6. Electron 主进程国际化

### 6.1 共享翻译文件

主进程（Node.js）与渲染进程共享同一套翻译文件：

```typescript
// main/utils/i18n.ts
import i18n from 'i18next';
import Backend from 'i18next-fs-backend';

i18n.use(Backend).init({
  lng: 'zh-CN',
  fallbackLng: 'zh-CN',
  ns: ['common', 'errors'],
  defaultNS: 'common',
  backend: {
    loadPath: path.join(__dirname, '../../locales/{{lng}}/{{ns}}.json'),
  },
});

export function t(key: string, options?: Record<string, unknown>): string {
  return i18n.t(key, options);
}
```

### 6.2 主进程中的使用场景

- 系统通知（macOS/Windows 原生通知）
- 对话框标题和按钮（`dialog.showMessageBox`）
- 日志和错误消息
- 菜单标签（`Menu.buildFromTemplate`）

## 7. 语言切换

### 7.1 运行时切换

```tsx
// settings/GeneralSettings.tsx
import { useTranslation } from 'react-i18next';

function GeneralSettings() {
  const { i18n } = useTranslation();
  
  const changeLanguage = (lng: string) => {
    i18n.changeLanguage(lng);
    // 保存到 settings.json
    window.electron.invoke('main:settings:update', { language: lng });
  };
  
  return (
    <Select value={i18n.language} onValueChange={changeLanguage}>
      <SelectItem value="zh-CN">简体中文</SelectItem>
      <SelectItem value="en">English (预留)</SelectItem>
    </Select>
  );
}
```

### 7.2 启动时加载

```typescript
// main/index.ts
import { app } from 'electron';

async function initializeLanguage() {
  const settings = await loadSettings();
  const systemLang = app.getLocale();           // 如 'zh-CN', 'en-US'
  const lang = settings.language || systemLang;
  
  // 如果系统语言不在支持列表，fallback 到 zh-CN
  const supportedLangs = ['zh-CN', 'en'];
  const finalLang = supportedLangs.includes(lang) ? lang : 'zh-CN';
  
  await i18n.changeLanguage(finalLang);
}
```

## 8. 翻译工作流

### 8.1 提取文案

使用 `i18next-parser` 自动从代码中提取翻译键：

```bash
# package.json
"scripts": {
  "i18n:extract": "i18next-parser --config i18next-parser.config.js"
}
```

```javascript
// i18next-parser.config.js
module.exports = {
  locales: ['zh-CN', 'en'],
  output: 'public/locales/$LOCALE/$NAMESPACE.json',
  input: ['src/**/*.{ts,tsx}'],
  keySeparator: '.',
  namespaceSeparator: ':',
  defaultNamespace: 'common',
};
```

### 8.2 翻译维护

| 阶段 | 工作 | 负责人 |
|------|------|--------|
| 开发 | 代码中使用 `t('key')`，填写中文默认值 | 开发者 |
| 提取 | 运行 `pnpm i18n:extract`，更新 JSON 文件 | CI |
| 翻译 | 专业翻译人员填写英文（或其他语言） | 翻译者 |
| 校对 | 检查翻译完整性、上下文准确性 | 校对者 |
| 发布 | 翻译文件随应用打包 | CI |

## 9. 布局适配

### 9.1 文本长度差异

| 语言 | 相对中文长度 | 注意事项 |
|------|-------------|----------|
| 中文 | 1× | 基准 |
| 英语 | 1.3-1.5× | 按钮文字可能溢出 |
| 德语 | 1.5-2× | 需要更大弹性空间 |
| 俄语 | 1.3-1.6× | |

### 9.2 布局弹性设计

```css
/* 按钮最小宽度，允许文字撑开 */
.btn-i18n {
  min-width: 80px;
  padding: 0 16px;
  white-space: nowrap;
}

/* 标签右对齐，文字长度变化不影响布局 */
.form-label {
  width: 120px;
  text-align: right;
  flex-shrink: 0;
}

/* 标题截断 */
.panel-title {
  max-width: 200px;
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

### 9.3 图标+文字组合

所有操作按钮使用「图标 + 文字」或「纯图标 + Tooltip」：

```tsx
// 图标+文字（主操作）
<Button>
  <PlayIcon className="w-4 h-4" />
  <span>{t('playback.play')}</span>
</Button>

// 纯图标 + Tooltip（工具栏节省空间）
<Tooltip content={t('playback.play')}>
  <IconButton icon={<PlayIcon />} />
</Tooltip>
```
