# 构建与部署

## 概述

本文档定义 EasyMotion 的构建配置、打包策略、分发渠道和自动更新机制，确保应用可以高效、可靠地交付给最终用户。

---

## 技术选型

| 组件 | 工具 | 说明 |
|------|------|------|
| Electron 打包 | `electron-builder` | 支持多平台打包、签名、自动更新 |
| 前端构建 | Vite | 渲染进程开发服务器和打包 |
| 包管理 | pnpm + workspace | monorepo 依赖管理 |
| 自动更新 | `electron-updater` | 基于 electron-builder 的自动更新 |
| 分发渠道 | GitHub Releases + 官网下载 | 主渠道；可选国内镜像 |
| 安装包格式 | macOS: `.dmg` / Windows: `.exe` (NSIS) / Linux: `.AppImage` | 主流格式 |

---

## 目录构建流程

```
开发源码
  ↓
[pnpm install] 安装依赖
  ↓
[构建渲染进程] Vite build
  ├── 输出：apps/electron/dist/renderer/
  ├── 包含：React 应用打包产物（HTML、CSS、JS）
  └── 环境：production 模式，代码压缩
  ↓
[构建 Remotion 模板]
  ├── 输出：resources/templates/default-project/remotion/dist/
  └── 包含：默认项目模板的编译产物
  ↓
[构建 Python 模块]
  ├── 输出：resources/python/
  └── 包含：Python 可执行文件 + 依赖库
  ↓
[复制静态资源]
  ├── resources/templates/ → 打包目录
  ├── resources/icons/ → 打包目录
  └── 其他静态文件
  ↓
[electron-builder 打包]
  ├── 平台：macOS / Windows / Linux
  ├── 架构：x64 / arm64（macOS）
  └── 输出：安装包文件
```

## electron-builder 配置

### 基础配置

```yaml
# electron-builder.yml
appId: com.easymotion.app
productName: EasyMotion
copyright: "Copyright © 2026 EasyMotion Team"

# 输出目录
directories:
  output: dist
  buildResources: build

# 渲染进程打包产物
files:
  - "dist/renderer/**/*"
  - "dist/main/**/*"
  - "resources/**/*"
  - "package.json"

# 额外资源（不打包进 asar，保持可读写）
extraResources:
  - from: "resources/templates"
    to: "templates"
    filter: "**/*"
  - from: "resources/python"
    to: "python"
    filter: "**/*"

# 排除开发依赖
asarUnpack:
  - "resources/python/**/*"
  - "resources/templates/**/*"

# 自动更新配置
publish:
  provider: github
  owner: EasyMotion
  repo: EasyMotion
  releaseType: release

# macOS 配置
mac:
  category: public.app-category.video
  target:
    - target: dmg
      arch:
        - x64
        - arm64
    - target: zip
      arch:
        - x64
        - arm64
  icon: build/icon.icns
  hardenedRuntime: true
  gatekeeperAssess: false
  entitlements: build/entitlements.mac.plist
  entitlementsInherit: build/entitlements.mac.plist

# Windows 配置
win:
  target:
    - target: nsis
      arch:
        - x64
  icon: build/icon.ico
  publisherName: "EasyMotion Team"
  certificateFile: ""  # 代码签名证书路径（CI 环境注入）
  certificatePassword: ""  # 证书密码（CI 环境注入）

# Linux 配置
linux:
  target:
    - target: AppImage
      arch:
        - x64
  icon: build/icon.png
  category: AudioVideo

# NSIS 安装程序配置
nsis:
  oneClick: false
  allowToChangeInstallationDirectory: true
  installerIcon: build/icon.ico
  uninstallerIcon: build/icon.ico
  installerSidebar: build/installerSidebar.bmp
  createDesktopShortcut: always
  createStartMenuShortcut: true
```

### 签名配置

| 平台 | 签名方式 | 说明 |
|------|----------|------|
| macOS | Apple Developer ID + Notarization | `.dmg` 和 `.zip` 必须签名和公证，否则 Gatekeeper 阻止 |
| Windows | 代码签名证书（OV/EV） | EV 证书可消除 SmartScreen 警告；OV 证书需积累信誉 |
| Linux | 无需签名 | AppImage 自带校验 |

**CI 环境变量：**
```
# macOS
APPLE_ID=xxx@example.com
APPLE_APP_SPECIFIC_PASSWORD=xxxx-xxxx-xxxx-xxxx
APPLE_TEAM_ID=XXXXXXXXXX
CSC_LINK=base64-encoded-cert
CSC_KEY_PASSWORD=password

# Windows
WIN_CSC_LINK=base64-encoded-cert
WIN_CSC_KEY_PASSWORD=password
```

## 多平台构建策略

### 本地构建（开发/测试）

```bash
# 当前平台打包（不签名，用于测试）
pnpm build:local

# 输出：
# macOS: dist/EasyMotion-0.1.0-mac.dmg
# Windows: dist/EasyMotion Setup 0.1.0.exe
# Linux: dist/EasyMotion-0.1.0.AppImage
```

### CI 构建（发布）

```yaml
# .github/workflows/release.yml（GitHub Actions 示例）
name: Build and Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    strategy:
      matrix:
        os: [macos-latest, windows-latest, ubuntu-latest]
    runs-on: ${{ matrix.os }}
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          
      - name: Setup pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 9
          
      - name: Install dependencies
        run: pnpm install
        
      - name: Build renderer
        run: pnpm build:renderer
        
      - name: Build Python module
        run: pnpm build:python
        
      - name: Build and sign (macOS)
        if: matrix.os == 'macos-latest'
        run: pnpm build:mac
        env:
          APPLE_ID: ${{ secrets.APPLE_ID }}
          APPLE_APP_SPECIFIC_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
          APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
          CSC_LINK: ${{ secrets.MAC_CSC_LINK }}
          CSC_KEY_PASSWORD: ${{ secrets.MAC_CSC_PASSWORD }}
          
      - name: Build and sign (Windows)
        if: matrix.os == 'windows-latest'
        run: pnpm build:win
        env:
          WIN_CSC_LINK: ${{ secrets.WIN_CSC_LINK }}
          WIN_CSC_KEY_PASSWORD: ${{ secrets.WIN_CSC_PASSWORD }}
          
      - name: Build (Linux)
        if: matrix.os == 'ubuntu-latest'
        run: pnpm build:linux
        
      - name: Upload artifacts
        uses: actions/upload-artifact@v4
        with:
          name: dist-${{ matrix.os }}
          path: dist/
          
  release:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Download all artifacts
        uses: actions/download-artifact@v4
        
      - name: Create Release
        uses: softprops/action-gh-release@v1
        with:
          files: dist-*/**
          draft: true
          prerelease: false
```

## 安装包体积控制

### 当前体积预估

| 组件 | 体积（预估） | 优化策略 |
|------|-------------|----------|
| Electron 运行时 | ~180MB | 不可优化 |
| Chromium | 包含在 Electron 中 | 不可优化 |
| React + UI 库 | ~5MB（打包后） | Tree shaking、代码分割 |
| Remotion 运行时 | ~20MB | 按需导入组件 |
| Python + Playwright | ~150MB | 精简依赖、分离可选组件 |
| FFmpeg | ~80MB | Remotion 自带，不可优化 |
| 模板和资源 | ~10MB | 压缩图片、精简模板 |
| **总计（未压缩）** | **~450MB** | |
| **安装包目标** | **< 300MB** | 压缩、拆分 |

### 体积优化策略

1. **Electron 瘦身**
   - 使用 `electron-builder` 的 `files` 配置精确控制打包内容
   - 排除开发依赖、测试文件、源码映射（`.map`）
   - `asar` 压缩：启用 `asar: true` 压缩应用代码

2. **Python 模块优化**
   - 使用 `pyinstaller --onefile` 打包为单可执行文件
   - 移除 Playwright 不需要的浏览器二进制（Electron 自带 Chromium）
   - 只安装必要的 Python 依赖（`requirements.txt` 精简）
   - **可选拆分**：将 Python 模块作为首次启动时的下载组件（而非打包进安装包）

3. **资源文件优化**
   - 模板图片使用 WebP 格式（替代 PNG/JPG）
   - 默认项目模板只保留最小必要文件
   - 大体积素材（视频、音频）不在安装包中，首次使用时下载

4. **增量更新**
   - 自动更新使用增量补丁（`electron-updater` 的 differential download）
   - 只下载变更部分，减少更新体积

## 自动更新机制

### 更新策略

| 类型 | 频率 | 用户行为 |
|------|------|----------|
| 稳定版 | 每月发布 | 用户手动确认更新（默认） |
| 自动更新 | 可选开启 | 后台静默下载，下次启动时安装 |
| Beta 通道 | 每两周 | 需在设置中手动开启 |

### 更新流程

```
应用启动
  ↓
[检查更新]（后台异步，不阻塞启动）
  ├── 调用 autoUpdater.checkForUpdates()
  ├── 查询 GitHub Releases API
  └── 比较当前版本 vs 最新版本
        ↓
  有更新 → 显示"发现新版本"提示
    ├── 用户点击"下载" → 后台下载更新包
    ├── 下载完成 → 提示"重启安装更新"
    └── 用户确认 → 调用 autoUpdater.quitAndInstall()
        ↓
  无更新 → 静默继续，不打扰用户
```

### 配置

```typescript
// main/index.ts
import { autoUpdater } from "electron-updater";

// 设置更新服务器
autoUpdater.setFeedURL({
  provider: "github",
  owner: "EasyMotion",
  repo: "EasyMotion",
});

// 检查更新间隔（每 4 小时）
const CHECK_INTERVAL = 4 * 60 * 60 * 1000;

// 自动下载（可配置）
autoUpdater.autoDownload = false; // 默认手动确认

// 事件监听
autoUpdater.on("checking-for-update", () => {
  console.log("Checking for update...");
});

autoUpdater.on("update-available", (info) => {
  // 通知渲染进程显示更新提示
  mainWindow?.webContents.send("renderer:update:available", info);
});

autoUpdater.on("update-not-available", () => {
  // 无更新，静默
});

autoUpdater.on("download-progress", (progress) => {
  // 通知渲染进程更新下载进度
  mainWindow?.webContents.send("renderer:update:progress", progress);
});

autoUpdater.on("update-downloaded", (info) => {
  // 下载完成，提示重启
  mainWindow?.webContents.send("renderer:update:downloaded", info);
});

autoUpdater.on("error", (err) => {
  console.error("Update error:", err);
});

// 启动后延迟检查（避免启动时网络竞争）
setTimeout(() => {
  autoUpdater.checkForUpdates();
}, 10000);

// 定时检查
setInterval(() => {
  autoUpdater.checkForUpdates();
}, CHECK_INTERVAL);
```

### 版本兼容性

| 场景 | 处理策略 |
|------|----------|
| 大版本升级（v1.x → v2.x） | 提示"此更新包含重大变更，建议查看更新日志"，可选跳过 |
| 项目格式变更 | 更新包附带迁移脚本，首次打开旧项目时自动执行 |
| 强制更新 | 仅用于安全补丁，设置 `force: true`，不更新则无法使用 |

## 首次启动体验

### 启动流程

```
用户首次运行 EasyMotion
  ↓
[欢迎窗口]（可选，可跳过）
  ├── 产品介绍（1-2 页幻灯片）
  ├── 设置 LLM API Key（可跳过，后续再设置）
  └── 选择默认项目路径
        ↓
[初始化]
  ├── 创建 ~/.easymotion/ 目录结构
  ├── 初始化 SQLite 数据库
  ├── 检查 Python 环境（如未打包则下载）
  └── 复制默认项目模板到 resources/
        ↓
[进入项目窗口]
```

### 离线运行能力

| 组件 | 离线可用性 | 说明 |
|------|-----------|------|
| 核心功能 | ✅ 完全可用 | 新建/编辑/预览项目无需网络 |
| AI 生成 | ❌ 需要网络 | LLM API 调用依赖网络 |
| 素材导入 | ✅ 完全可用 | 本地文件导入 |
| 导出视频 | ✅ 完全可用 | 本地渲染 |
| 导出工程 | ✅ 完全可用 | 本地打包 |
| 检查更新 | ❌ 需要网络 | 可选功能 |
| 爬虫 | ❌ 需要网络 | 网页抓取依赖网络 |

## 分发渠道

### 主要渠道

| 渠道 | 形式 | 受众 |
|------|------|------|
| GitHub Releases | `.dmg`、`.exe`、`.AppImage` | 技术用户、开发者 |
| 官网下载页 | 上述安装包 + 更新日志 | 普通用户 |
| Homebrew（macOS） | `brew install --cask easymotion` | macOS 开发者 |
| winget（Windows） | `winget install EasyMotion` | Windows 用户 |

### 国内镜像（可选）

针对国内网络环境，提供备用下载：
- 阿里云 OSS / 腾讯云 COS 存储安装包
- 更新服务器配置国内 CDN 地址
- Python 依赖使用国内 PyPI 镜像
