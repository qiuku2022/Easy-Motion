## TypeScript 规范

### 严格模式

```json
// tsconfig.json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "strictBindCallApply": true,
    "strictPropertyInitialization": true,
    "noImplicitThis": true,
    "alwaysStrict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true
  }
}
```

**核心规则：**
- **必须开启 `strict: true`**，不允许任何类型放宽
- **禁止 `any`**：所有变量、参数、返回值必须有明确类型
- **禁止 `@ts-ignore` 和 `@ts-expect-error`**：遇到类型错误时修复根源，而不是压制
- **接口优于类型别名**：优先使用 `interface` 定义对象结构，仅在需要联合类型时使用 `type`

### 命名规范

| 类型 | 命名法 | 示例 | 说明 |
|------|--------|------|------|
| 组件 | PascalCase | `TextLayer.tsx` | React 组件文件名和组件名 |
| Props 接口 | 组件名 + Props | `TextLayerProps` | 组件的 Props 类型定义 |
| 接口 | PascalCase | `TimelineConfig` | 数据结构定义 |
| 类型别名 | PascalCase | `AnimationType` | 联合类型、工具类型 |
| 枚举 | PascalCase + 大写键 | `enum TrackType { Text = 'text' }` | 枚举名和键 |
| 变量/函数 | camelCase | `currentFrame`, `calculateOpacity` | 普通变量和函数 |
| 常量 | UPPER_SNAKE_CASE | `DEFAULT_FPS`, `MAX_TRACKS` | 真正的常量（编译期确定）|
| 布尔变量 | is/has/should + 名词 | `isPlaying`, `hasKeyframes` | 明确语义 |
| 私有属性 | _ + camelCase | `_internalState` | 仅在类内部使用 |
| 文件命名 | 同类型规则 | `projectService.ts` | 工具函数/模块 |

### 状态管理规范（Zustand）

```typescript
// ✅ 正确：Store 文件结构
// stores/timelineStore.ts
import { create } from 'zustand';
import { immer } from 'zustand/middleware/immer';

interface TimelineState {
  tracks: Track[];
  selectedClipId: string | null;
  isPlaying: boolean;
  currentFrame: number;
}

interface TimelineActions {
  addTrack: (track: Track) => void;
  removeTrack: (trackId: string) => void;
  moveClip: (clipId: string, newStart: number) => void;
  selectClip: (clipId: string | null) => void;
  seekTo: (frame: number) => void;
}

export const useTimelineStore = create<TimelineState & TimelineActions>()(
  immer((set) => ({
    // 状态
    tracks: [],
    selectedClipId: null,
    isPlaying: false,
    currentFrame: 0,
    
    // Actions
    addTrack: (track) =>
      set((state) => {
        state.tracks.push(track);
      }),
    
    removeTrack: (trackId) =>
      set((state) => {
        state.tracks = state.tracks.filter((t) => t.id !== trackId);
      }),
    
    moveClip: (clipId, newStart) =>
      set((state) => {
        const track = state.tracks.find((t) =>
          t.clips.some((c) => c.id === clipId)
        );
        if (track) {
          const clip = track.clips.find((c) => c.id === clipId);
          if (clip) {
            clip.startInFrames = newStart;
          }
        }
      }),
    
    selectClip: (clipId) =>
      set((state) => {
        state.selectedClipId = clipId;
      }),
    
    seekTo: (frame) =>
      set((state) => {
        state.currentFrame = frame;
      }),
  }))
);

// ✅ 正确：在组件中使用 Store
function Timeline() {
  const { tracks, selectedClipId, moveClip } = useTimelineStore();
  // ...
}

// ❌ 错误：直接修改 Store 状态
timelineStore.tracks.push(newTrack); // ❌

// ❌ 错误：在组件外调用 set
timelineStore.getState().addTrack(track); // ❌ 应在组件内通过 hook 调用
```

## Git 提交规范

### 分支策略

| 分支 | 说明 | 保护规则 |
|------|------|----------|
| `main` | 生产分支，只接受 PR | 必须 Code Review，CI 通过 |
| `develop` | 开发分支，集成各功能 | 必须 CI 通过 |
| `feature/*` | 功能分支 | 无保护，开发完成后合并到 develop |
| `hotfix/*` | 紧急修复 | 从 main 切出，修复后合并到 main 和 develop |
| `release/*` | 发布分支 | 从 develop 切出，测试通过后合并到 main |

### Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type：**
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具/依赖更新

**Scope：**
- `main`: 主进程
- `renderer`: 渲染进程
- `generator`: 代码生成器
- `python`: Python 模块
- `docs`: 文档

**示例：**
```
feat(timeline): 实现片段拖拽调整位置

- 支持水平拖拽改变 startInFrames
- 自动吸附到片段边界
- 拖拽时显示时间提示

Closes #123
```
