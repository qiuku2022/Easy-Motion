### 1.3 组件主题覆盖

主题 token 定义在 `apps/electron/src/renderer/src/index.css`（oklch + Cinema Dark / Action Red），`tailwind.config.js` 将语义色与 `em-*` 别名映射到同一组 CSS 变量。`html` 固定 `class="dark"`（v1.0 仅暗色）。

**滚动条工具类**（`index.css`）：

| 类名 | 用途 |
|------|------|
| `scrollbar-theme` | 细窄半透明原生滚动条（预设库、属性栏、关键帧轨道等） |
| `scrollbar-hidden` | 可滚动但隐藏滚动条（时间线主体） |
| `scrollbar-shadcn` | 隐藏滚动条、保留滚轮（对话 Markdown 等） |

面板内推荐结构：外层 `p-3` 留白 + 内层 `scrollbar-theme -mr-1 overflow-y-auto pr-1`，避免滚动条贴边被 `overflow-hidden` 裁切。

## 4. 组件开发规范

### 4.1 目录结构

```
src/components/
├── layout/           # 布局组件
├── timeline/         # 时间线组件
├── preview/          # 预览组件
├── conversation/     # 对话组件
├── property-panel/   # 属性面板组件
└── common/           # 通用组件

每个组件目录：
├── ComponentName.tsx       # 组件实现
├── ComponentName.test.tsx  # 单元测试
├── index.ts                # 导出
└── types.ts                # 组件专属类型（可选）
```

### 4.2 Props 命名规范

```typescript
interface ComponentNameProps {
  // 必填在前，选填在后
  requiredProp: string;
  optionalProp?: number;
  
  // 回调以 on 开头
  onClick?: () => void;
  onChange?: (value: string) => void;
  
  // 子组件传递以 render 开头
  renderItem?: (item: Item) => React.ReactNode;
  
  // 样式覆盖
  className?: string;
}
```

### 4.3 可访问性要求

每个组件必须：
- 可聚焦元素有 `aria-label` 或可见文本。
- 支持 `ref` 转发（使用 `React.forwardRef`）。
- 颜色信息有图标/文字冗余。
- 动画支持 `prefers-reduced-motion`。

### 4.4 性能要求

- 列表组件使用 `React.memo` + `useMemo` 避免不必要重渲染。
- 滚动区域使用虚拟化（`react-window`）当项目数 > 100。
- 避免在 `render` 中创建新对象/函数。

## 5. 图标库

使用 **Lucide React** 作为图标库，统一尺寸：

| 场景 | 尺寸 | 示例 |
|------|------|------|
| 工具栏图标 | 16px (`w-4 h-4`) | 播放、暂停、撤销 |
| 按钮图标 | 16-20px | 发送、添加、删除 |
| 面板图标 | 20px (`w-5 h-5`) | 轨道类型、素材类型 |
| 空状态图标 | 48px (`w-12 h-12`) | 空素材库、无项目 |
| 菜单图标 | 16px | 文件、编辑菜单项 |

**禁止**：使用 Emoji 作为 UI 图标。

## 6. 动画组件

| 组件 | 库 | 用途 |
|------|-----|------|
| 面板展开/收起 | CSS `transition` | 左/右/底部面板 |
| 弹窗出现/消失 | `framer-motion` | Dialog、Sheet |
| 列表项进入 | `framer-motion` | 消息列表、素材列表 |
| 拖拽反馈 | `@dnd-kit` | 时间线片段拖拽、素材拖拽 |
| 播放头移动 | 无动画（实时） | 精确到帧 |
| 进度条 | CSS `transition` | 导入、导出、生成 |
