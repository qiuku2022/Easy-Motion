# 错误码统一字典

> **实施状态（2026-06-26）**：M0–M8 已落地，M9（E2E/打包/性能）进行中。权威入口：仓库根 [README.md](../../README.md)、[开发里程碑与路线图.md](./开发里程碑与路线图.md)。

## 概述

本文档定义 EasyMotion 项目的统一错误码体系，覆盖 Node.js 主进程、渲染进程、Python FastAPI 和 LLM/Agent 四层。所有错误码按层次和模块分组，确保跨模块调用时错误可追踪、可处理。

---

## 错误码结构

```
E[层级][模块][序号]

层级：
  1 = Python 服务层
  2 = Node.js 主进程业务层
  3 = 渲染进程 UI 层
  4 = LLM/Agent 层
  9 = 未知/系统层

模块：
  00 = 通用/基础设施
  01 = 项目服务
  02 = 文件服务
  03 = 素材服务
  04 = 时间线/Generator
  05 = 预览/Remotion
  06 = 导出/渲染
  07 = 对话/Agent
  08 = LLM 服务
  09 = Python 爬虫
  10 = 配置/设置
  99 = 未知模块

序号：000-999（模块内顺序编号）
```

**示例：** `E2015` = Node.js 主进程(E2) → 项目服务(01) → 第15号错误（项目已存在）

## 错误响应格式

### Node.js 主进程 → 渲染进程

```typescript
interface IPCErrorResponse {
  success: false;
  error: {
    code: string;           // 如 "E2101"
    message: string;        // 用户友好的错误描述
    details?: Record<string, unknown>;  // 技术细节（调试用）
  };
}

// 示例
{
  success: false,
  error: {
    code: "E2101",
    message: "目标路径已存在同名文件夹",
    details: {
      path: "/Users/xxx/EasyMotion/我的项目",
      suggestion: "请选择其他路径或打开现有项目"
    }
  }
}
```

### Python → Node.js

```json
{
  "success": false,
  "error": {
    "code": "E1102",
    "message": "页面加载失败",
    "details": {
      "url": "https://example.com",
      "statusCode": 500,
      "responseTime": 30000
    }
  }
}
```

## 错误处理最佳实践

### 1. 错误码分配规则

- **新增错误码时**：在对应模块的编号范围内选择下一个可用序号
- **错误码不可复用**：一旦分配，永久保留（即使该错误场景不再存在）
- **模块内预留**：每个模块预留 20% 的序号空间（如 E2100-E2199 用完后，可扩展到 E2100-E2299）

### 2. 错误上报与日志

```typescript
// 日志格式
[2026-05-30T14:23:45.123Z] [ERROR] [project-service] E2101: 目标路径已存在同名文件夹 {"path":"/Users/xxx/..."}

// 上报内容（脱敏后）
{
  "errorCode": "E2101",
  "module": "project-service",
  "appVersion": "1.0.0",
  "os": "macOS 14.5",
  "timestamp": "2026-05-30T14:23:45.123Z"
  // 不包含用户路径、文件名、API Key 等敏感信息
}
```

### 3. 用户提示策略

| 错误层级 | 用户提示方式 | 示例 |
|----------|-------------|------|
| 可恢复 | Toast 自动消失 | "已自动保存" |
| 需确认 | 弹窗（单按钮） | "项目创建成功" |
| 需选择 | 弹窗（多按钮） | "文件已存在，覆盖/重命名/跳过？" |
| 需引导 | 弹窗 + 解决建议 | "磁盘空间不足，请清理后重试" |
| 严重 | 弹窗 + 重启提示 | "应用发生错误，建议重启" |
