### 1.2 威胁清单

| 编号 | 威胁 | 攻击面 | 风险等级 |
|------|------|--------|----------|

| T5 | 依赖供应链攻击（恶意 npm/pypi 包） | 构建/运行时 | 🟡 中 |
| T6 | 预览 iframe 绕过主进程权限 | iframe → 主进程 | 🔴 高 |
| T7 | 本地服务端口暴露（Python FastAPI） | 网络层 | 🟢 低 |

## 2. 内容安全策略（CSP）

### 2.1 Electron 主窗口 CSP

```javascript
// main/index.ts
session.defaultSession.webRequest.onHeadersReceived((details, callback) => {
  callback({
    responseHeaders: {
      ...details.responseHeaders,
      'Content-Security-Policy': [
        "default-src 'self';",
        "script-src 'self' 'unsafe-inline';",  // React 开发需要 inline script，生产禁用
        "style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;",
        "font-src 'self' https://fonts.gstatic.com;",
        "img-src 'self' blob: data:;",
        "media-src 'self' blob:;",
        "connect-src 'self' http://localhost:8000;",  // Python FastAPI
        "frame-src 'self' http://localhost:*;",     // Remotion dev server
        "object-src 'none';",
        "base-uri 'self';",
        "form-action 'self';",
        "upgrade-insecure-requests;"
      ].join(' ')
    }
  });
});
```

### 2.2 Remotion iframe CSP（更严格）

```javascript
// 子项目 iframe 的 CSP（通过 webpack dev server 响应头注入）
const REMOTION_CSP = [
  "default-src 'none';",
  "script-src 'self';",           // 禁止 inline script
  "style-src 'self' 'unsafe-inline';",
  "img-src 'self' blob: data:;",
  "media-src 'self' blob:;",
  "connect-src 'none';",         // 禁止 iframe 内网络请求
  "frame-src 'none';",
  "object-src 'none';"
].join(' ');
```

### 2.3 CSP 违规处理

- 开发模式：CSP 违规输出到 DevTools Console，不阻断。
- 生产模式：CSP 违规上报到 `~/.easymotion/logs/csp-violations.log`。
- 所有 `eval`、`Function`、`setTimeout(string)` 触发 CSP 违规 → 被阻止。

## 3. 代码注入防护（LLM 生成代码安全）

### 3.1 生成代码的危险 API 黑名单

LLM 生成的 TSX 代码中，以下 API/模式**绝对禁止**：

| 类别 | 禁止模式 | 风险 |
|------|----------|------|
| 代码执行 | `eval()`、`Function()`、`setTimeout(string)`、`setInterval(string)`、`setInterval()` | 任意代码执行 |
| 动态导入 | `import()`、`require()`（动态） | 加载未知模块 |
| 网络请求 | `fetch()`（非白名单 URL）、`XMLHttpRequest`、`WebSocket` | 数据外泄、CORS 绕过 |
| DOM 操作 | `document.write()`、`innerHTML`（非白名单内容） | XSS |
| 系统访问 | `process`、`__dirname`、`__filename` | 暴露 Node.js 环境 |
| 全局变量 | `window.parent`、`top`、`opener` | 绕过 iframe 隔离 |
| 内联事件 | `onerror=`、`onclick=` 等 HTML 内联处理器 | XSS |

### 3.2 AST 安全扫描

Generator 生成代码后，通过 `@babel/parser` 解析 AST，递归检查所有节点：

```typescript
// main/utils/security-check.ts
import { parse } from '@babel/parser';
import traverse from '@babel/traverse';

const DANGEROUS_IDENTIFIERS = [
  'eval', 'Function', 'setTimeout', 'setInterval',
  'fetch', 'XMLHttpRequest', 'WebSocket',
  'document', 'window', 'process'
];

function scanForDangerousCode(code: string): string[] {
  const ast = parse(code, { sourceType: 'module', plugins: ['jsx', 'typescript'] });
  const violations: string[] = [];

  traverse(ast, {
    Identifier(path) {
      if (DANGEROUS_IDENTIFIERS.includes(path.node.name)) {
        violations.push(`发现危险标识符: ${path.node.name} 在 ${path.node.loc?.start.line} 行`);
      }
    },
    CallExpression(path) {
      if (path.node.callee.type === 'Import') {
        violations.push('发现动态 import()');
      }
    }
  });

  return violations;
}
```

### 3.3 生成代码沙箱执行（预览阶段）

Remotion 预览 iframe 运行在独立上下文：
- `sandbox="allow-scripts"`（仅允许脚本，不允许表单、弹窗、插件）
- `allow="autoplay"`（允许视频自动播放）
- 禁用 `allow-same-origin`（iframe 视为跨域，无法访问父窗口 DOM）

### 3.4 安全白名单（允许使用的 API）

LLM 生成代码**仅允许**使用以下 API 和模块：

```typescript
// 允许使用的 Remotion API
import {
  useCurrentFrame, useVideoConfig, AbsoluteFill,
  Sequence, Img, Video, Audio, interpolate, Easing
} from 'remotion';

// 允许使用的 React API
import React, { useState, useEffect, useMemo } from 'react';

// 允许使用的自定义组件（由 Generator 注入）
import { ChartLayer, TextLayer, ShapeLayer } from './components';
```

**白名单维护**：`main/utils/allowed-modules.ts`，列出所有允许 import 的模块路径。

## 4. 文件系统沙箱

### 4.1 路径访问控制

主进程所有文件操作必须通过 `PathResolver` 校验：

```typescript
// main/utils/path-resolver.ts
class PathResolver {
  private allowedRoots: string[];

  constructor(projectRoots: string[]) {
    this.allowedRoots = projectRoots.map(r => path.resolve(r));
  }

  resolve(inputPath: string): string {
    const resolved = path.resolve(inputPath);

    // 1. 规范化路径（去除 ../ 和符号链接）
    const normalized = fs.realpathSync(resolved);
    // ⚠️ 必须用 root + path.sep 后缀，否则 /allowed-root-evil 也能通过 /allowed-root 检查
    const isAllowed = this.allowedRoots.some(root =>
      normalized.startsWith(root + path.sep) || normalized === root
    );

    if (!isAllowed) {
      throw new SecurityError(
        `路径越界访问: ${inputPath}`,
        'E2099'  // MAIN_PATH_TRAVERSAL（错误码字典中实际存在的码；E2036 不存在）
      );
    }

    return normalized;
  }
}

    return normalized;
  }
}
```

### 4.2 允许访问的目录

| 目录 | 用途 | 访问模式 |
|------|------|----------|
| `项目根目录/` | 项目文件 | 读写 |
| `~/.easymotion/` | 应用配置、日志、临时文件 | 读写 |
| `~/EasyMotion/` | 素材库数据库 | 读写 |
| 用户选择的导出目录 | 导出视频 | 只写 |
| 系统临时目录 | 临时文件 | 读写（自动清理） |

### 4.3 禁止访问的目录

| 目录 | 原因 |
|------|------|
| `/etc/`、`C:\Windows\` | 系统配置 |
| `~/.ssh/` | 私钥 |
| 其他项目目录 | 隔离 |
| 外部可执行文件 | 防止启动恶意程序 |

## 5. 对话面板 XSS 防护

### 5.1 消息渲染策略

对话面板中的消息内容**禁止直接渲染为 HTML**，必须使用纯文本或受控的 Markdown：

```tsx
// 安全的消息渲染
function MessageContent({ content }: { content: string }) {
  // 1. 转义 HTML 特殊字符
  const escaped = escapeHtml(content);

  // 2. 使用 marked + DOMPurify 渲染 Markdown（如需要）
  const sanitized = DOMPurify.sanitize(marked.parse(escaped));

  // 3. 禁止的 Markdown 元素：HTML 标签、JavaScript 链接
  return <div dangerouslySetInnerHTML={{ __html: sanitized }} />;
}
```

### 5.2 图片 URL 校验

对话面板中显示的图片（用户上传或 AI 生成）必须通过白名单：

```typescript
function isSafeImageUrl(url: string): boolean {
  // 仅允许本地文件路径和 blob URL
  return url.startsWith('blob:') ||
         url.startsWith('file://') ||
         url.startsWith('assets/');
}
```

## 6. Python 子进程安全

### 6.1 本地绑定策略

Python FastAPI 服务**仅绑定 localhost**：

```python
# main.py
uvicorn.run(app, host="127.0.0.1", port=8000)
# 禁止 host="0.0.0.0"，防止外部网络访问
```

### 6.2 请求来源校验

FastAPI 中间件校验请求是否来自本机：

```python
@app.middleware("http")
async def validate_source(request: Request, call_next):
    client_ip = request.client.host
    if client_ip not in ("127.0.0.1", "::1"):
        return JSONResponse(status_code=403, content={"error": "Forbidden"})
    return await call_next(request)
```

### 6.3 爬虫模块沙箱

Playwright 爬虫运行在隔离的浏览器上下文：

```python
# playwright_crawler.py
browser = await playwright.chromium.launch(
    args=['--no-sandbox', '--disable-setuid-sandbox'],
    handleSIGINT=False,
    handleSIGTERM=False
)
context = await browser.new_context(
    java_script_enabled=True,
    bypass_csp=True,  # 仅用于爬取，不影响主应用
    user_agent='EasyMotionBot/1.0'
)
```

## 7. 依赖供应链安全

### 7.1 依赖审计

- 使用 `pnpm audit` 定期扫描 npm 依赖漏洞。
- Python 依赖使用 `pip-audit` 扫描。
- CI 构建时自动执行审计，发现 High/Critical 漏洞时阻断构建。

### 7.2 依赖锁定

- `package-lock.json` / `pnpm-lock.yaml` 必须提交到版本控制。
- `requirements.txt` 使用精确版本号（`==`），禁止松散版本（`>=`、`^`）。
- 私有 registry：配置 `.npmrc` 和 `pip.conf` 使用可信镜像源。

### 7.3 最小权限原则

- Electron `nodeIntegration`：**禁用**（默认）
- Electron `contextIsolation`：**启用**（默认）
- Electron `allowRunningInsecureContent`：**禁用**
- Electron `experimentalFeatures`：**禁用**

## 8. 安全事件处理

### 8.1 安全违规响应

| 场景 | 响应策略 |
|------|----------|
| LLM 生成代码含危险 API | 拒绝写入，标记为"安全违规"，提示用户调整描述 |
| 路径越界访问 | 拒绝操作，记录日志，提示"无权访问该路径" |
| CSP 违规 | 阻止加载，上报日志 |
| 发现 XSS 尝试 | 转义内容，不上报网络（防止反射 XSS） |
| Python 服务收到外部请求 | 拒绝（403），记录来源 IP |

### 8.2 安全日志

```
~/.easymotion/logs/security.log
```

记录内容：
- 时间戳
- 违规类型（CODE_INJECTION / PATH_TRAVERSAL / CSP_VIOLATION）
- 来源（channel / 用户操作 / LLM 生成）
- 违规详情（脱敏）
- 处理结果

日志保留策略：30 天自动轮转，最大 100MB。
