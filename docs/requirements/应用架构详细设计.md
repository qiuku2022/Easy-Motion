# 应用架构详细设计

## 概述

本文档定义 Electron 应用的详细架构设计，包括主进程/渲染进程的模块划分、窗口管理、生命周期管理和模块间通信接口。

> **实施状态（2026-06-26）**：主进程为 **CommonJS（`.js`）**，无 `tsc` 编译步骤；预览默认走 timeline JSON → 动态 `MainSequence`；Generator 用于导出与 Remotion 同步。Legacy UI 保留于 `renderer/legacy/`（`pnpm dev:legacy`）。

## 主进程（Main Process）架构

### 模块职责

| 模块 | 文件 | 职责 | 说明 |
|------|------|------|------|
| **入口** | `main/index.ts` | 应用生命周期管理 | 初始化、窗口创建、退出处理 |
| **窗口管理** | `main/window-manager.ts` | 多窗口创建、销毁、状态管理 | 主窗口、项目窗口、设置窗口 |
| **项目服务** | `main/project-service.ts` | 项目 CRUD | 新建/打开/保存/删除总项目和子项目 |
| **文件服务** | `main/file-service.ts` | 文件系统抽象 | 项目目录操作、素材复制、缩略图生成 |
| **Python 服务** | `main/python-service.ts` | Python 子进程生命周期 | 启动、心跳检测、崩溃重启、端口管理 |
| **LLM 服务** | `main/llm-service.ts` | LLM API 调用代理 | 请求转发、错误处理、流式响应 |
| **生成器** | `main/generator/` | Remotion 代码生成 | Timeline JSON → TSX 代码 |
| **IPC 处理器** | `main/ipc-handlers/` | 各域名的 IPC 请求处理 | 按领域分离，避免单文件膨胀 |

### 主进程启动流程

```
main/index.ts 启动
  ├── 1. 单实例检查（确保只有一个 EasyMotion 实例运行）
  ├── 2. 初始化配置目录（~/.easymotion/）
  ├── 3. 初始化 SQLite 数据库（~/EasyMotion/library.db）
  ├── 4. 注册 IPC 处理器（加载 ipc-handlers/ 下所有模块）
  ├── 5. 启动 Python FastAPI 子进程
  │     └── 等待 health check 通过（最多 30 秒）
  ├── 6. 创建主窗口（加载 renderer/）
  ├── 7. 注册系统事件（before-quit、will-quit、window-all-closed）
  └── 8. 就绪状态广播 → renderer
```

### 退出流程

```
before-quit 事件触发
  ├── 1. 发送 "应用即将关闭" 广播 → 所有 renderer
  ├── 2. renderer 保存未持久化的 Store 状态
  ├── 3. 主进程等待 renderer 确认（最多 5 秒，超时强制继续）
  ├── 4. 停止 Python 子进程（发送 SIGTERM，5 秒后未退出则 SIGKILL）
  ├── 5. 关闭所有窗口
  ├── 6. 清理临时文件（~/.easymotion/temp/）
  └── 7. 退出应用
```

## 窗口管理

### 窗口类型

| 窗口 | 标识 | 尺寸 | 特性 | 说明 |
|------|------|------|------|------|
| **主窗口** | `main` | 1440×900，最小 1200×700 | 可调整大小，有菜单栏 | 核心工作区 |
| **项目窗口** | `project-manager` | 800×600 | 固定大小，模态/独立 | 新建/打开项目 |
| **设置窗口** | `settings` | 700×500 | 固定大小，模态 | 应用设置 |
| **导出窗口** | `export` | 600×400 | 固定大小，模态 | 导出参数配置 |

### 窗口生命周期

```
应用启动
  └── 创建主窗口
        ├── 加载渲染进程（vite dev server 或打包后的 index.html）
        ├── 注册窗口事件（resize、close、focus、blur）
        └── 窗口就绪后显示（避免白屏）

用户操作 → 打开项目窗口
  ├── 主进程创建项目窗口
  ├── 项目窗口加载独立渲染进程
  ├── 项目窗口与主窗口通信（通过 IPC 中转）
  └── 用户选择项目 → 项目窗口关闭 → 主窗口加载项目

主窗口关闭
  ├── macOS：隐藏窗口（不退出应用，点击 Dock 可恢复）
  └── Windows/Linux：触发应用退出流程
```

### 窗口状态持久化

- 主窗口：关闭时保存位置和大小，下次启动恢复
- 面板宽度：保存到 `~/.easymotion/ui-state.json`
- 主题设置：保存到 `~/.easymotion/settings.json`

## IPC 接口详细定义

### 命名规范

- `main:{domain}:{action}`：渲染进程 → 主进程请求
- `renderer:{domain}:{event}`：主进程 → 渲染进程通知

### project 域

```typescript
// 渲染进程 → 主进程
interface ProjectIPC {
  'main:project:create': (config: ProjectConfig) => Promise<{ success: boolean; path: string; error?: string }>;
  'main:project:open': (path: string) => Promise<{ success: boolean; project: Project; error?: string }>;
  'main:project:save': () => Promise<{ success: boolean; error?: string }>;
  'main:project:close': () => Promise<void>;
  'main:project:listRecent': () => Promise<RecentProject[]>;
  'main:project:delete': (path: string) => Promise<{ success: boolean; error?: string }>;
  'main:project:createSubproject': (name: string, template?: string) => Promise<{ success: boolean; subproject: Subproject; error?: string }>;
  'main:project:switchSubproject': (subprojectId: string) => Promise<{ success: boolean; error?: string }>;
```

## 生命周期与错误处理

### 应用启动错误

| 阶段 | 可能错误 | 处理策略 |
|------|----------|----------|
| 单实例检查 | 已有实例在运行 | 激活已有实例窗口，新实例退出 |
| 配置目录初始化 | 磁盘权限不足 | 提示用户检查权限，退出应用 |
| SQLite 初始化 | 数据库文件损坏 | 尝试重建数据库，失败则提示并退出 |
| Python 启动 | 端口被占用 | 自动尝试其他端口（8080-8090），仍失败则提示 |
| Python 启动 | 可执行文件缺失 | 提示"Python 环境异常，请重新安装应用" |
| 主窗口创建 | 渲染进程加载失败 | 重试 2 次，仍失败提示并退出 |

### 运行时错误

| 场景 | 检测方式 | 恢复策略 |
|------|----------|----------|
| 渲染进程崩溃 | `webContents.on('render-process-gone')` | 自动重载窗口，提示"界面已刷新，未保存内容可能丢失" |
| Python 子进程崩溃 | 心跳检测（5 秒 ping）失败 | 自动重启 Python，重启期间返回 "服务暂不可用" |
| 主进程内存不足 | 监听 `process.memoryUsage()` | 提示"系统资源不足"，建议关闭其他应用 |
| 项目文件损坏 | JSON 解析失败 | 尝试从 `project.json.bak` 恢复，失败则提示创建新项目 |

## 性能设计

### 渲染进程优化

- **虚拟滚动**：时间线轨道列表、素材库列表使用虚拟滚动，避免 DOM 节点过多
- **防抖**：Generator 触发 500ms 防抖，属性面板输入 100ms 防抖
- **懒加载**：对话历史超过 50 条时，只渲染可视区域 + 上下缓冲 10 条
- **组件拆分**：时间线每个轨道独立 React 组件，减少不必要的重渲染

### 主进程优化

- **并发控制**：项目文件写队列串行执行；素材导入支持批量并行（最多 5 个并发）
- **缓存**：缩略图生成结果缓存到文件系统；素材元数据缓存到 SQLite
- **流式处理**：LLM 响应流式返回渲染进程，不等待完整响应
