# 可访问性（a11y）设计

> **实施状态（2026-06-30）**：**M0–M9 ✅**（Windows MVP 预发行 v0.1.2）；后续以功能增强为主。权威入口：仓库根 [README.md](../../README.md)、[开发里程碑与路线图.md](./开发里程碑与路线图.md)。

## 概述

本文档定义 EasyMotion 在键盘导航、屏幕阅读器支持、高对比度模式和减少动画等方面的可访问性设计标准。目标用户包括使用键盘操作的视频编辑者和依赖辅助技术的用户。

> **版本**：v0.1  
> **最后更新**：2026-06-26  
> **关联文档**：UI布局与交互设计-优化版.md、交互流程.md、技术规格.md

## 1. 设计原则

### 1.1 WCAG 2.1 目标

EasyMotion 遵循 **WCAG 2.1 Level AA** 标准，核心要求：

| 原则 | 说明 | 本项目关键应用 |
|------|------|---------------|
| **可感知** | 信息和 UI 组件必须以用户可感知的方式呈现 | 颜色不是唯一信息载体、支持屏幕阅读器 |
| **可操作** | UI 组件必须可操作 | 全键盘导航、不依赖时间限制 |
| **可理解** | 信息和操作必须可理解 | 一致的导航、清晰的错误提示 |
| **健壮性** | 兼容辅助技术 | 语义化 HTML、正确的 ARIA 标签 |

### 1.2 核心场景

| 用户类型 | 需求 | 设计响应 |
|----------|------|----------|
| 键盘用户 | 不使用鼠标，全程键盘操作 | 完整的 Tab 顺序、快捷键系统 |
| 低视力用户 | 难以看清小文字、对比度不足 | 高对比度模式、文字缩放 |
| 色盲用户 | 无法区分某些颜色 | 颜色+图标/文字双重提示 |
| 前庭障碍用户 | 动画引起不适 | 减少动画模式、无自动播放 |
| 屏幕阅读器用户 | 依赖语音反馈 | 语义化结构、ARIA 标签 |

## 2. 键盘导航

### 2.1 全局导航顺序

```
Tab 顺序（按视觉布局从上到下、从左到右）：

顶部菜单栏
  ├── 文件 → 编辑 → 视图 → 帮助 → 渲染 → 导出
  │
左侧面板
  ├── Tab 切换（项目/素材/预设）
  ├── 内容区域（树/列表/网格）
  │
中间工作区
  ├── 预览窗口（焦点时启用播放控制快捷键）
  ├── 对话面板（输入框 → 发送按钮 → 历史消息）
  │
右侧面板
  ├── 属性面板（表单字段按逻辑顺序）
  │
底部时间线
  ├── 时间线工具栏
  ├── 轨道区域（轨道头 → 片段）
  └── 播放控制栏
```

### 2.2 焦点管理

**焦点可见性：**
- 所有可聚焦元素必须有清晰的焦点指示器。
- 焦点样式：`outline: 2px solid #E11D48; outline-offset: 2px;`（强调色）
- 禁止使用 `outline: none` 除非提供替代焦点样式。

**焦点陷阱防护：**
- 模态弹窗打开时，焦点锁定在弹窗内（`FocusTrap`）。
- 弹窗关闭时，焦点返回到触发按钮。
- 快捷键面板、设置面板使用相同策略。

**焦点恢复：**
- 切换子项目后，焦点返回到时间线区域。
- 导入素材完成后，焦点移动到素材库中新导入的项。

### 2.4 快捷键冲突避免

- 不覆盖系统级快捷键（`Ctrl+Alt+Del`、`Cmd+Space` 等）。
- 不覆盖 Electron 默认快捷键（`Ctrl+W` 关闭窗口、`Ctrl+R` 刷新 → 开发模式保留，生产模式禁用）。
- 所有快捷键在「设置」→「快捷键」中可自定义。

## 3. 屏幕阅读器支持

### 3.1 语义化结构

```tsx
// 正确的语义化示例
<main>                              {/* 主内容区 */}
  <nav aria-label="顶部菜单">        {/* 菜单栏 */}
    <ul role="menubar">
      <li role="none">
        <button role="menuitem">文件</button>
      </li>
    </ul>
  </nav>
  
  <section aria-label="时间线编辑器"> {/* 时间线 */}
    <div role="toolbar" aria-label="时间线工具">
      <button aria-label="添加轨道">+</button>
    </div>
    <div role="list" aria-label="轨道列表">
      <div role="listitem" aria-label="文字轨道，包含 3 个片段">
        {/* 轨道内容 */}
      </div>
    </div>
  </section>
</main>
```

### 3.2 ARIA 标签规范

| 组件 | 角色 | 必要标签 | 状态属性 |
|------|------|----------|----------|
| 轨道头 | `role="listitem"` | `aria-label="{轨道名}轨道，{片段数}个片段"` | `aria-selected` |
| 时间线片段 | `role="button"` | `aria-label="{片段名}，{开始帧}到{结束帧}"` | `aria-selected`, `aria-grabbed` |
| 播放头 | `role="slider"` | `aria-label="播放头，当前{frame}帧"` | `aria-valuenow`, `aria-valuemax` |
| 预览窗口 | `role="img"` | `aria-label="视频预览，{当前帧}/{总帧数}"` | `aria-busy`（加载中） |
| 对话消息 | `role="log"` | `aria-label="对话历史"` | `aria-live="polite"` |
| 属性面板 | `role="form"` | `aria-label="属性编辑"` | |
| Tab 切换 | `role="tablist"` | `aria-label="{面板名}"` | `aria-selected` |
| 按钮 | `role="button"` 或原生 `<button>` | 必须有可见文字或 `aria-label` | `aria-disabled`, `aria-pressed` |

### 3.3 实时区域（Live Region）

需要即时通知屏幕阅读器的变化：

```tsx
// 导出进度实时播报
<div aria-live="polite" aria-atomic="true" className="sr-only">
  导出进度：{progress}%，预计剩余 {remaining} 秒
</div>

// Agent 生成状态播报
<div aria-live="assertive" aria-atomic="true" className="sr-only">
  {agentStatus === 'generating' ? '正在生成动画，请稍候' : '生成完成'}
</div>
```

| 场景 | Live 模式 | 说明 |
|------|-----------|------|
| 导出进度更新 | `polite` | 不打断用户，空闲时播报 |
| 错误提示 | `assertive` | 立即打断，优先播报 |
| 对话新消息 | `polite` | |
| 播放头帧号 | `off` | 变化太频繁，不播报（用户按方向键时单独播报） |

### 3.4 跳过链接（Skip Link）

```tsx
// 页面顶部提供跳过导航的链接
<a href="#main-content" className="sr-only focus:not-sr-only">
  跳转到主内容
</a>
<a href="#timeline" className="sr-only focus:not-sr-only">
  跳转到时间线
</a>
```

## 4. 视觉辅助

### 4.1 高对比度模式

```css
/* 高对比度模式（系统级或应用内切换） */
@media (prefers-contrast: more),
       .high-contrast {          /* 应用内手动开启 */
  
  /* 边框增强 */
  * { border-color: currentColor !important; }
  
  /* 去除透明、模糊效果 */
  .glass-panel { background: #000 !important; }
  .backdrop-blur { backdrop-filter: none !important; }
  
  /* 强制对比度 */
  .text-secondary { color: #FFF !important; }
  .border-subtle { border-color: #FFF !important; border-width: 1px !important; }
  
  /* 焦点指示器增强 */
  :focus { outline: 3px solid #FFFF00 !important; outline-offset: 3px !important; }
}
```

**应用内切换：**
- 设置面板 →「可访问性」→「高对比度模式」开关。
- 切换时立即生效，无需重启。

### 4.2 文字缩放支持

- 界面支持浏览器级文字缩放（`Ctrl++` / `Ctrl+-`）。
- 使用 rem/em 单位，不固定 px。
- 文字放大到 200% 时，布局不崩坏（垂直滚动替代水平溢出）。

### 4.3 颜色与信息冗余

| 信息 | 颜色指示 | 辅助指示 |
|------|----------|----------|
| 轨道选中 | 蓝色边框 | 左侧竖条 + 名称高亮 |
| 片段静音 | 灰色 | 静音图标 + 文字标签 |
| 锁定状态 | 无明显颜色变化 | 锁图标 + `aria-label="已锁定"` |
| 错误状态 | 红色 | 错误图标 + 文字说明 |
| 成功状态 | 绿色 | 对勾图标 + 文字说明 |

## 5. 减少动画（Prefers Reduced Motion）

### 5.1 动画降级

```css
@media (prefers-reduced-motion: reduce),
       .reduced-motion {        /* 应用内手动开启 */
  
  /* 禁用所有过渡和动画 */
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
  
  /* 播放头移动无动画 */
  .playhead { transition: none !important; }
  
  /* 面板展开/收起无动画 */
  .panel-collapse { transition: none !important; }
  
  /* 弹窗出现无动画 */
  .modal-enter { animation: none !important; }
}
```

### 5.2 例外

以下动画在减少动画模式下**保留**（功能性必需）：
- 播放头移动（实时反馈）
- 视频播放（本身即运动内容）
- 生成进度条（进度指示）

### 5.3 应用内切换

- 设置面板 →「可访问性」→「减少动画」开关。
- 与系统偏好联动：`prefers-reduced-motion` 变化时自动同步。

## 6. 焦点模式与键盘陷阱

### 6.1 时间线焦点模式

当焦点进入时间线区域时，启用**时间线导航模式**：

```
时间线导航模式（状态栏显示 "时间线导航模式 - 按 Esc 退出"）
├── 方向键：在轨道和片段间移动焦点
├── Enter：选中当前片段（打开属性面板）
├── Space：播放/暂停
├── Delete：删除选中片段
├── Ctrl+C / Ctrl+V：复制/粘贴
└── Esc：退出导航模式，焦点返回上级区域
```

### 6.2 拖拽的键盘替代

拖拽操作提供键盘替代方案：

| 拖拽操作 | 键盘替代 |
|----------|----------|
| 移动片段 | 选中片段 → `Ctrl+Shift+方向键`（移动 1 帧/10 帧） |
| 调整片段时长 | 选中片段边缘 → `Shift+方向键`（扩展/收缩） |
| 轨道排序 | 选中轨道 → `Ctrl+Shift+↑/↓` |
| 素材导入 | 选中素材 → `Enter`（添加到当前轨道末尾） |

### 6.3 右键菜单的键盘访问

- 焦点在可右键元素上时，`Shift+F10` 或 `Menu` 键打开上下文菜单。
- 上下文菜单打开后，支持方向键导航和 Enter 选择。
