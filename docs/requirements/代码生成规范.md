# 代码生成规范

## 概述

本文档定义 Remotion 代码生成器的输出规范，包括标准模板、组件规范、安全约束和参数化规则。

> **实施状态（2026-06-26）**：Generator 在 **导出预检**、**Remotion 双向同步**、Legacy 路径时运行。**日常预览**由模板内 `MainSequence` 接收 `timeline` prop 动态渲染（含 `animation` 轨道预设 `source.props`），改参不必每次重新生成 TSX。Agent 生成的 timeline 变更走预览动态路径；M5.2 自定义组件写入 `remotion/src/components/custom/**`。

---

## 生成器输出目录结构

生成器每次运行后，输出以下文件到子项目的 `remotion/src/` 目录：

```
subprojects/{子项目名}/remotion/src/
├── index.tsx                    # React 应用入口（不变）
├── Root.tsx                     # Remotion Composition 定义（由 Generator 生成）
├── components/
│   ├── MainSequence.tsx           # 主时间线组件（由 Generator 生成）
│   ├── layers/                    # Layer 组件目录
│   │   ├── TextLayer.tsx          # 文字层（标准组件，Generator 引用）
│   │   ├── ImageLayer.tsx         # 图片层（标准组件，Generator 引用）
│   │   ├── VideoLayer.tsx         # 视频层（标准组件，Generator 引用）
│   │   ├── AudioLayer.tsx         # 音频层（标准组件，Generator 引用）
│   │   ├── ShapeLayer.tsx         # 形状层（标准组件，Generator 引用）
│   │   ├── ChartLayer.tsx         # 图表层（标准组件，Generator 引用）
│   │   └── AnimationLayer.tsx     # 动画层（标准组件，Generator 引用）
│   └── transitions/               # 转场效果组件（可选）
│       ├── FadeTransition.tsx
│       └── SlideTransition.tsx
```

**规则：**
- `index.tsx` 为固定模板，Generator 不修改
- **预览路径**：`MainSequence.tsx` 支持 `timeline` prop，运行时由 `PreviewClipSequence` 渲染（预设、关键帧、`apply-keyframes`）
- **关键帧工具**：`src/lib/apply-keyframes.ts`；`src/components/layers/*Layer.tsx` 须 `import … from "../../lib/apply-keyframes"`（勿用 `../lib/`，会解析到不存在的 `components/lib/`）
- **旧项目修复**：预览启动时 `ensureLayerKeyframesImport()` 自动修正错误 import 并补全缺失的 `apply-keyframes.ts`
- **Generator 路径**：`Root.tsx` 与 `MainSequence.tsx` 在 `main:timeline:generate` 时覆盖写入（导出/同步）
- `layers/` 下组件为模板标准组件；RVE 预设 vendoring 在 `renderer/public/presets/`

## 标准模板

### 1. index.tsx（固定模板）

```tsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { RemotionRoot } from "./Root";

const rootElement = document.getElementById("root");
if (!rootElement) {
  throw new Error("Root element not found");
}

createRoot(rootElement).render(
  <StrictMode>
    <RemotionRoot />
  </StrictMode>
);
```

**规则：**
- 必须使用 `StrictMode`
- 使用 `createRoot` API（React 18）
- 不做任何修改

### 2. Root.tsx（Generator 生成）

```tsx
import { Composition } from "remotion";
import { MainSequence } from "./components/MainSequence";

export const RemotionRoot: React.FC = () => {
  return (
    <Composition
      id="Main"
      component={MainSequence}
      durationInFrames={300}
      fps={30}
      width={1920}
      height={1080}
      defaultProps={{
        // 可注入的外部参数（预留扩展）
      }}
    />
  );
};
```

**规则：**
- `id` 固定为 `"Main"`（与预览系统约定一致）
- `durationInFrames`、`fps`、`width`、`height` 从 `subproject.json` 的 `timeline` 字段读取
- 不使用 `calculateMetadata`（避免异步复杂度）
- 所有可变内容放在 `MainSequence` 中，不直接写在 `Root.tsx`

### 3. MainSequence.tsx（Generator 生成）

```tsx
import { AbsoluteFill, Sequence } from "remotion";
import { TextLayer } from "./components/layers/TextLayer";
import { ImageLayer } from "./components/layers/ImageLayer";
// ... 其他 Layer 导入

// Props 类型定义（可选扩展）
interface MainSequenceProps {
  // 目前为空，预留外部数据注入
}

export const MainSequence: React.FC<MainSequenceProps> = () => {
  return (
    <AbsoluteFill>
      {/* 
        轨道按 order 升序渲染
        每个 Sequence 对应一个 clip
        轨道 type 决定使用的 Layer 组件
      */}
      
      {/* Track: 文字标题 (order: 1) */}
      <Sequence from={0} durationInFrames={90}>
        <TextLayer
          clipId="clip-001"
          source={{ kind: "inline", content: "Hello World" }}
          transform={{
            position: { x: 960, y: 540 },
            scale: 1,
            rotation: 0,
            opacity: 1,
          }}
          style={{
            fontFamily: "Inter",
            fontSize: 64,
            color: "#ffffff",
            textAlign: "center",
          }}
          keyframes={[
            {
              id: "kf-001",
              property: "transform.position.x",
              frame: 0,
              value: 960,
              easing: "linear",
              interpolation: "linear",
            },
          ]}
          inAnimation={{ type: "fade", durationInFrames: 15 }}
          outAnimation={{ type: "slide-left", durationInFrames: 15 }}
        />
      </Sequence>
      
      {/* Track: 背景图片 (order: 0) */}
      <Sequence from={0} durationInFrames={300}>
        <ImageLayer
          clipId="clip-002"
          src="assets/images/a1b2c3d4_bg.png"
          transform={{
            position: { x: 960, y: 540 },
            scale: 1.2,
            rotation: 0,
            opacity: 0.3,
          }}
          style={{
            objectFit: "cover",
          }}
        />
      </Sequence>
    </AbsoluteFill>
  );
};
```

**生成规则：**
1. **导入**：只导入实际使用的 Layer 组件，不导入未使用的
2. **Sequence**：每个 clip 对应一个 `<Sequence from={start} durationInFrames={duration}>`
3. **轨道顺序**：按 `track.order` 升序排列，先渲染 order 小的（底层），后渲染 order 大的（上层覆盖）
4. **clipId**：每个 Layer 组件必须传入 `clipId`，用于 iframe 点击选中时反向映射
5. **素材路径**：使用相对路径 `assets/{类型}/{uuid}_{文件名}`
6. **无硬编码魔法数字**：所有数值来自 `subproject.json` 的数据结构

## Layer 组件规范

### 通用 Props 接口

所有 Layer 组件必须实现以下通用接口：

```typescript
// 通用变换属性
interface Transform {
  position: { x: number; y: number };
  scale: number;
  rotation: number;
  opacity: number;
}

// 关键帧
// 注意：value 字段的类型由 property 路径决定（数字、字符串、颜色、向量等），
// 用判别联合 + unknown 兜底，避免使用 any（违反项目编码规范"禁止 any"规则）。
export type KeyframeValue =
  | number                                       // 标量属性：fontSize, opacity, scale, rotation 等
  | string                                       // 字符串属性：color, fontFamily, text 等
  | { x: number; y: number }                     // 向量属性：transform.position
  | { width: number; height: number }            // 尺寸属性：transform.size
  | [number, number, number, number]              // 颜色属性：[r, g, b, a]，0-1 范围
  | { type: 'bezier'; cp: [number, number, number, number] }  // 自定义贝塞尔
  | { type: 'spring'; config: { damping: number; stiffness: number; mass: number } }
  | unknown;                                     // 兜底：新引入的属性类型未来扩展

interface Keyframe {
  id: string;
  property: string;          // 如 "transform.position.x"
  frame: number;
  value: KeyframeValue;
  easing: 'linear' | 'ease-in' | 'ease-out' | 'ease-in-out' | 'spring';
  interpolation: 'linear' | 'bezier' | 'hold';
}

// 入场/出场动画
interface AnimationConfig {
  type: 'fade' | 'slide-left' | 'slide-right' | 'slide-up' | 'slide-down'
      | 'scale-up' | 'scale-down' | 'rotate' | 'blur' | 'none';
  durationInFrames: number;  // 动画持续帧数
}

// Layer 通用 Props
interface LayerProps {
  clipId: string;            // 用于选中映射
  transform: Transform;
  keyframes?: Keyframe[];
  inAnimation?: AnimationConfig;
  outAnimation?: AnimationConfig;
}
```

## 动画参数化规范

### 可调参数列表

以下参数**必须**通过 Props 传入 Layer 组件，不能硬编码在组件内部，确保 Agent 可以通过对话调整：

| 参数路径 | 类型 | 调整示例 |
|----------|------|----------|
| `transform.position.x` | number | "移到左边" |
| `transform.position.y` | number | "往下一点" |
| `transform.scale` | number | "放大一点" |
| `transform.rotation` | number | "旋转 45 度" |
| `transform.opacity` | number | "透明一点" |
| `style.color` | string | "改成红色" |
| `style.fontSize` | number | "字体大一点" |
| `inAnimation.type` | string | "改成缩放入场" |
| `inAnimation.durationInFrames` | number | "入场快一点" |
| `outAnimation.type` | string | "改成淡出" |
| `outAnimation.durationInFrames` | number | "出场慢一点" |

### 不可调参数（内部常量）

以下参数由 Layer 组件内部固定，不暴露给 Agent：

- Layer 组件的 DOM 结构（div 嵌套层级）
- 入场/出场动画的具体实现细节（如 fade 的具体 opacity 曲线）
- 图表的默认网格线样式
- 文字的行高计算逻辑（基于 fontSize 的倍数）

### 缓动函数映射

```typescript
// Layer 组件内部实现
import { Easing, interpolate } from "remotion";

// ⚠️ fps 必须从 useVideoConfig() 动态获取，禁止硬编码（支持 24/25/30/60 fps 项目）
const easingMap: Record<string, (t: number) => number> = (() => {
  const { fps } = useVideoConfig();
  return {
    "linear": Easing.linear,
    "ease-in": Easing.in(Easing.ease),
    "ease-out": Easing.out(Easing.ease),
    "ease-in-out": Easing.inOut(Easing.ease),
    "spring": (t: number) => spring({ frame: t * fps, fps, config: { damping: 10 } }),
  };
})();

// 关键帧插值时根据 keyframe.easing 选择对应函数
```

## 安全约束

### 白名单机制

Generator 生成的代码只允许使用以下 API 和模块：

**允许使用的 Remotion API：**
```
remotion:
  - useCurrentFrame
  - useVideoConfig
  - interpolate
  - spring
  - Easing
  - AbsoluteFill
  - Sequence
  - Composition
  - Video
  - Audio
  - Img
  - OffthreadVideo
  - staticFile
  - delayRender / continueRender
  - measureSpring
  - useRandomSeed
  - Random

@remotion/gif:
  - Gif

@remotion/shapes:
  - makeTriangle
  - makeCircle
  - makeRect
  - makeEllipse
```

**允许使用的 React API：**
```
react:
  - useState
  - useEffect
  - useMemo
  - useCallback
  - useRef
  - memo
  - forwardRef

react-dom:
  - createRoot（仅在 index.tsx 中）
```

**允许使用的浏览器原生 API：**
```
- Math（所有方法）
- Array（所有方法）
- Object（所有方法）
- JSON
- console（仅用于调试，生产环境禁用）
- setTimeout / clearTimeout
- requestAnimationFrame / cancelAnimationFrame
- fetch（仅限 HTTPS 请求图片/字体资源）
```

### 禁止使用的 API（黑名单）

以下 API 在生成的代码中**绝对禁止**出现：

| 禁止项 | 示例 | 检测方式 |
|--------|------|----------|
| Node.js 内置模块 | `require('fs')`, `require('path')`, `require('os')`, `require('child_process')` | AST 静态分析 |
| 网络请求（除 fetch 外） | `XMLHttpRequest`, `WebSocket`, `EventSource` | AST 静态分析 |
| 危险 DOM 操作 | `document.write`, `eval`, `new Function` | AST 静态分析 |
| 外部 iframe | `<iframe>`, `<object>`, `<embed>` | AST + 正则匹配 |
| 本地存储 | `localStorage`, `sessionStorage`, `indexedDB` | AST 静态分析 |
| 浏览器历史/导航 | `history.pushState`, `location.href`, `window.open` | AST 静态分析 |
| 系统信息获取 | `navigator.userAgent`, `navigator.platform`, `screen` | AST 静态分析 |
| 文件操作 | `FileReader`, `Blob`, `URL.createObjectURL` | AST 静态分析 |
| 定时器（可能滥用） | `setInterval` | AST 静态分析 |
| CSS 危险样式 | `pointer-events: none`（用于穿透点击但不允许自定义） | 正则匹配 |

### 安全校验流程

```
Generator 生成 TSX 代码
  ↓
[AST 静态分析]（使用 @babel/parser 或 esbuild）
  ├── 检查 import 声明：只允许白名单模块
  ├── 检查 require 调用：禁止出现
  ├── 检查全局变量访问：禁止黑名单 API
  └── 检查 JSX 元素：禁止 iframe/object/embed
        ↓
  通过 → 写入文件
  失败 → 标记为 "SECURITY_VIOLATION"
         → 记录违规详情
         → 向用户提示"生成内容包含不安全代码，已阻止"
         → 不执行写入
```

### 安全校验代码示例

```typescript
// security-check.ts（主进程中的校验模块）
import { parse } from "@babel/parser";
import traverse from "@babel/traverse";

const ALLOWED_MODULES = new Set([
  "remotion",
  "react",
  "react-dom/client",
  "@remotion/gif",
  "@remotion/shapes",
]);

const FORBIDDEN_GLOBALS = new Set([
  "eval",
  "Function",
  "setInterval",
  "localStorage",
  "sessionStorage",
  "indexedDB",
  "XMLHttpRequest",
  "WebSocket",
  "document.write",
]);

const FORBIDDEN_JSX = new Set([
  "iframe",
  "object",
  "embed",
  "script",
]);

export function validateGeneratedCode(code: string): { valid: boolean; violations: string[] } {
  const violations: string[] = [];
  
  const ast = parse(code, {
    sourceType: "module",
    plugins: ["jsx", "typescript"],
  });
  
  traverse(ast, {
    // 检查 import 来源
    ImportDeclaration(path) {
      const source = path.node.source.value;
      // 允许相对路径导入（项目内部模块）
      if (source.startsWith(".")) return;
      // 检查是否在白名单
      const moduleName = source.split("/")[0];
      if (!ALLOWED_MODULES.has(moduleName)) {
        violations.push(`禁止导入模块: ${source}`);
      }
    },
    
    // 检查 require
    CallExpression(path) {
      if (path.node.callee.type === "Identifier" && path.node.callee.name === "require") {
        violations.push("禁止使用 require()");
      }
    },
    
    // 检查危险全局变量
    Identifier(path) {
      if (FORBIDDEN_GLOBALS.has(path.node.name)) {
        violations.push(`禁止使用全局 API: ${path.node.name}`);
      }
    },
    
    // 检查危险 JSX 元素
    JSXOpeningElement(path) {
      const name = path.node.name;
      if (name.type === "JSXIdentifier" && FORBIDDEN_JSX.has(name.name)) {
        violations.push(`禁止使用 JSX 元素: <${name.name}>`);
      }
    },
  });
  
  return {
    valid: violations.length === 0,
    violations,
  };
}
```

## 代码风格规范

### TypeScript 规范

- 使用 `.tsx` 扩展名
- 显式声明 Props 接口（不使用 `any`）
- 函数组件使用箭头函数：`const Component: React.FC<Props> = (props) => {}`
- 使用单引号字符串（与 Prettier 默认配置一致）
- 缩进：2 个空格
- 最大行宽：100 字符

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件名 | PascalCase | `TextLayer`, `MainSequence` |
| Props 接口 | 组件名 + Props | `TextLayerProps` |
| 变量/函数 | camelCase | `currentFrame`, `calculateOpacity` |
| 常量 | UPPER_SNAKE_CASE | `DEFAULT_FPS` |
| 文件名 | PascalCase（组件）/ camelCase（工具） | `TextLayer.tsx`, `utils.ts` |

## 错误处理规范

### Layer 组件内部错误边界

每个 Layer 组件必须包裹错误边界，防止单个组件崩溃导致整个预览失败：

```tsx
// 在 MainSequence.tsx 中为每个 Layer 添加
<Sequence from={0} durationInFrames={90}>
  <ErrorBoundary fallback={<div style={{ color: 'red' }}>渲染错误</div>}>
    <TextLayer {...props} />
  </ErrorBoundary>
</Sequence>
```

### 素材加载失败处理

```tsx
// ImageLayer.tsx
const ImageLayer: React.FC<ImageLayerProps> = ({ src, ...props }) => {
  const [error, setError] = React.useState(false);
  
  if (error) {
    return (
      <div style={{ width: '100%', height: '100%', backgroundColor: '#333', display: 'flex', alignItems: 'center', justifyContent: 'center' }}>
        <span style={{ color: '#fff' }}>图片加载失败</span>
      </div>
    );
  }
  
  return <Img src={src} onError={() => setError(true)} {...props} />;
};
```

## 性能规范

### 渲染优化

1. **memo 包裹**：所有 Layer 组件使用 `React.memo` 包裹，避免不必要的重渲染
2. **useMemo**：复杂计算（如关键帧插值）使用 `useMemo`
3. **useCurrentFrame 使用**：每个组件只调用一次 `useCurrentFrame()`，不重复调用
4. **避免匿名函数**：事件处理器使用 `useCallback`

### 代码体积
