# 时间线编辑

## 多轨道编辑功能（完整时间线编辑器）
- **可视化时间线**：支持缩放、滚动、播放头控制
- **多轨道管理**：支持视频、音频、图片、文字、动画、数据图表等多种轨道类型
- **拖拽编辑**：支持素材拖拽调整位置、时长、裁剪
- **关键帧系统**：支持属性关键帧设置（位置、缩放、旋转、透明度等），Remotion 原生支持
- **图层控制**：图层叠加、混合模式、锁定、隐藏、排序
- **轨道操作**：添加/删除轨道、轨道分组、轨道独奏/静音
- **实时预览**：时间线操作与 Remotion 预览联动
- **AI 辅助**：通过对话自动添加/调整轨道和关键帧

## 时间线数据模型

> **设计原则**：UI 上的"轨道"和"片段"是用户心智模型，底层存储为 JSON，运行时通过生成器转换为 Remotion 的 React 组件树。

### 1. 子项目时间线配置（`subproject.json` 的 `timeline` 字段）

```json
{
  "timeline": {
    "version": "1.0",
    "fps": 30,
    "durationInFrames": 300,
    "width": 1920,
    "height": 1080,
    "tracks": [
      { /* Track 对象，见下文 */ }
    ],
    "snapGrid": {
      "enabled": true,
      "intervalInFrames": 1
    }
  }
}
```

### 2. 轨道（Track）数据结构

```json
{
  "id": "track-001",
  "name": "文字标题",
  "type": "text",
  "order": 0,
  "visible": true,
  "locked": false,
  "muted": false,
  "clips": [
    { /* Clip 对象，见下文 */ }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 全局唯一，UUID |
| `name` | string | 用户可编辑的轨道名称 |
| `type` | string | 轨道类型：`text`/`image`/`video`/`audio`/`shape`/`chart`/`animation`/`group` |
| `order` | number | 图层叠放顺序，0 为最底层，数值越大越上层 |
| `visible` | boolean | 是否可见 |
| `locked` | boolean | 是否锁定（禁止编辑） |
| `muted` | boolean | 是否静音（仅音频轨道有效） |
| `clips` | Clip[] | 该轨道上的所有片段 |

### 3. 片段（Clip）数据结构

```json
{
  "id": "clip-001",
  "type": "text",
  "name": "开场标题",
  "startInFrames": 0,
  "durationInFrames": 90,
  "source": {
    "kind": "inline",
    "content": "Hello World"
  },
  "transform": {
    "position": { "x": 960, "y": 540 },
    "scale": 1.0,
    "rotation": 0,
    "opacity": 1.0
  },
  "style": {
    "fontFamily": "Inter",
    "fontSize": 64,
    "color": "#ffffff",
    "textAlign": "center"
  },
  "keyframes": [
    { /* Keyframe 对象，见下文 */ }
  ],
  "animations": {
    "in": { "type": "fade", "durationInFrames": 15 },
    "out": { "type": "slide-left", "durationInFrames": 15 }
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 全局唯一 |
| `type` | string | 片段类型，与所在轨道 `type` 一致 |
| `name` | string | 用户可编辑的片段名称 |
| `startInFrames` | number | 在时间线上的起始帧号（从 0 开始） |
| `durationInFrames` | number | 持续帧数 |
| `source` | object | 数据来源，见下方 |
| `transform` | object | 基础变换属性 |
| `style` | object | 样式属性，因类型而异 |
| `keyframes` | Keyframe[] | 关键帧列表 |
| `animations` | object | 预设入场/出场动画 |

#### 片段数据来源（`source`）

| `kind` | 说明 | 示例 |
|--------|------|------|
| `inline` | 直接内联内容 | 文字内容、形状参数、图表数据 |
| `asset` | 引用素材库文件 | `{ "assetId": "asset-001", "path": "assets/logo.png" }` |
| `data` | 绑定数据文件 | `{ "file": "data/sales.csv", "field": "revenue" }` |

### 4. 关键帧（Keyframe）数据结构

```json
{
  "id": "kf-001",
  "property": "transform.position.x",
  "frame": 45,
  "value": 1200,
  "easing": "ease-in-out",
  "interpolation": "linear"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `property` | string | 目标属性路径，支持点号嵌套 |
| `frame` | number | 关键帧所在帧号 |
| `value` | any | 属性值，类型与目标属性一致 |
| `easing` | string | 缓动函数：`linear`/`ease-in`/`ease-out`/`ease-in-out`/`spring` |
| `interpolation` | string | 插值方式：`linear`/`bezier`/`hold` |

### 关键帧属性路径解析

- **`property` 字段格式**：使用点号分隔的嵌套路径，如 `"transform.position.x"`、`"style.color"`、`"transform.scale"`
- **运行时解析**：Layer 组件内部使用统一的 `getValueByPath(obj, path)` / `setValueByPath(obj, path, value)` 工具函数，按 `.` 分割路径并递归访问对象属性
- **支持的目标属性**：
  - `transform.position.x` / `transform.position.y`
  - `transform.scale`
  - `transform.rotation`
  - `transform.opacity`
  - `style.color`
  - `style.fontSize`
  - `style.letterSpacing`
  - 自定义属性（由 Layer 组件扩展）

### 关键帧缓动函数映射

| 时间线缓动名称 | Remotion 对应实现 | 说明 |
|----------------|-------------------|------|
| `linear` | `Easing.linear` | 匀速 |
| `ease-in` | `Easing.in(Easing.ease)` | 先慢后快 |
| `ease-out` | `Easing.out(Easing.ease)` | 先快后慢 |
| `ease-in-out` | `Easing.inOut(Easing.ease)` | 慢-快-慢 |
| `spring` | 使用 `spring()` 函数（Remotion 内置） | 弹性物理效果，无需 duration |

- **插值方式映射**：
  - `linear`：使用 `interpolate()` 线性插值
  - `bezier`：使用 `interpolate()` + 自定义贝塞尔控制点（`[x1, y1, x2, y2]`）
  - `hold`：直接返回最近关键帧的值（阶梯式变化）

### 5. 轨道类型与 Remotion 组件映射

| 轨道类型 | Remotion 组件/处理方式 | 说明 |
|----------|------------------------|------|
| `text` | 自定义 `<TextLayer>` 组件，基于绝对定位的 `div` | 支持 `fontFamily`、`fontSize` 等样式属性 |
| `image` | `<Img>` 或 `<OffthreadVideo>` | 静态图片用 `Img`，动态序列用 `OffthreadVideo` |
| `video` | `<Video>` 或 `<OffthreadVideo>` | 支持音量控制、裁剪 |
| `audio` | `<Audio>` 或 `<Sequence>` 包裹 | 纯音频轨道，无视觉输出 |
| `shape` | `<div>` + CSS / SVG | 矩形、圆形、线条等基础形状 |
| `chart` | 自定义图表组件（Recharts/D3 + Remotion） | 数据驱动，支持动画更新 |
| `animation` | 自定义动画组件 | 粒子效果、复杂动效等 |
| `group` | 嵌套轨道组 | 用于组织和管理复杂图层 |

### `group` 轨道类型的数据结构

`group` 是一种特殊的容器轨道，不直接包含 `clips`，而是包含**子轨道数组**：

```json
{
  "id": "track-group-001",
  "name": "标题组",
  "type": "group",
  "order": 2,
  "visible": true,
  "locked": false,
  "muted": false,
  "clips": [],
  "children": [
    { /* Track 对象：如 text 轨道 */ },
    { /* Track 对象：如 image 轨道 */ }
  ],
  "collapsed": false
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `children` | Track[] | 该组包含的子轨道，子轨道可具有独立的 `order`（组内排序）或继承父组相对顺序 |
| `collapsed` | boolean | 时间线中该组是否折叠（收起） |

**编译规则：**
- group 轨道的 `children` 在编译时**扁平化**到主轨道列表中
- 子轨道的实际渲染 `order` = 父组 `order` + 子轨道组内偏移
- group 的 `visible` / `locked` 会级联应用到所有子轨道
- group 本身不生成 `<Sequence>`，仅作为组织容器

## 轨道到 Remotion 的映射架构

### 核心问题
Remotion 没有"轨道"概念。Remotion 的模型是：
- **Composition**：一个视频的整体定义（时长、分辨率、帧率）
- **Sequence**：按时间排列的片段容器，可嵌套
- **组件**：普通的 React 组件，通过 `useCurrentFrame()` 获取当前帧号驱动动画

因此，UI 时间线的"轨道"和"片段"必须被**编译**为 Remotion 的组件树。

### 编译流程（Timeline JSON → Remotion Code）

```
subproject.json 的 timeline 字段
  ↓ 生成器（Generator）
Remotion 项目代码（src/Root.tsx + src/components/）
  ↓ Remotion Player / 渲染器
视频预览 / 导出文件
```

### 生成的 Remotion 代码结构

#### `src/Root.tsx`（入口）

```tsx
import { Composition } from "remotion";
import { MainSequence } from "./components/MainSequence";

export const RemotionRoot: React.FC = () => {
  return (
    <Composition
      id="Main"
      component={MainSequence}
      durationInFrames={300}
      fps={30}
      width={1920}
      height={1080}
    />
  );
};
```

#### `src/components/MainSequence.tsx`（时间线编译结果）

```tsx
import { AbsoluteFill, Sequence } from "remotion";
import { TextLayer } from "./layers/TextLayer";
import { ImageLayer } from "./layers/ImageLayer";

export const MainSequence: React.FC = () => {
  return (
    <AbsoluteFill>
      {/* 轨道按 order 从小到大渲染，后面的覆盖前面的（图层叠加） */}
      <Sequence from={0} durationInFrames={90}>
        <TextLayer
          text="Hello World"
          transform={{ position: { x: 960, y: 540 }, scale: 1, rotation: 0, opacity: 1 }}
          style={{ fontFamily: "Inter", fontSize: 64, color: "#ffffff" }}
          keyframes={[...]}
          inAnimation={{ type: "fade", durationInFrames: 15 }}
          outAnimation={{ type: "slide-left", durationInFrames: 15 }}
        />
      </Sequence>
      
      <Sequence from={30} durationInFrames={120}>
        <ImageLayer
          src="assets/logo.png"
          transform={{...}}
        />
      </Sequence>
    </AbsoluteFill>
  );
};
```

### 关键映射规则

1. **轨道顺序 → 渲染顺序**
   - `track.order` 升序排列，先渲染 order 小的，后渲染 order 大的（实现图层覆盖）
   - 所有轨道的内容包裹在 `<AbsoluteFill>` 中，默认绝对定位

2. **片段时间 → Sequence 的 `from` 和 `durationInFrames`**
   - `clip.startInFrames` → `Sequence.from`
   - `clip.durationInFrames` → `Sequence.durationInFrames`
   - 片段超出 Composition 时长的，自动截断或提示用户

3. **关键帧 → 组件内部动画逻辑**
   - 每个 `xxxLayer` 组件内部通过 `useCurrentFrame()` 读取当前帧号
   - 根据 `keyframes` 数组插值计算当前属性值
   - 缓动函数通过 `remotion` 的 `interpolate()` 和 `Easing` 实现

4. **入场/出场动画 → 组件内部包装**
   - `inAnimation` 和 `outAnimation` 由 Layer 组件内部处理
   - 例如 `fade` 入场：组件挂载后的前 N 帧内从 opacity 0 渐变到基础 opacity

5. **混合模式**
   - Remotion 导出视频时，CSS `mix-blend-mode` 在 Chromium 中有效
   - 但部分模式在 FFmpeg 合成时可能表现不一致，需在文档中标注受支持的模式列表（`normal`、`multiply`、`screen`、`overlay`）

## Generator 触发机制

### 触发策略：防抖 + 手动

| 触发方式 | 触发条件 | 说明 |
|----------|----------|------|
| **自动触发（防抖）** | timeline 修改后 500ms 内无新操作 | 默认行为。适用于大多数拖拽、属性微调操作 |
| **手动触发** | 用户点击"刷新预览"按钮 | 用于 Generator 自动触发被禁用，或用户想立即看到效果时 |
| **强制触发** | AI 生成完成后、项目打开时 | 必须立即同步到 Remotion 代码 |

### 触发场景明细

**自动触发（500ms 防抖）：**
- 拖拽片段位置/时长
- 调整属性面板数值
- 修改关键帧
- 轨道排序变化

**立即触发（无防抖）：**
- AI 对话调整完成
- 新建/删除片段
- 新建/删除轨道
- 导入素材到时间线

**不触发 Generator（纯 UI 状态）：**
- 播放头位置变化
- 面板展开/收起
- 选中元素变化
- 缩放/滚动时间线

### 并发控制
- Generator 执行期间，新的触发请求排队，待当前 Generator 完成后按最新状态重新触发（合并中间状态）
- Generator 执行耗时较长（> 2 秒）时，显示"正在生成预览..."进度指示器

## 时间线交互细节

### 1. 拖拽规则

| 操作 | 行为 | 约束 |
|------|------|------|
| 拖拽片段调整位置 | 修改 `clip.startInFrames` | 不能为负数；不能与其他同轨道片段重叠（除非开启叠加模式） |
| 拖拽片段边缘调整时长 | 修改 `clip.durationInFrames` | 最小 1 帧；不能超过素材原始时长（视频/音频） |
| 拖拽素材到时间线 | 创建新片段，`startInFrames` 为放置位置 | 自动分配到对应类型轨道；若无匹配轨道则自动创建 |
| 拖拽轨道调整顺序 | 修改 `track.order` | 所有轨道重新排序 |

### 2. 吸附（Snap）

- 默认开启，吸附间隔 1 帧（可在设置中调整）
- 吸附目标：
  - 其他片段的起始/结束边界
  - 播放头当前位置
  - 用户自定义标记点（Marker）
- 吸附距离阈值：10px（视缩放比例动态调整）
- 按住 `Alt` 键临时禁用吸附

### 3. 播放头同步

- **用户拖拽播放头**：计算目标帧号 → 更新 `previewStore.currentFrame` → 通过 IPC 通知 iframe 的 Remotion Player `seekTo(frame)`
- **播放中时间线滚动**：播放头到达视口边缘时，时间线自动滚动保持播放头可见（可选关闭）
- **iframe 点击选帧**：iframe 内点击元素 → `postMessage` 通知父窗口 → 时间线选中对应片段，播放头跳到该帧

### 4. 关键帧编辑

- 在时间线轨道区域下方显示**属性轨道**（可折叠）
- 每个可关键帧属性显示一条曲线/线段，关键帧以菱形标记
- 双击属性值 → 添加关键帧
- 拖拽关键帧 → 修改 `frame` 和 `value`
- 右键关键帧 → 删除 / 切换缓动函数

### 5. 撤销/重做

- 每次时间线修改（添加/删除/移动/调整片段、修改关键帧）生成一条 **Command** 记录
- Command 包含 `execute()` 和 `undo()` 方法
- 撤销栈存储在内存中，每 10 条自动快照写入 `timeline-snapshots/`（崩溃恢复用）
- 切换子项目时清空撤销栈（避免跨项目误操作）

## 边界与约束

1. **最大轨道数**：暂定 32 条（性能考虑），超出时提示用户合并或分组
2. **最大片段数**：单轨道不限，但同轨道片段重叠时默认拒绝（可开启"叠加模式"允许多片段共存）
3. **时间精度**：最小单位为 1 帧，不支持子帧编辑
4. **混合模式限制**：导出时仅保证 `normal`、`multiply`、`screen` 的跨平台一致性，其他模式在预览中可见但在导出时可能降级为 `normal`
5. **音频轨道**：Remotion 音频混合在 Chromium 中完成，复杂多轨道音频混音时建议在预览时降低采样率以保证性能
6. **修改来源标记（`lastModifiedBy`）**：
   - Clip 数据结构中包含 `lastModifiedBy: 'user' | 'ai' | null` 字段，**持久化存储**到 `subproject.json`
   - `'user'`：用户通过属性面板或时间线手动修改
   - `'ai'`：通过对话调整由 Agent 修改
   - `null`：新建片段或初始状态
   - **用途**：AI 后续调整该属性时，检测 `lastModifiedBy === 'user'` 则提示"您手动调整了此属性，是否覆盖？"
   - 用户手动修改某属性后，`lastModifiedBy` 自动设为 `'user'`；AI 修改后自动设为 `'ai'`
