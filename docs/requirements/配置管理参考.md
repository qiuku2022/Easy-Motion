# 配置管理参考

> **实施状态（2026-06-26）**：M0–M8 已落地，M9（E2E/打包/性能）进行中。权威入口：仓库根 [README.md](../../README.md)、[开发里程碑与路线图.md](./开发里程碑与路线图.md)。

## 概述

本文档定义 EasyMotion 项目中所有配置文件的字段定义、默认值、热重载策略和配置优先级。

> **配置层级**：命令行参数 > 环境变量 > 用户配置（`~/.easymotion/settings.json`）> 项目配置（`project.json`）> 默认配置

---

## 配置文件清单

| 文件路径 | 格式 | 用途 | 读写方 |
|---------|------|------|--------|
| `~/.easymotion/settings.json` | JSON | 用户级应用设置 | 渲染进程（通过 IPC） |
| `~/.easymotion/secrets.json` | JSON（加密） | 敏感信息（API Key） | 主进程（独占读写） |
| `~/.easymotion/ui-state.json` | JSON | UI 布局状态 | 渲染进程 |
| ~~`~/.easymotion/recent-projects.json`~~ | **已废弃** | ~~最近项目列表（冗余，SQLite 为主）~~ | 由 SQLite `projects.last_opened_at` 字段承担 |
| `项目根目录/project.json` | JSON | 总项目配置 | 主进程（独占写） |
| `项目根目录/subprojects/{id}/subproject.json` | JSON | 子项目配置（含 timeline） | 主进程（独占写） |
| `项目根目录/subprojects/{id}/conversation.json` | JSON | 对话历史 | 主进程 |

### 字段详解

#### 界面（UI）

| 字段 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| `theme` | string | `"system"` | `"light"` / `"dark"` / `"system"` | 界面主题 |
| `language` | string | `"zh-CN"` | `"zh-CN"` / `"en-US"` | 界面语言（预留） |
| `panels.*.visible` | boolean | `true` | `true` / `false` | 面板是否可见 |
| `panels.*.width` | number | 240/280 | ≥ 180 | 面板宽度（像素） |
| `panels.*.height` | number | 180/200 | ≥ 120 | 面板高度（像素） |

#### 编辑器（Editor）

| 字段 | 类型 | 默认值 | 范围 | 说明 |
|------|------|--------|------|------|
| `editor.defaultResolution` | object | `{width:1920,height:1080}` | 常见分辨率 | 新建项目默认分辨率，**统一用 `{width, height}` object 格式**（IPC / DB 也遵循，存储前转为 `"WIDTHxHEIGHT"` 字符串） |
| `editor.defaultFps` | number | 30 | 1-120 | 新建项目默认帧率 |
| `editor.defaultDurationInFrames` | number | 300 | ≥ 1 | 新建项目默认时长 |
| `editor.autoSaveInterval` | number | 30 | 10-300 | 自动保存间隔（秒） |
| `editor.snapToGrid` | boolean | true | - | 是否启用吸附 |
| `editor.snapGridInterval` | number | 1 | ≥ 1 | 吸附间隔（帧） |

#### 预览（Preview）

| 字段 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| `preview.defaultQuality` | string | `"high"` | `"high"` / `"medium"` / `"low"` | 默认预览质量 |
| `preview.defaultScale` | number | 1.0 | 0.5-2.0 | 默认预览缩放 |
| `preview.autoDegrade` | boolean | true | - | 帧率低时自动降级 |
| `preview.fpsWarningThreshold` | number | 15 | 5-30 | 帧率警告阈值 |

#### LLM

| 字段 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| `llm.provider` | string | `"openai"` | `"openai"` / `"anthropic"` / `"azure"` / `"ollama"` | LLM 提供商，**全文档统一使用此命名**（`anthropic` 是官方名称，IPC 层禁止别名 `claude`） |
| `llm.model` | string | `"gpt-4o"` | 根据提供商 | LLM 模型 |
| `llm.maxTokens` | number | 4096 | 1024-8192 | 最大生成 token 数 |
| `llm.temperature` | number | 0.7 | 0.0-2.0 | 生成随机性 |
| `llm.timeout` | number | 60 | 10-300 | 超时时间（秒） |
| `llm.streamResponse` | boolean | true | - | 是否流式响应 |

#### 导出（Export）

| 字段 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| `export.defaultFormat` | string | `"mp4"` | `"mp4"` / `"webm"` / `"gif"` | 默认导出格式（**支持 gif**） |
| `export.defaultQuality` | string | `"medium"` | `"high"` / `"medium"` / `"low"` | 默认导出质量 |
| `export.defaultOutputPath` | string | `"project"` | `"project"` / `"desktop"` / `"last"` | 默认输出路径 |
| `export.autoOpenFolder` | boolean | true | - | 导出后自动打开文件夹 |

#### 素材（Assets）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `assets.importPath` | string | `"last"` | `"last"` / `"project"` / `"desktop"` — 导入对话框默认路径 |
| `assets.autoGenerateThumbnail` | boolean | true | 导入时自动生成缩略图 |
| `assets.thumbnailSize` | number | 200 | 缩略图宽度（像素） |

#### 网络（Network）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `network.checkUpdate` | boolean | true | 启动时检查更新 |
| `network.updateChannel` | string | `"stable"` | `"stable"` / `"beta"` — 更新通道 |
| `network.proxy` | string | null | HTTP 代理地址，如 `http://proxy:8080` |

#### 隐私（Privacy）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `privacy.enableTelemetry` | boolean | false | 匿名使用数据统计 |
| `privacy.enableCrashReport` | boolean | false | 崩溃报告（需用户确认后上报） |
| `privacy.shareUsageData` | boolean | false | 共享使用数据用于产品改进 |

#### 高级（Advanced）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `advanced.debugMode` | boolean | false | 调试模式（显示 FPS、内存等） |
| `advanced.devToolsEnabled` | boolean | false | 允许打开 DevTools（正式版禁用） |
| `advanced.generatorDebounceMs` | number | 500 | Generator 防抖间隔（毫秒） |
| `advanced.logLevel` | string | `"warn"` | `"debug"` / `"info"` / `"warn"` / `"error"` — 日志级别 |
| `advanced.maxUndoSteps` | number | 50 | 撤销栈最大步数 |
| `advanced.maxSnapshots` | number | 50 | 快照最大数量 |

## secrets.json（敏感信息，加密存储）

```json
{
  "version": "1.0",
  "llm_api_keys": {
    "openai": {
      "encrypted": "base64-encoded-encrypted-data",
      "provider": "openai",
      "last_used_at": 1717056000000
    },
    "anthropic": {
      "encrypted": "base64-encoded-encrypted-data",
      "provider": "anthropic",
      "last_used_at": null
    },
    "azure": {
      "encrypted": null,
      "provider": "azure",
      "last_used_at": null
    }
  },
  "proxy_password": "base64-encoded-encrypted-data"
}
```

**加密方式：** AES-256-GCM，密钥由系统钥匙串派生（见 `技术规格.md` → 安全设计）。

**访问规则：**
- 只有主进程可读写此文件
- 渲染进程通过 IPC 请求主进程解密（如验证 Key 时）
- 内存中仅保留临时解密值，使用后立即覆写

## 配置热重载策略

### 支持热重载的配置

以下配置变更后，无需重启应用即可生效：

| 配置项 | 重载方式 | 延迟 |
|--------|----------|------|
| `theme` | 切换 CSS 变量 | 即时 |
| `language` | 重新加载 i18n 资源 | 即时 |
| `preview.defaultQuality` | 重新渲染预览 | 即时 |
| `editor.snapToGrid` | 更新时间线吸附逻辑 | 即时 |
| `llm.*` | 下次 LLM 调用生效 | 即时 |
| `advanced.debugMode` | 显示/隐藏调试面板 | 即时 |
| `advanced.logLevel` | 更新日志过滤器 | 即时 |
| `panels.*` | 调整面板尺寸 | 即时 |

### 需要重启的配置

以下配置变更后，需提示用户重启应用：

| 配置项 | 原因 |
|--------|------|
| `editor.defaultResolution` | 影响新建项目，不影响现有项目 |
| `editor.defaultFps` | 同上 |
| `advanced.generatorDebounceMs` | 影响 Generator 定时器，需重新初始化 |
| `network.proxy` | 影响网络模块初始化 |

## 配置验证规则

### 校验时机

1. **读取时校验**：应用启动读取 `settings.json` 时，校验每个字段类型和范围
2. **写入时校验**：用户通过设置面板修改配置时，前端即时校验
3. **升级时迁移**：应用版本升级时，自动将旧配置迁移到新格式

### 错误处理

```typescript
// 校验示例
function validateSettings(settings: unknown): Settings {
  const result = settingsSchema.safeParse(settings);
  
  if (!result.success) {
    // 1. 记录错误日志
    logger.warn('配置校验失败，使用默认值', result.error.errors);
    
    // 2. 尝试修复（删除无效字段，保留有效字段）
    const repaired = repairSettings(settings, result.error);
    
    // 3. 仍失败则返回默认配置
    return repaired || defaultSettings;
  }
  
  return result.data;
}

// 修复策略
function repairSettings(raw: unknown, errors: ZodError): Settings | null {
  if (typeof raw !== 'object' || raw === null) return null;
  
  // 遍历错误，删除无效字段
  const fixed = { ...raw };
  for (const error of errors.errors) {
    if (error.path.length > 0) {
      delete (fixed as any)[error.path[0]];
    }
  }
  
  // 与默认值合并
  return { ...defaultSettings, ...fixed };
}
```

## 配置与数据的关系

**单一数据源（Single Source of Truth）**：

| 数据类型 | 唯一存储位置 | 备注 |
|---------|-------------|------|
| 用户级应用设置 | `~/.easymotion/settings.json` | **唯一源**，启动时加载到内存 |
| 窗口位置/UI 状态 | `~/.easymotion/ui-state.json` | 与 settings.json 分开，便于热重载 |
| 敏感信息（API Key） | `~/.easymotion/secrets.json` | AES-256-GCM 加密 |
| 最近项目列表 | SQLite `projects` 表 | 通过 `last_opened_at` 排序 |
| 项目级配置 | `项目根目录/project.json` | 主进程独占写 |
| 子项目配置 | `子项目目录/subproject.json` | 主进程独占写 |
| 对话历史 | `子项目目录/conversation.json` | 主进程独占写 |

> **2026-06 重构说明**：本版本起，**SQLite 不再承担应用设置存储**（删除 `app_settings` 表）。所有应用设置通过 `settings.json` 单一来源管理，启动时加载到内存，修改时立即写回文件。这样消除了之前"settings.json ↔ app_settings 表"双重存储带来的同步复杂性和冲突风险。

```
用户修改设置
  ↓
[设置面板]（渲染进程）
  ├── 即时生效的配置 → 直接更新 Store
  └── 需重启的配置 → 标记为 "pending_restart"
        ↓
用户点击"保存"
  ├── 写入 ~/.easymotion/settings.json（**唯一源**）
  ├── 即时生效的配置 → 广播更新事件
  └── 需重启的配置 → 弹窗提示"部分设置需要重启后生效"

[主进程]
  ├── 读取 settings.json
  ├── 校验字段
  └── ⚠️ 不再同步到 SQLite（已废弃 app_settings 表）
```
